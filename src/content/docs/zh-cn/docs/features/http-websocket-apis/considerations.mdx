## Markers

有些方法返回的数据量太大，无法高效地放进单个响应中。当结果超出响应限制时，响应里会包含一个 `marker` 字段。这个字段允许你在后续请求中继续获取额外的数据页。要继续拉取数据，只需在下一个请求中包含上一条响应返回的 `marker` 值。如果响应中没有 `marker`，则表示你已经到达数据集末尾。

`marker` 字段的格式是故意未指定的。每个服务器都可以按需定义自己的 `marker`，它可能是字符串、嵌套对象或其他类型。`marker` 的格式不仅可能因服务器而异，也可能因同一服务器上的不同方法而不同。每个 `marker` 都是临时的，大约 10 分钟后可能失效。

## Rate Limit

`xahaud` 服务器会对使用公共 API 的客户端施加速率限制，以防止请求过量。限流基于客户端的 IP 地址进行，因此多个共享同一个 [network address translation (NAT)](https://en.wikipedia.org/wiki/Network_address_translation) 的客户端，会共享与其公网 IP 对应的同一速率限制。

当客户端接近速率限制时，服务器会在 API 响应的顶层加入一个 `"warning": "load"` 字段。这个警告不会出现在每一条响应中，但服务器在断开客户端之前可能会发送数次。以 admin 身份连接的客户端不受速率限制影响。

如果客户端超过速率限制，服务器会断开该客户端，并临时阻止来自该 IP 地址的后续请求。WebSocket 和 JSON-RPC APIs 对断开连接的处理方式不同，详见下文说明。
