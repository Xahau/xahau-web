`cbak` 是一个由开发者定义的回调函数，用于在此前发射的交易得到最终结果后执行。

当 `xahaud` 处理某个由 Hook 发射的交易并在之后触发回调时，会调用 `cbak`。

在 `cbak` 中：

- 可以继续修改状态
- 可以继续发射新的交易
- 不能回滚已经发生的交易

当 `cbak` 执行时，先前发射的交易会被视为新的原始交易上下文。

## C

```c
int64_t cbak(uint32_t what);
```

## JavaScript

```javascript
type Callback = (reserved: number) => number
```

## 参数

### `what`

表示回调原因的保留参数。

- `0` 表示发射交易已被账本接受
- `1` 表示发射交易在过期前未被接受

## 返回值

返回任意由开发者定义的整数代码，该值会记录到元数据中。

## 说明

并非所有 Hook 都需要定义 `cbak`，但如果你的逻辑依赖发射交易的后续结果，就需要实现它。
