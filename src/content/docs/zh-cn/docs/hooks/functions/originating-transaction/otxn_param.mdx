---
title: otxn_param
description: 获取指定名称的 Invoke 交易参数值
---
import { Tabs, TabItem } from '@astrojs/starlight/components';

### 行为

<Tabs>
<TabItem label="C">
* 在原始交易中查找 `read_ptr` 指定名称的参数值
* 将该参数值写入 `write_ptr`
</TabItem>

<TabItem label="JavaScript">
* 查找原始交易中指定名称的参数值。
* 成功时返回指定参数值，查找失败时返回 ErrorCode。
</TabItem>
</Tabs>

### 定义

<Tabs>
<TabItem label="C">
```c
int64_t otxn_param (
    uint32_t write_ptr,
    uint32_t write_len,
    uint32_t read_ptr,
    uint32_t read_len
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function otxn_param(name: ByteArray | HexString): ErrorCode | ByteArray
```
</TabItem>
</Tabs>

### 示例

<Tabs>
<TabItem label="C">
```c
uint8_t pname[] = {0xCAU, 0xFEU};
uint8_t pvalue[32];
int64_t value_len = 
    otxn_param(pvalue, 32, pname, 2);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
const pname = [0xCA, 0xFE]
const pvalue = otxn_param(pname)
```
</TabItem>
</Tabs>

### 参数

<Tabs>
<TabItem label="C">
| Name       | Type      | Description |
| ---------- | --------- | ----------- |
| write_ptr | uint32_t | 指向用于存储输出的合适缓冲区，至少应为 32 字节 |
| write_len | uint32_t | 输出缓冲区长度 |
| read_ptr  | uint32_t | 指向包含参数名称缓冲区的指针 |
| read_len  | uint32_t | 参数名称长度 |
</TabItem>

<TabItem label="JavaScript">
| Name | Type                   | Description |
| ---- | ---------------------- | ----------- |
| name | ByteArray \| HexString | 要查找的参数名称，可以是 ByteArray 或字符串 |
</TabItem>
</Tabs>

### 返回码

<Tabs>
<TabItem label="C">
| Type     | Description |
| -------- | ----------- |
| int64_t | <p>写入的字节数<br /><br />若为负数，则表示错误：<br /><code>OUT_OF_BOUNDS</code><br />- 指针或长度超出 Hook 内存范围。<br /><br /><code>DOESNT_EXIST</code><br />- 指定参数不存在或为 null<br /><br /><code>TOO_SMALL</code><br />- 参数名不能为空<br /><br /><code>TOO_BIG</code><br />- 参数名超过 32 字节</p> |
</TabItem>

<TabItem label="JavaScript">
| Type                   | Description |
| ---------------------- | ----------- |
| ErrorCode \| ByteArray | 成功时返回指定参数值，失败时返回 ErrorCode。 |
</TabItem>
</Tabs>
