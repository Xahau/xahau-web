---
title: meta_slot
description: 将原始交易的元数据加载到一个槽位中
---
import { Tabs, TabItem } from '@astrojs/starlight/components';

### 行为

* 如果 Hook 正在[弱执行](/docs/hooks/concepts/weak-and-strong)，则将原始交易的元数据放入指定槽位；如果未指定槽位，则放入一个新的槽位

### 定义

<Tabs>
<TabItem label="C">
```c
int64_t meta_slot (
    uint32_t slot_no
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function meta_slot(slotno: number): ErrorCode | number
```
</TabItem>
</Tabs>

### 示例

<Tabs>
<TabItem label="C">
```c
int64_t meta_slot_no = 
		meta_slot(0);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
const meta_slot_no = meta_slot(0)
```
</TabItem>
</Tabs>

### 参数

<Tabs>
<TabItem label="C">
| Name     | Type      | Description |
| -------- | --------- | ----------- |
| slot_no | uint32_t | 要放入的槽位编号；若为 0，则选择下一个可用槽位 |
</TabItem>

<TabItem label="JavaScript">
| Name   | Type   | Description |
| ------ | ------ | ----------- |
| slotno | number | 要读取元数据的槽位编号 |
</TabItem>
</Tabs>

### 返回码

<Tabs>
<TabItem label="C">
| Type     | Description |
| -------- | ----------- |
| int64_t | <p>原始交易被放入的槽位号<br /><br /><code>INVALID_ARGUMENT</code><br />- 指定的槽位号超过最大可能槽位号<br /><br /><code>NO_FREE_SLOTS</code><br />- 没有空闲槽位可用于放置原始交易。<br /><br /><code>PREREQUISITE_NOT_MET</code><br />- Hook 正在<a href="/docs/hooks/concepts/weak-and-strong">强执行</a>，因此没有可用的交易元数据。</p> |
</TabItem>

<TabItem label="JavaScript">
| Type                | Description |
| ------------------- | ----------- |
| ErrorCode \| number | 返回错误码或包含元数据的槽位号。 |
</TabItem>
</Tabs>
