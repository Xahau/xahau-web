---
title: state_set
description: 为指定键和值设置 Hook State
---
import { Tabs, TabItem, Aside } from '@astrojs/starlight/components';

### 行为

<Tabs>
<TabItem label="C">
* 从 `kread_ptr` 读取一个 32 字节 Hook State 键
* 从 `read_ptr` 读取任意数据作为值
* 使用该值更新 Hook State 键
</TabItem>

<TabItem label="JavaScript">
* 使用指定值和键设置 Hook State
* 成功时返回写入长度，失败时返回负错误码
</TabItem>
</Tabs>

### 定义

<Tabs>
<TabItem label="C">
```c
int64_t state_set (
    uint32_t read_ptr,
    uint32_t read_len,
    uint32_t kread_ptr,
    uint32_t kread_len  
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function state_set(
    value: ByteArray | HexString | undefined | null,
    key: ByteArray | HexString
  ): ErrorCode | number
```
</TabItem>
</Tabs>

### 参数

<Tabs>
<TabItem label="C">
| Name       | Type      | Description |
| ---------- | --------- | ----------- |
| read_ptr  | uint32_t | 指向要写入 Hook State 的数据；若为 `0` 则删除该键数据 |
| read_len  | uint32_t | 数据长度；若为 `0` 则删除该键数据 |
| kread_ptr | uint32_t | Hook State 键指针 |
| kread_len | uint32_t | 键长度（应为 32） |

<Aside type="caution">
Xahaud 内部会限制最大 Hook 数据大小。撰写本文时，在 mainnet 上该值硬编码为 256 字节，未来可能改为由验证者投票决定。
</Aside>
</TabItem>

<TabItem label="JavaScript">
| Name  | Type                   | Description |
| ----- | ---------------------- | ----------- |
| value | ByteArray or HexString | 要持久化的数据值 |
| key   | ByteArray or HexString | 要写入值的 Hook State 键 |
</TabItem>
</Tabs>

### 返回码

<Tabs>
<TabItem label="C">
| Type     | Description |
| -------- | ----------- |
| int64_t | <p>写入 Hook State 的字节数。<br /><br />若为负数，则表示错误：<br /><code>OUT_OF_BOUNDS</code><br />- 指针或长度超出 Hook 内存范围。<br /><br /><code>TOO_BIG</code><br />- <code>kread_len</code> 大于 32，或 <code>read_len</code> 大于最大 Hook 数据大小。<br /><br /><code>TOO_SMALL</code><br />- <code>kread_len</code> 为 0。</p> |
</TabItem>

<TabItem label="JavaScript">
| Type                | Description |
| ------------------- | ----------- |
| number or ErrorCode | 返回写入 Hook State 的字节数，失败时返回错误码。 |
</TabItem>
</Tabs>
