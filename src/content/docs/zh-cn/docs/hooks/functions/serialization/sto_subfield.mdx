import { Tabs, TabItem, LinkButton, Aside } from '@astrojs/starlight/components';

### 概念

<LinkButton href="/docs/hooks/concepts/serialized-objects">序列化对象</LinkButton>

### 行为

<Tabs>
<TabItem label="C">
* 解析 `read_ptr` 指向的 STObject
* 查找 `field_id` 指定字段
* 如果找到该字段：
*
  1. 如果它是数组，则返回包含起始和结束字节在内的数组位置与长度
*
  2. 如果它不是数组，则返回字段负载部分的位置与长度（不含前导字节）

<Aside type="caution" title="Field ID 编码">
`sto_` API 接收的 `field_id` 编码方式为：`(type << 16U) + field`
因此 type 1、field 2 会写作 `0x10002U`。
</Aside>
</TabItem>

<TabItem label="JavaScript">
* 解析 `sto` 中的 STObject
* 查找 `field_id` 指定字段
</TabItem>
</Tabs>

### 定义

<Tabs>
<TabItem label="C">
```c
int64_t sto_subfield (
    uint32_t read_ptr,
    uint32_t read_len,
    uint32_t field_id
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function sto_subfield(
    sto: ByteArray | HexString,
    field_id: number
  ): ErrorCode | bigint
```
</TabItem>
</Tabs>

### 参数

<Tabs>
<TabItem label="C">
| Name      | Type      | Description |
| --------- | --------- | ----------- |
| read_ptr | uint32_t | 包含 STObject 的缓冲区指针 |
| read_len | uint32_t | STObject 长度 |
| field_id | uint32_t | 要查找字段的 `sf` 编码 |
</TabItem>

<TabItem label="JavaScript">
| Name      | Type                   | Description |
| --------- | ---------------------- | ----------- |
| sto       | ByteArray or HexString | 要提取子字段的 STO 对象（二进制账本数据） |
| field_id | number                 | 要提取的字段 ID |
</TabItem>
</Tabs>

### 返回码

<Tabs>
<TabItem label="C">
| Type     | Description |
| -------- | ----------- |
| int64_t | <p>字段在指定缓冲区中的位置：<br />- 高 32 位为偏移<br />- 低 32 位为长度<br /><br />若为负数，则表示错误：<br /><code>OUT_OF_BOUNDS</code><br />- 指针或长度超出 Hook 内存范围。<br /><br /><code>TOO_SMALL</code><br />- 输入缓冲区过小，不可能包含合法对象。<br /><br /><code>DOESNT_EXIST</code><br />- 提供对象中不存在该字段。<br /><br /><code>PARSE_ERROR</code><br />- 提供的 STObject 格式错误，或并非 STObject。</p> |
</TabItem>

<TabItem label="JavaScript">
| Type               | Description |
| ------------------ | ----------- |
| bigint / ErrorCode | 返回字段位置结果：高 32 位为偏移，低 32 位为长度；失败时返回错误码。 |
</TabItem>
</Tabs>
