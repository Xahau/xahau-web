---
title: sto_validate
description: 验证 STObject
---
import { Tabs, TabItem, LinkButton } from '@astrojs/starlight/components';

### 概念

<LinkButton href="/docs/hooks/concepts/serialized-objects">序列化对象</LinkButton>

### 行为

<Tabs>
<TabItem label="C">
* 解析 `read_ptr` 指向的 STObject
* 如果序列化有效则返回 `1`，否则返回 `0`
</TabItem>

<TabItem label="JavaScript">
* 接收要验证的 `blob`（例如序列化交易）
* 如果 STObject 有效返回 `1`，无效返回 `0`，验证失败则返回错误码
</TabItem>
</Tabs>

### 定义

<Tabs>
<TabItem label="C">
```c
int64_t sto_validate (
    uint32_t read_ptr,
    uint32_t read_len
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function sto_validate(blob: ByteArray | HexString): ErrorCode | number
```
</TabItem>
</Tabs>

### 示例

<Tabs>
<TabItem label="C">
```c
int64_t result = 
  sto_validate(tx_out, sizeof(tx_out));

if (tx_len <= 0)
    rollback("Invalid STO.", 12, 1);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
const result = sto_validate(tx_out)

if (result <= 0)
    rollback("Invalid STO.", 1)
```
</TabItem>
</Tabs>

### 参数

<Tabs>
<TabItem label="C">
| Name      | Type      | Description |
| --------- | --------- | ----------- |
| read_ptr | uint32_t | 用于读取源 STObject 的缓冲区 |
| read_len | uint32_t | 源对象长度 |
</TabItem>

<TabItem label="JavaScript">
| Name | Type                   | Description |
| ---- | ---------------------- | ----------- |
| blob | ByteArray \| HexString | 要验证的 blob（例如序列化交易） |
</TabItem>
</Tabs>

### 返回码

<Tabs>
<TabItem label="C">
| Type     | Description |
| -------- | ----------- |
| int64_t | <p>如果 <code>read_ptr</code> 指向的是合法 STObject，则返回 <code>1</code>；否则返回 <code>0</code>。<br /><br />若为负数，则表示错误：<br /><code>OUT_OF_BOUNDS</code><br />- 指针或长度超出 Hook 内存范围。</p> |
</TabItem>

<TabItem label="JavaScript">
| Type                | Description |
| ------------------- | ----------- |
| ErrorCode \| number | 如果 STObject 有效返回 `1`，无效返回 `0`，验证失败则返回错误码。 |
</TabItem>
</Tabs>
