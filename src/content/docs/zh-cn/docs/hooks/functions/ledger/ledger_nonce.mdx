---
title: ledger_nonce
description: 为发射交易生成一个 32 字节 nonce
---
import { Tabs, TabItem } from '@astrojs/starlight/components';

### 行为

<Tabs>
<TabItem label="C">
* 向 `write_ptr` 写入一个 32 字节随机值
</TabItem>

<TabItem label="JavaScript">
* 获取当前账本的 nonce。
</TabItem>
</Tabs>

### 定义

C

<Tabs>
<TabItem label="C">
```c
int64_t ledger_nonce (
    uint32_t write_ptr,
    uint32_t write_len
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function ledger_nonce(): ErrorCode | ByteArray
```
</TabItem>
</Tabs>

### 示例

<Tabs>
<TabItem label="C">
```c
uint8_t n[32];
int64_t bytes_written = 
    ledger_nonce(n, 32);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
const nonce = ledger_nonce()
```
</TabItem>
</Tabs>

### 参数

<Tabs>
<TabItem label="C">
| Name       | Type      | Description |
| ---------- | --------- | ----------- |
| write_ptr | uint32_t | 指向合适大小输出缓冲区的指针，至少应为 32 字节 |
| write_len | uint32_t | 输出缓冲区长度 |
</TabItem>

<TabItem label="JavaScript">

</TabItem>
</Tabs>

### 返回码

<Tabs>
<TabItem label="C">
| Type     | Description |
| -------- | ----------- |
| int64_t | <p>写入的字节数<br /><br />若为负数，则表示错误：<br /><code>OUT_OF_BOUNDS</code><br />- 指针或长度超出 Hook 内存范围。</p> |
</TabItem>

<TabItem label="JavaScript">
| Type                   | Description |
| ---------------------- | ----------- |
| ErrorCode \| ByteArray | 如果出错返回错误码，成功时返回表示当前账本 nonce 的数组。 |
</TabItem>
</Tabs>
