## 从源代码构建软件

开发者编写软件时，通常使用适合人类阅读的编程语言。然而，计算机直接运行这种人类可读代码并不高效。因此，需要将软件从人类可读代码“编译”为字节码，以便计算机快速读取和解释。

使这个过程更复杂的是，开发者通常会依赖其他开发者编写的软件，也就是所谓的 “dependencies”，因为一个软件包依赖另一个软件包。为了让这些 dependencies 被包含进最终产物，它们必须在编译前与代码完成“链接”。链接过程本身也可能非常繁重，因为原始软件包的依赖项本身也可能有自己的依赖项（这些依赖又可能继续依赖其他组件，形成多层结构）。开发者通常依赖 Conan 之类的工具，来简化查找、配置和链接依赖项的流程。

确保依赖齐全、将软件编译成适合机器运行的格式，并对最终产物运行测试的整个过程，被称为 “building”。

由于 `xahaud` 是以[开源软件形式发布在 GitHub 上](https://github.com/Xahau/xahaud)的，任何人都可以查看、审计、修改、构建并运行其代码。从源代码构建能够让你确信所运行的产品确实基于公开可见、并且具备完整审计历史的源代码。因此，重视最高级别安全性的 validator 运营者应考虑从源码自行构建。

## 依赖项

为了保证兼容性，必须特别关注构建过程中所使用的软件版本。`xahaud` 使用 C++20 编写。

当前构建 `xahaud` 所需条件包括：


| 依赖项 | 版本 |
| :----- | :--- |
| Python3 | >= 3.7 |
| GCC / G++ | >= 14.x |
| CMake | >= 3.16 |
| Conan2 | 2.x |
| openssl | 1.1.1 |
| boost | 1.86.0 |
| SQLite3 | 3.42.0 |
| Snappy* | 1.1.10 |
| SOCI* | 4.0.3 |
| WasmEdge* | 0.11.2 |

\* 已包含在 Xahau/xahaud GitHub 仓库中。
