## 1. 摘要

1. Xahau Ledger（Xahau）是 XRPL 生态系统的智能合约侧链。它是 XRP Ledger（XRPL）开源 `rippled` 代码库的一个分叉版本，保留了 XRPL 的全部实用且创新的特性，包括其环境可持续性，同时对代码库进行了调整与升级，以支持智能合约。
2. Xahau 的核心特性包括：
   * **XRPL 核心能力：** Xahau 保留了使 XRPL 成为最持久、最受欢迎网络之一的关键特性，包括 **XRP Ledger Consensus Protocol**（此前称为 Ripple Protocol Consensus Algorithm）、**DEX**，以及通过以原生代币收取并销毁费用来保护账本免受垃圾交易和膨胀影响的逻辑；但它使用更简洁、也更干净的 **URITokens** 取代了较新的 **XLS-20 NFTs**。
   * **Hooks 智能合约：** Xahau 最大的新特性是 Hooks，即为 `rippled` 提供的智能合约实现。Hooks 是安装在账户上的小型代码片段，会在账户发送或接收的交易最终确认之前，对这些交易施加规则。
   * **原生代币与更优 Tokenomics：** Xahau 将由其原生代币 XAH（货币代码：XAH）提供安全保障，并通过更合理的代币经济模型来奖励验证者并支持智能合约。
   * **Genesis Hook Governance Game：** Xahau 的 Genesis 账户由一个 Hook 驱动，该 Hook 会调节包括新 XAH 发行在内的多项事务，而这个 Hook 则由一个双层治理博弈管理，参与者最多可包括 20 个独立拥有的验证者。
3. 这些特性共同构成了 XRPL 生态系统中的一个新网络，通过实现 Hooks amendment，为 XRPL 生态带来快速、低成本且安全的智能合约，并由得到适当激励的验证者与开发者社区提供支持。
4. Xahau Ledger 的开发由 Xahau Launch Alliance 推动完成。该联盟由 XRPL 社区中五家经验丰富且投入坚定的实体组成，它们在没有 ICO 的情况下承担了建立这一新网络的成本与风险。
5. Xahau 将在完全可用且去中心化的状态下启动，启动时将拥有 10 个以上独立拥有的验证者，这些验证者由 XRPL 生态中具有代表性的参与者运营，包括开发者、交易所和长期社区成员。

## 2. Xahau：承袭 XRPL 的基因

1. `xahaud` 是运行 Xahau 的软件，它是 XRPL 开源代码库 `rippled` 的一个分叉。它保留了 XRPL 自 2012 年上线以来使其成为最受欢迎、最持久网络之一的关键特性。

### XRP Ledger Consensus Protocol

2. Xahau 使用 XRP Ledger Consensus Protocol（此前称为 Ripple Protocol Consensus Algorithm），有时也称为 Proof of Association（PoA）。它旨在克服传统共识机制（如 Proof of Work，PoW；或 Proof of Stake，PoS）的局限性，尤其是 PoW 带来的高能耗问题。
3. XRP Ledger Consensus Protocol（此前称为 Ripple Protocol Consensus Algorithm）的工作方式如下：
   * **验证者提案：** 每个验证者都会提出一组新的交易，即其认为应当被纳入下一本账本的“候选集合”。该候选集合既包括新交易，也包括先前未被纳入已验证账本的交易。
   * **达成一致阶段：** 验证者彼此通信，交换并评估候选集合。它们会独立验证其他验证者所提出交易的有效性和顺序。通过一系列迭代轮次，验证者尝试收敛到一个由多数成员认可的单一候选集合。
   * **最终确认：** 一旦验证者中的超多数（至少 80%）对某个特定候选集合达成一致，该集合就会被视为“最终确认”。已最终确认的候选集合将成为下一本账本的基础。
   * **账本关闭：** 当前账本关闭，并根据最终确认候选集合中的交易生成一本新账本。在 XRPL 中，这一过程大约每 3 到 5 秒发生一次，因此能够实现快速交易结算。
4. 在 XRP Ledger Consensus Protocol（此前称为 Ripple Protocol Consensus Algorithm）下，每个验证者都会指定一个 UNL（Unique Node List，唯一节点列表），即其所信任的其他验证者列表。规范账本通过所有验证者 UNL 的重叠部分来确定。
5. 为了促进围绕适当 UNL 的社会共识，包括 XRPL Foundation 在内的多个实体会发布推荐 Validator List（VL），即它们认为足够可信的验证者列表。遵循广泛接受的 dUNL 有助于确保你停留在主账本上，而不会无意中跟随一个无关紧要的分叉。Xahau 也将采用类似的社会共识方式来定义 dUNL，并由诸如 XRPL Foundation 之类的组织发布其推荐 VL。
6. XRP Ledger Consensus Protocol（此前称为 Ripple Protocol Consensus Algorithm）提供了多项优势，包括高扩展性、低能耗，以及对审查和网络分叉的抗性。通过利用一组受信任的验证者和这一独特的共识算法，Xahau 在保持去中心化与可靠性的同时，实现了快速且安全的交易处理。

### DEX

7. Xahau 保留了 XRPL 的限价订单簿去中心化交易所（DEX）版本，允许用户直接在账本上交易和交换资产。DEX 借助 Xahau 去中心化且无需信任的架构，使用户无需中介即可进行点对点资产交易。
8. 总体来说，这一去中心化交易所的工作方式如下：
   * **创建订单：** 用户可以在 DEX 上创建买单或卖单，指定其想交易的资产、数量以及愿意接受或支付的价格。
   * **订单撮合：** DEX 的订单簿会记录所有未成交的买单和卖单。当新订单被提交时，DEX 会根据价格和数量自动与现有订单进行撮合。
   * **跨币种交易：** DEX 支持跨币种交易，使用户能够在不同资产之间进行兑换。为实现这一点，DEX 使用账本原生货币作为桥接货币，使用户可以先将一种资产兑换为原生货币，再兑换为另一种资产。
   * **路径查找功能：** 路径查找是 DEX 的关键组成部分。它帮助用户在将一种资产转换为另一种资产时找到最高效的交易路径。例如，如果用户希望将资产 A 换成资产 D，但市场上没有直接交易对，路径查找功能将搜索最佳交易序列（如 A 到 B、B 到 C、C 到 D），以最小滑点和费用实现所需兑换。
   * **自动执行订单：** 一旦交易被撮合，DEX 会自动执行该交易。账本的共识机制会确保交易被验证并写入账本，从而提供高水平的安全性与不可篡改性。
   * **无需信任且去中心化：** DEX 以无需信任、去中心化的方式运行。它不依赖中心化交易所，也不要求用户将资金存入第三方钱包。相反，用户在整个交易过程中始终控制自己的资产，从而降低对手方风险。
   * **低费用与快速结算：** 与传统中心化交易所相比，在 DEX 上交易仅需极少费用。Xahau 的共识协议支持快速结算，使交易能够近乎即时完成。
9. 借助这些特性，DEX 为用户提供了一个便捷、安全的平台，用于进行点对点代币交易。通过利用 Xahau 无需信任和去中心化的特性，DEX 在保持用户控制权和隐私的同时，提供高效的跨代币交易与流畅的订单撮合能力。

### 被销毁的交易费用

10. 为了防止垃圾交易与账本膨胀，XRPL 以其原生货币收取交易费用和账户储备，并将所有交易费用直接销毁，而不是重新分配给验证者。Xahau 也遵循这种“收取并销毁费用”的模式，以同样的方式保护自身。
11. 虽然这是保护账本的一种成熟机制，但它也会对 Xahau 的 Tokenomics 产生影响，后文将进一步讨论。Xahau 智能合约所销毁的费用虽然依然不高，但会远高于 XRPL 上的标准交易费用，而且波动性更大。

### 以 URIToken NFT 取代 XLS-20 NFT

12. 与引入到 XRPL 的 XLS-20 NFT 标准不同，后者依赖将 NFT 压缩进链上“pages”，而 Xahau 将使用 URIToken 对象。
13. URITokens（Uniform Resource Identifier Token 的缩写）是在 Xahau 中表示和管理 NFT 及其元数据与所有权信息的一种创新方式。每个代币都是一级账本对象，拥有一个唯一地址，并且该地址不会随着当前所有者变化而变化。
14. 相比 XLS-20 NFT 标准，URIToken NFTs 具有许多优势，包括：
   * **轻量且高效：** URITokens 是一级链上对象，可以被创建、销毁、转移、买卖，并且能够轻松与 Hooks 互操作。每个代币的交易仅限于一个卖出报价，该报价可以由指定目标账户接受；如果未指定目标账户，则公众均可接受。不存在 brokered mode。
   * **互操作性与不可变性：** URITokens 内置可扩展的元数据 JSON 标准，并支持可选的 Digest 字段（一个哈希，用于使 NFT 内容不可变）。这为开发者和创作者提供了灵活性，使其可以自由选择偏好的存储方案，例如 IPFS 或传统 Web 服务器。它也促进了与现有 NFT 标准和平台的互操作，更易融入更广泛的 NFT 生态。
   * **改进的用户体验：** URITokens 可以通过其 keylet（链上地址）轻松定位，而该地址不依赖代币当前所有者。NFT 的元数据和所有权信息也可以通过标准 Web 协议轻松访问和更新。用户只需点击与 NFT 关联的 URI，即可查看包括图像、描述和来源信息在内的详细内容。这种方式简化了用户体验，并降低了在 Xahau 上与 NFT 交互的复杂度。
15. 因此，URITokens 为 Xahau 上的 NFT 提供了一种轻量、灵活且成本高效的解决方案。

## 3. Hooks：面向 XRPL 生态系统的智能合约

1. Hooks 是专为 XRPL 生态系统开发的一种智能合约方案。Hooks 是定义在 Xahau 账户上的小型高效代码片段，会在发送到该账户或由该账户发出的交易被最终写入账本之前执行逻辑。因此，Hooks 为开发者在 Xahau 上创建和部署智能合约提供了方式，开启了去中心化应用（dApps）与自动化交易的广泛可能性。
2. Xahau 通过侧链形式实现 Hooks amendment，使 XRPL 生态系统能够更早从 Hooks 中受益，而不是继续等待。这也使 XRPL 社区能够更好地评估未来是否以及如何在 XRPL 主网上实现 Hooks。

### Hooks 的优势

3. 相比其他智能合约方案，Hooks 的一些关键优势包括：
   * **简单与高效：** 与传统智能合约方案相比，Hooks 更轻量、更高效。Xahau 的底层架构聚焦于交易结算和快速共识，这确保了智能合约可以在不牺牲性能的前提下快速执行。
   * **原生集成：** Hooks 是一种第一层智能合约方案，因此原生存在于账本内部，使开发者能够创建与链上对象、余额和交易直接交互的合约。与第二层智能合约方案相比，这种紧密集成简化了开发流程，并确保了与 Xahau 生态系统的无缝互操作。
   * **低费用与高扩展性：** Hooks 充分利用 Xahau 的低交易费用与高扩展能力。与大多数链一样，Xahau 每秒可处理大量交易，适合需要快速且低成本执行智能合约的应用场景。
   * **安全性与可靠性：** Hooks 继承了 XRPL 强大的安全性与可靠性。驱动 Xahau 的 XRP Ledger Consensus Protocol（此前称为 Ripple Protocol Consensus Algorithm）确保网络由受信任验证者达成共识并去中心化运行。这一强安全基础为开发者和用户都提供了信心。
   * **生态兼容性：** Hooks 被设计为兼容现有 XRPL 工具与服务，使开发者更容易将智能合约集成到自己的应用中。这种兼容性使其能够与 XRPL 的其他功能（如去中心化交易所和支付通道）无缝协同。
   * **账户去中心化：** 借助 Hooks，Xahau 账户可以完全自治，按照其 Hook 逻辑发送与接收交易，而不再依赖外部私钥持有人手动授权交易。

### 示例用例

4. 通过为 XRPL 生态系统引入智能合约能力，Hooks 为希望构建去中心化应用并自动化交易的开发者提供了强大且高效的解决方案。Hooks 可实现的功能示例包括：
   * **白名单：** 启用 Hook 的账户可以通过代码限制，只接受来自白名单账户的交易，或只向白名单账户发送交易，从而防止欺诈或受制裁交易。
   * **黑名单：** 启用 Hook 的账户可以通过代码拒绝接收或发送任何涉及被禁止（黑名单）账户的交易。
   * **更复杂的托管：** 启用 Hook 的账户可以持有来自指定账户的任意资产，并仅在收到符合预设条件的有效交易指令时，才转发或退还这些资产。
   * **自动注册系统：** 启用 Hook 的账户可以作为自动注册系统，铸造与赎回 URI NFTs，作为注册或注册权利的凭证。
   * **自主国库：** 发行货币可以通过存入由自身拥有、且带有 Hook 的账户中来模拟去中心化、无对手方风险的资产，该 Hook 会根据公开披露且预先设定的发行计划来发出该货币。
5. 简单性、低费用、可扩展性、安全性和生态兼容性的结合，意味着 Xahau 为 XRPL 生态系统提供了一个具吸引力的智能合约方案，并让用户能够更早接触它。Xahau 在 Hooks 上的实践经验，也将帮助社区决定未来是否以及如何在 XRPL 上实现 Hooks。

## 4. XAH：原生代币与 Tokenomics

1. Xahau 由其原生代币 XAH 提供安全保障。其主要特征如下：
   * **原生代币：** XAH
   * **Ticker：** XAH
   * **最小单位：** 1 drop，即 0.000001 XAH
   * **功能：** XAH 是网络货币，是一种用于购买网络服务的实用型代币。与 XRPL 类似，Xahau 上的交易会产生以 XAH 计价的费用或储备，用于防止账本遭受垃圾交易和膨胀。
   * **初始流动性：** 向启动参与方分发 6 亿。
   * **最大供应量：** 不设上限。
   * **发行机制：** XAH 的供应量由 Xahau 共识协议中启用的 amendments 决定：
   * **每月余额调整：** 为了将部分被交易费用销毁的 XAH 重新投入流通，每个活跃用户都可以按其月平均余额申领约 `0.34%` 的账户余额增长（相当于年化 4% 复利）。这一发行机制由 Protocol Governance Game 控制，并依赖 Hooks、BalanceRewards 与 XahauGenesis amendments 的组合。
   * **每月 Seat 奖励：** 为了将部分被交易费用销毁的 XAH 重新投入流通，并激励积极且恰当的治理，与当月已申领的全部 Monthly Balance Adjustments 等额的奖励，也会发放给 Governance Game 验证者。

### 初始分配

2. 启动方为 Xahau 的启动投入了大量时间和资源。初始分配通过协议层面的 XahauGenesis amendment 提供。
3. 协议将按如下方式发放共计 6 亿 Xahau XAH：启动时，Governance Game 的 8 个验证者 Seat 中每个 Seat 获得 1200 万。
   * 额外向 GateHub 发放 1600 万，用于 DEX 稳定币流动性冷启动。
   * 额外向 XRPL Labs（Xaman）发放 1.6 亿，用于知识产权补偿。Xaman 承担了 Hooks 与 `xahaud` 开发中的主要技术和财务负担，这部分额外分配就是对该努力的补偿。
   * 向 XRPL Foundation（现为 INFTF）发放 3.28 亿，以帮助确保 XRPL Protocol（现为 Xahau）生态系统未来的健康发展。

### 每月余额调整

4. Xahau 需要应对的一项挑战，是智能合约相较普通 XRPL 交易所带来的额外成本。
5. 在 XRPL 上，交易费用最低可低至 1 drop，即 0.000001 XRP。相对于 1000 亿 XRP 的总供应量而言，这是一个微不足道但足够有效的成本。而 Hooks 交易的设置成本与触发成本都可能高出若干数量级。此外，XRPL 的每笔交易都是标准化的，而 Hooks 在复杂度和成本上可能差异极大。为了适当地防范 Hooks 带来的垃圾交易与账本膨胀，账本必须根据每笔 Hook 交易的体积与逻辑复杂度动态且适当地收费。
6. 结果是，作为一条销毁费用的智能合约链，XAH 很容易被快速销毁到接近零，且销毁速度肯定会远高于 XRPL 上等值 XRP 的销毁速度，这可能导致一种扭曲激励：人们倾向囤积 Xahau，而不是将其用于部署与运行 Hooks。
7. 为防范这些风险，Xahau 需要一个协议层面的机制，把被销毁的代币重新投放回流通中，但这种机制又不能简单地鼓励更高费用。这正是每月余额调整的目的。
8. 每月余额调整的工作方式如下：
   * 每个月，每个 Xahau 账户都可以选择根据其平均 XAH 余额（自上次调整以来计算）向协议申领一次调整。调整金额相当于年化 4% 复利（大约每月 0.34%）。
   * 用户可以在距上次申领满 30 天后，随时向 Genesis 账户发送一笔 `RewardClaim` 交易来申领余额调整。这表明他们是活跃用户。
   * 在接收到该申领交易后，协议会通过 BalanceRewards 和 XahauGenesis amendments，将相应的余额调整增加到用户账户中。
   * 未申领的余额调整会被视为因不活跃而放弃。
   * 4% 是启动时的初始利率，但如果未来证明过高或过低，可以通过 Governance Game 进行调整。
9. 这一机制使活跃用户可以每月申领一次调整，从而将更多 XAH 投入流通，降低该代币因智能合约执行费用而被销毁至接近零的风险。

### 每月治理验证者奖励

10. Xahau Genesis 账户启用了一个 Hook，用于控制包括 XAH 发行在内的多项事务。该 Hook 需要一套治理安排，以便在面对不断变化的网络条件与现实世界条件时，可以被监控、修订和/或替换。
11. 众所周知，XRPL 主网验证者并不会因为为主网账本提供底层服务而获得任何形式的奖励。然而，运行基础设施是有成本的，而现实经验表明，缺乏激励会导致缺乏被积极维护的验证者。
12. Xahau Genesis Hook 需要知道，或者被告知，它在决定是否更新自身时必须信任哪些账户。因此，这些账户必须预先已知，而不是像重叠 UNL 那样自然涌现。
13. 针对这个问题的技术性解决方案，是一个带激励的 Genesis Hook Governance Game，用以奖励特定验证者来监督该 Hook。其工作方式如下：
   * Genesis Hook Governance Game 中共有 20 个“Seats”。
   * 每个 Seat 要么由一个单独的 Xahau 账户控制（Level 1），要么由一个 3 至 20 个账户组成的自管理委员会控制（Level 2）。
   * 对于每个已占用的 Seat，协议会铸造并分发当月全部已申领 Monthly Balance Adjustments 总量的 `1/20`。
   * 如果某个 Seat 由一个账户委员会持有，则该 Seat 的奖励按照该委员会自行决定的方式进行分配。
   * 如果当月没有用户申领 Balance Adjustment，则不会向任何验证者支付奖励。
   * 每个 Flag Ledger（256 本账本）都会生成一个名为 `UNLReport` 的链上记录。该报告包含一份验证者公钥列表，表示有 80% 的其他验证者认为这些验证者自上一个 Flag Ledger 以来充分参与了共识。Genesis Hook 只会向其公钥出现在 UNLReport 中的 Seat 分发验证者奖励。
14. 因此，Genesis Hook Governance Game 包含一套奖励机制，用于确保受信任账户因运行可信验证者而获得奖励，这些奖励与用户活跃度挂钩，并且能够将新的 XAH 投入流通，从而缓解因高额智能合约执行费用而引发的代币流动性紧缩风险。

## 5. Xahau Governance Game

1. 使用 XRP Ledger Consensus Protocol（此前称为 Ripple Protocol Consensus Algorithm）的链依赖验证者 UNL 之间强有力的重叠，来定义规范账本。如果某个验证者所信任的机器与其他人共同信任的机器重叠不足，那么它就有可能在不知情的情况下跟随一个无关的账本分叉。确定最佳 UNL 是一种社会性（链下）共识问题。
2. 为帮助形成这种社会共识，多个实体会发布 dUNL，即“所有人都应跟随的验证者列表”。Xahau 也会采用同样的方法，并预计 XRPL Foundation 及其他机构会发布此类列表。因此，和 XRPL 一样，Xahau 将是一条任何人都可以运行验证者的公开、无许可链；并且和 XRPL 一样，任何协议更新都将依赖超过 80% 的 UNL 验证者支持变更。
3. 尽管 dUNL 有助于确定哪些验证者负责规范账本，Xahau 还面临 XRPL 主网中不存在的第二项治理挑战：Genesis Hook 的治理。XRPL 主网在创建时一次性铸造了全部 XRP，而 Xahau 无法采用这种方式。其原生代币需要在预定义事件发生时铸造（例如 Burn2Mint、Account Adjustments、Validator Rewards），为此它使用了一个安装在 Genesis Account 上的 Hook。
4. 这意味着，如果 Genesis Hook 需要被调整或替换，Xahau 必须拥有一套治理机制。这就是所谓的 Governance Game：
   * **Purpose：** Governance Game 的存在，是为了确保 UNL 验证者保持活跃，并允许这些活跃成员维护控制 XAH 分发的 Genesis 账户 Hook。
   * **Seats：** 桌上共有 20 个 Seat。Seat 是一个 Xahau 账户，其投票会计入 Governance Game。
   * **L1 Table：** Governance Game 至少需要顶层表，这称为 Level 1 table（L1 Table）。按照安装在 Genesis 账户上的 Governance Game Hook 定义，该表只存在于 Genesis 账户上。L1 表上的 Seat 可以由某个账户占据，也可以为空，这些 Seat 被称为 L1。
   * **L2 Table：** 某个 L1 Seat 本身可以由一个安装了另一种不同 Governance Hook 的账户占据。该表上的 Seat 被称为 L2，该表则称为 L2 Table。它本质上是一个由 3 至 20 个账户组成的自管理委员会，共同控制该 L1 Seat 及其投票权。因此，根据 L1 与 L2 Seats 的构成混合程度，双层 Governance Game 中最多可有 400 个账户参与。
   * **填充 Seats：** L1 Seat 由现有已填充 L1 Seats 中 80% 的投票来决定填充或空出。对于 L2 Seats，共同持有该 L1 Seat 的账户委员会则可通过多数票，或依据其自行设定的其他逻辑与规则，邀请或移除新成员（最多 20 个，最少 3 个）。其他 Seats 无权控制某个 L2 Table 的成员。
   * **Voting：** 每个 L1 Seat 在 Governance Game 中拥有 1 票。对于 L2 Seats，其所在 L2 Table 的单张票如何决定，取决于该 L2 Table 自己设定的逻辑。默认方式为该 L2 Table 成员中超过 50% 投票通过。
   * **Hook Changes：** 所有对 Hook 的修改都通过 Governance Game 投票决定。只有当定义比例的 Seats 支持时，Hook 变更才会成功。常规投票门槛为 80%，但若要提高余额调整利率，则必须达到 100%。
   * **Rewards：** 为了激励 L1 Seats 运行可靠验证者，只要它们积极参与共识，就可以按月获得奖励。
   * **dUNL 的基础：** 参与 Genesis Hook Governance Game 的账户所运行的验证者，应当构成任何 dUNL 的基础，因为表现不可靠的 Seat 会面临被其他 Seats 投票移出的风险。
5. 不同类型的 Seat 使更多样化的账户得以参与，同时又不削弱安全性。Level 1 Seats 适合由社区可以信任的重要实体所控制的账户，而 Layer 2 Seats 则允许更广泛的参与者进入；但由于它们共同只计作 1 票，因此不会压倒社区对 Level 1 Seats 的信任。
6. 因此，Governance Game 允许最多 400 个账户参与，并在这些账户同时通过运行可靠验证者参与共识的前提下，对其给予奖励。

## 6. Xahau Launch Alliance

1. Xahau 由 5 个彼此独立、且在 XRPL 生态系统中拥有建设与支持记录的实体组成的联盟开发完成。这些实体如下：
   * XRPL Labs（Xaman），即 XUMM 钱包、Hooks 和 Xahaud 的软件开发方，承担了 Xahau 开发的大部分技术专长与成本。
   * GateHub Limited，一家跨国技术公司、加密货币交易所和加密服务提供商，也包括在 XRPL 上发行稳定币。
   * Titanium OU，一家 IT 咨询与基础设施公司，专长于提供安全托管，同时也是 XRPL 基础设施的重要提供者（以 Alloy Networks 名义运营）。
   * Evernode Labs Ltd，即 Evernode 智能合约项目的开发者，并负责将其部署到 Xahau。
   * Digital Governing OU，与一家在会计、审计与法律服务方面全球活跃的机构相关联的注册实体。

## 7. 以完全可用且去中心化的形式启动

1. Xahau 将在不进行融资的前提下，以完全可用且去中心化的状态启动。不会存在集中控制。
2. Xahau 启动时，其 20 个 Governance Game Seats 中将有 8 个被填满：
   * 其中 5 个为 Layer 1 Seats，分别由 Xahau Launch Alliance 的成员持有。
   * 其中 3 个为 Layer 2 Tables，其中一个将由交易所委员会持有，另一个将由 XRP 社区项目、开发者和长期支持者组成的多元委员会持有。
   * 其中 1 个将由 FYEO 持有，这是一家提供区块链开发代码审计与评审服务的公司，也包括对 Hooks 这类软件的审计与评审。

### 不进行融资

3. 不会进行 ICO，启动过程中也不会向外部募集资金。
4. Xahau Alliance 的各成员都各自独立承担了自身的开发投入。不存在公共资金池，也不存在为 Xahau 后续开发而进行的资源或资金集中。也不存在任何路线图、“Xahau Foundation”或其他中心化国库。任何进一步开发都完全由生态系统中的独立参与者自行决定。

### 不作承诺

5. Governance Game 的成员彼此独立。它们之间不存在关于如何投票、如何部署资源，或如何使用其资产（包括任何 XAH）的协议、安排或默契。推动验证者行为的，只有该游戏自身的逻辑：在其他成员眼中成为可信验证者，否则就会被投票移出该游戏。这正是 Xahau 所采用的 XRP Ledger Consensus Protocol（此前称为 Ripple Protocol Consensus Algorithm）（PoA）机制的固有特征。

### 不再需要额外功能

6. 尽管软件按其本性需要持续维护，但就 `xahaud` 作为 XRPL 智能合约侧链的用途而言，它不存在任何关键功能缺失，也不存在任何必须继续开发才能具备实用价值的核心功能。
