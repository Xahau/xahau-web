---
title: TES Codes
---
`tesSUCCESS` 与 `tesPARTIAL` 是唯一表示交易成功的代码。这并不总意味着交易达成了你期待的业务效果。（例如，一个 [OfferCancel][] 即使实际上没有任何可取消的 offer，也可能“成功”。）

`tesPARTIAL` 表示一笔交易部分成功。它专门用于某些操作超出一次处理上限的场景，例如删除 hook state 记录时超过一定阈值（如 512 或 1024 条）。在这种情况下，交易会先删除达到上限的部分，然后要求你使用新的 sequence number 再提交后续交易，以继续删除剩余记录，直到全部完成。这样可以把大规模操作拆分成可管理的小部分。

| 代码 | 说明 |
| --- | --- |
| `tesSUCCESS` | 交易已被应用并转发到其他服务器。如果它出现在已验证账本中，则其成功状态就是最终结果。 |
| `tesPARTIAL` | 交易部分成功，并且需要额外交易才能完成较大规模操作，例如删除超过固定上限的记录。 |
