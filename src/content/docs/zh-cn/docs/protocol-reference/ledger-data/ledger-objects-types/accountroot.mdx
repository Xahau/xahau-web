`AccountRoot` ledger entry 类型描述单个账户、其设置以及 XAH 余额。

### 示例 JSON

```json
{
    "Account": "rf1BiGeXwwQoi8Z2ueFYTEXSwuJYfV2Jpn",
    "AccountTxnID": "0D5FB50FA65C9FE1538FD7E398FFFE9D1908DFA4576D8D7A020040686F93C77D",
    "AccountIndex": "59",
    "Balance": "148446663",
    "Domain": "6D64756F31332E636F6D",
    "HookStateScale": 10,
    "EmailHash": "98B4375E1D753E5B91627516F6D70977",
    "Flags": 8388608,
    "LedgerEntryType": "AccountRoot",
    "MessageKey": "0000000000000000000000070000000300",
    "OwnerCount": 3,
    "PreviousTxnID": "0D5FB50FA65C9FE1538FD7E398FFFE9D1908DFA4576D8D7A020040686F93C77D",
    "PreviousTxnLgrSeq": 14091160,
    "Sequence": 336,
    "TransferRate": 1004999999,
    "index": "13F1A95D7AAB7108D5CE7EEAF504B2894B8C674E6D68499076441C4837282BF8"
}
```

### 字段

`AccountRoot` 对象包含以下字段：

| 字段 | JSON 类型 | \[Internal Type]\[] | 必填？ | 说明 |
| ---- | --------- | ------------------- | ------ | ---- |
| `Account` | String | AccountID | Yes | 该账户的标识性（classic）地址。 |
| `Sequence` | Number | UInt32 | Yes | 该账户下一笔有效交易的 sequence number。 |
| `Balance` | String | Amount | No | 账户当前的 [XAH 余额（以 drops 表示）][drops of XAH]，以字符串形式表示。 |
| `OwnerCount` | Number | UInt32 | Yes | 该账户在账本中拥有的对象数量，这会影响其 owner reserve。 |
| `PreviousTxnID` | String | Hash256 | Yes | 最近一次修改此对象的交易的标识性哈希。 |
| `PreviousTxnLgrSeq` | Number | UInt32 | Yes | 包含最近一次修改此对象的交易所在的 \[账本索引]\[Ledger Index]。 |
| `AccountTxnID` | String | Hash256 | No | 最近一次由该账户发送的交易的标识性哈希。必须启用此字段后才能使用交易中的 `AccountTxnID` 字段。要启用它，请发送一个开启 `asfAccountTxnID` 标志的 AccountSet 交易。 |
| `RegularKey` | String | AccountID | No | 可用于替代 master key 为该账户签名交易的密钥对地址。可通过 \[SetRegularKey transaction]\[] 修改此值。 |
| `EmailHash` | String | Hash128 | No | 一个邮箱地址的 md5 哈希。客户端可以用它通过 [Gravatar](https://en.gravatar.com/) 之类的服务查找头像。 |
| `WalletLocator` | String | Hash256 | No | 用户可自行设置的任意 256 位值。 |
| `WalletSize` | Number | UInt32 | No | 未使用。（代码支持该字段，但没有办法设置它。） |
| `MessageKey` | String | Blob | No | 可用于向该账户发送加密消息的公钥。在 JSON 中以十六进制表示。长度必须恰好为 33 字节，首字节表示密钥类型：`0x02` 或 `0x03` 用于 secp256k1，`0xED` 用于 Ed25519。 |
| `TransferRate` | Number | UInt32 | No | 向彼此发送由该账户发行的货币时，对其他用户收取的 transfer fee。 |
| `Domain` | String | Blob | No | 与该账户关联的域名。在 JSON 中，这是域名 ASCII 表示的十六进制值。[长度不能超过 256 字节。](https://github.com/Xahau/xahaud/blob/55dc7a252e08a0b02cd5aa39e9b4777af3eafe77/src/ripple/app/tx/impl/SetAccount.h#L34) |
| `TickSize` | Number | UInt8 | No | 对涉及该地址发行货币的 Offers，其汇率应保留多少位有效数字。有效值范围为 `3` 到 `15`（含边界）。_(Added by the \[TickSize amendment]\[].)_ |
| `TicketCount` | Number | UInt32 | No | 该账户在账本中拥有的 Tickets 数量。它会自动更新，以确保账户始终不超过同时持有 250 个 Tickets 的硬上限。当账户没有任何 Ticket 时，该字段会被省略。_(Added by the \[TicketBatch amendment]\[].)_ |
| `NFTokenMinter` | String | AccountID | No | 可代表该账户铸造 non-fungible tokens 的另一个账户。_(Added by the \[NonFungibleTokensV1_1 amendment]\[])_ |
| `MintedNFTokens` | Number | UInt32 | No | 由该账户以及代表该账户总共铸造了多少 non-fungible tokens。_(Added by the \[NonFungibleTokensV1_1 amendment]\[])_ |
| `BurnedNFTokens` | Number | UInt32 | No | 该账户发行的 non-fungible tokens 中，已被销毁的总数量。这个数值始终小于或等于 `MintedNFTokens`。 |
| `HookStateCount` | Number | UInt32 | No | Hook 上存在的 state 数量。 |
| `HookNamespaces` | Array | Vector256 | No | 一个唯一值，用于防止安装在同一 XAHL 账户上的多个 Hooks 之间发生 state 冲突。 |
| `RewardLgrFirst` | Number | UInt32 | No | 最近一笔 ClaimReward 交易所在的账本号。 |
| `RewardLgrLast` | Number | UInt32 | No | 最近一笔进出该账户的交易对应的账本 sequence number。 |
| `RewardTime` | Number | UInt32 | No | 最近一次领取奖励时的账本时间。 |
| `RewardAccumulator` | String | UInt64 | No | 自上一次 ClaimReward 交易以来，该账户余额-时间图像下面积。 |
| `FirstNFTokenSequence` | Number | UInt32 | No | 该账户首次铸造 non-fungible token 时的 \[Sequence Number]\[]。_(Added by the \[fixNFTokenRemint amendment]\[] :not_enabled:)_ |
| `ImportSequence` | Number | UInt32 | No | XAHL Ledger 上 importing account 当前的 sequence number。 |
| `GovernanceFlags` | String | Hash256 | No |  |
| `GovernanceMarks` | String | Hash256 | No |  |
| `AccountIndex` | String | UInt64 | No | 该账户的创建索引号。_(Added by the \[xahauGenesis amendment]\[] :enabled:)_ |
| `TouchCount` | String | UInt64 | No | 每次该 ledger entry 被 “touched” 时递增的计数器。 |
| `HookStateScale` | Number | UInt16 | No | 用于判定 Hook state 条目何时变为 stale 的 scale factor。有效值为 `1` 到 `16`（含边界）。_(Added by the [ExtendedHookState amendment][].)_ |

### AccountRoot Flags

账户有若干选项可以启用或禁用。这些选项可以通过 \[AccountSet transaction]\[] 修改。在账本中，flags 使用二进制值表示，并通过按位 OR 组合。账本中的 flags 位值，与交易里用于启用或禁用这些 flags 的值并不相同。账本 flags 的名称以 **`lsf`** 开头。

### AccountRoot ID 格式

`AccountRoot` 对象的 ID 是对以下值按顺序拼接后计算得到的 \[SHA-512Half]\[]：

* Account space key（`0x0061`）
* 该账户的 AccountID
