---
title: Check
---
[\[Source\]](https://github.com/ripple/rippled/blob/master/src/ripple/protocol/impl/LedgerFormats.cpp#L157-L170)

_(Added by the \[Checks amendment]\[].)_

`Check` 对象描述一种类似纸质个人支票的 check，它可由目标接收方兑现，从发送方获得资金。（该潜在支付已经由发送方预先批准，但在真正兑现前不会有资金移动。与 Escrow 不同，Check 对应的资金并不会被预先锁定，因此兑现 Check 可能因资金不足而失败。）

### 示例 JSON

```json
{
  "Account": "rUn84CUYbNjRoTQ6mSW7BVJPSVJNLb1QLo",
  "Destination": "rfkE1aSy9G8Upk4JssnwBxhEv5p4mn2KTy",
  "DestinationNode": "0000000000000000",
  "DestinationTag": 1,
  "Expiration": 570113521,
  "Flags": 0,
  "InvoiceID": "46060241FABCF692D4D934BA2A6C4427CD4279083E38C77CBE642243E43BE291",
  "LedgerEntryType": "Check",
  "OwnerNode": "0000000000000000",
  "PreviousTxnID": "5463C6E08862A1FAE5EDAC12D70ADB16546A1F674930521295BC082494B62924",
  "PreviousTxnLgrSeq": 6,
  "SendMax": "100000000",
  "Sequence": 2,
  "index": "49647F0D748DC3FE26BDACBC57F251AADEFFF391403EC9BF87C97F67E9977FB0"
}
```

### 字段

`Check` 对象包含以下字段：

| 字段 | JSON 类型 | \[Internal Type]\[] | 必填？ | 说明 |
| ---- | --------- | ------------------- | ------ | ---- |
| `Account` | String | Account | Yes | Check 的发送方。兑现该 Check 时，会从该地址余额中扣款。 |
| `Destination` | String | Account | Yes | Check 的目标接收方。只有这个地址可以通过 \[CheckCash transaction]\[] 兑现该 Check。 |
| `DestinationNode` | String | UInt64 | No | 一个提示值，用于指示如果目标地址的 owner directory 由多页组成，则链接到此对象的是哪一页。 |
| `DestinationTag` | Number | UInt32 | No | 一个任意标签，用于进一步指定该 Check 的目标，例如目标地址上的托管子账户。 |
| `Expiration` | Number | UInt32 | No | 表示该 Check 在此时间之后被视为已过期。详见 \[Specifying Time]\[]。 |
| `Flags` | Number | UInt32 | Yes | 为该对象启用的一组布尔位标志。当前协议未为 `Check` 对象定义任何 flags，因此该值始终为 `0`。 |
| `InvoiceID` | String | Hash256 | No | 由发送方提供的任意 256 位哈希，用作该 Check 的具体理由或标识符。 |
| `LedgerEntryType` | String | UInt16 | Yes | 值 `0x0043`，映射为字符串 `Check`，表示该对象是一个 Check 对象。 |
| `OwnerNode` | String | UInt64 | Yes | 一个提示值，用于指示如果发送方的 owner directory 由多页组成，则链接到此对象的是哪一页。**注意：** 该对象本身不直接保存包含它的 owner directory 的链接，因为该值可从 `Account` 推导。 |
| `PreviousTxnID` | String | Hash256 | Yes | 最近一次修改该对象的交易的标识性哈希。 |
| `PreviousTxnLgrSeq` | Number | UInt32 | Yes | 包含最近一次修改该对象交易的 \[账本索引]\[Ledger Index]。 |
| `SendMax` | String or Object | Amount | Yes | 该 Check 最多可从发送方扣除的货币金额。如果该 Check 成功兑现，目标地址将以相同货币最多收到该金额。 |
| `Sequence` | Number | UInt32 | Yes | 创建该 Check 的 \[CheckCreate transaction]\[] 的 sequence number。 |
| `SourceTag` | Number | UInt32 | No | 一个任意标签，用于进一步指定该 Check 的来源，例如发送方地址上的托管子账户。 |

### Check ID 格式

`Check` 对象的 ID 是对以下值按顺序拼接后计算得到的 \[SHA-512Half]\[]：

* Check space key（`0x0043`）
* 创建该 `Check` 对象的 \[CheckCreate transaction]\[] 的发送方 AccountID
* 创建该 `Check` 对象的 \[CheckCreate transaction]\[] 的 `Sequence` number。如果 CheckCreate 交易使用了 Ticket，则应改用 `TicketSequence` 值。
