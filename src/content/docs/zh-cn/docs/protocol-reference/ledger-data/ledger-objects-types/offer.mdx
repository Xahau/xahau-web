`Offer` ledger entry 用于描述 Xahau 去中心化交易所中的一笔 Offer。（在金融领域，更常见的叫法是 _order_。）只有当一笔 \[OfferCreate transaction]\[] 无法通过消耗账本中已有的其他 Offers 而被立即完全执行时，它才会在账本中创建一个 `Offer` 对象。

一笔 Offer 在仍保留于账本中的同时，也可能因为网络中的其他活动而变成 unfunded。网络在处理交易时，会自动剪除这些交易遇到的任何 unfunded Offers。（否则，这些 unfunded Offers 会继续留在账本中，因为 _只有_ 交易才能改变账本状态。）

    "BookNode": "0000000000000000",

    "OwnerNode": "0000000000000000",

### 字段

`Offer` 对象包含以下字段：

| 字段名 | JSON 类型 | \[Internal Type]\[] | 必填？ | 说明 |
| ------ | --------- | ------------------- | ------ | ---- |
| `Account` | String | AccountID | Yes | 拥有这笔 Offer 的账户地址。 |
| `BookDirectory` | String | Hash256 | Yes | 指向这笔 Offer 的 Offer Directory 的 ID。 |
| `BookNode` | String | UInt64 | Yes | 一个提示值，用于指出如果该目录由多页组成，那么目录中的哪一页链接到了此对象。 |
| `Expiration` | Number | UInt32 | No | 表示在该时间之后，这笔 Offer 会被视为 unfunded。详见 \[Specifying Time]\[]。 |
| `Flags` | Number | UInt32 | Yes | 为此 offer 启用的一组布尔位标志。 |
| `LedgerEntryType` | String | UInt16 | Yes | 值 `0x006F`，映射为字符串 `Offer`，表示该对象描述的是一笔 Offer。 |
| `OwnerNode` | String | UInt64 | Yes | 一个提示值，用于指出如果 owner directory 由多页组成，那么其中哪一页链接到了此对象。**注意：** 该 offer 本身并不直接存储它所在 owner directory 的链接，因为这个值可以从 `Account` 推导出来。 |
| `PreviousTxnID` | String | Hash256 | Yes | 最近一次修改该对象的交易的标识性哈希。 |
| `PreviousTxnLgrSeq` | Number | UInt32 | Yes | 包含最近一次修改该对象交易的 \[账本索引]\[Ledger Index]。 |
| `Sequence` | Number | UInt32 | Yes | 创建此 `Offer` 对象的 \[OfferCreate]\[] 交易的 `Sequence` 值。它与 `Account` 一起用于唯一标识这笔 Offer。 |
| `TakerPays` | String or Object | Amount | Yes | Offer 创建者希望得到的、剩余未成交的金额及货币类型。 |
| `TakerGets` | String or Object | Amount | Yes | Offer 创建者愿意提供的、剩余未成交的金额及货币类型。 |

### Offer Flags

在通过 \[OfferCreate transaction]\[] 创建 offer 对象时，可以启用或禁用若干选项。在账本中，flags 使用二进制值表示，并通过按位 OR 组合。账本里的位值与交易中用于启用或禁用这些 flags 的值并不相同。账本 flags 的名称以 **`lsf`** 开头。

`Offer` 对象可具有以下 flag 值：

### Offer ID 格式

`Offer` 对象的 ID 是对以下值按顺序拼接后计算得到的 \[SHA-512Half]\[]：

* Offer space key（`0x006F`）
* 发出 Offer 的账户的 AccountID
* 创建该 Offer 的 \[OfferCreate transaction]\[] 的 Sequence number

  如果该 OfferCreate 交易使用了 Ticket，则应改用 `TicketSequence` 值。
