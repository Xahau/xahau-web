---
title: 货币格式
---
Xahau 有两类数字资产：XAH 与 tokens。两者都具备高精度，但格式不同。

### 对比

下表总结了 Xahau 中 XAH 与 tokens 的一些差异：

| XAH | Tokens |
| --- | ------ |
| 没有 issuer。 | 始终由某个 Xahau 账户发行。 |
| 以字符串形式指定。 | 以对象形式指定。 |
| 在账户中追踪。 | 在 trust lines 中追踪。 |
| 永远不能被创建；只能被销毁。 | 可以自由发行或赎回。 |
| 最小值： `0`。（不能为负数。） | 最小值： `-9999999999999999e80`。最小非零绝对值： `1000000000000000e-96`。 |
| 最大值： `100000000000`（10^11）XAH。也就是 `100000000000000000`（10^17）drops。 | 最大值： `9999999999999999e80`。 |
| 精确到最近的一个 “drop”（0.000001 XAH） | 具有 15 位十进制精度。 |
| 不能被冻结。 | issuer 可以冻结余额。 |
| 无转账费用；XAH 对 XAH 的支付始终是直接的。 | 可以通过间接路径转移，并且每个 issuer 可收取一定比例的 transfer fee。 |
| 可以用于 Payment Channels 和 Escrow。 | 也可以用于 Payment Channels 或 Escrow。 |

更多信息请参见 What is XAH? 与 Tokens。

### 指定货币金额

请根据你要表示的货币类型，使用对应格式：

* XAH Amounts
* Token Amounts

#### XAH Amounts

若要表示一笔 XAH 数量，请使用一个 String Number，表示 XAH 的 _drops_ 数量，其中每一个 drop 等于 0.000001 XAH。例如，若要表示 13.1 XAH：

```
"13100000"
```

**不要把 XAH 指定成对象。**

XAH 金额不能为负数。

#### Token Amounts

若要表示一笔（可替代）token 数量，请使用一个 Amount 对象。这是一个包含三个字段的 JSON 对象：

| `Field` | 类型 | 说明 |
| ------- | ---- | ---- |
| `currency` | String - Currency Code | 该 token 的任意货币代码。不能是 XAH。 |
| `value` | String Number | Token 数量的带引号十进制表示。它可以包含科学计数法，例如 `1.23e11` 表示 123,000,000,000。`e` 与 `E` 都可使用。在显示余额时它可以是负数，但在指定发送数量等其他场景中不允许为负值。 |
| `issuer` | String | 通常指发行该 token 的账户。在某些特殊场景中，它也可能改为指持有该 token 的账户（例如在 Clawback 交易中）。 |

**注意：** 这些字段名称区分大小写。

例如，要表示由账户 `r9cZA1mLK5R5Am25ArfXFmqgNwjZgnfk59` 发行的 153.75 美元，你应这样写：

```json
{
    "currency": "USD",
    "value": "153.75",
    "issuer": "r9cZA1mLK5R5Am25ArfXFmqgNwjZgnfk59"
}
```

#### 不带金额的指定方式

有些场景下，你需要定义一种资产（可能是 XAH，也可能是 token），但不指定具体金额，例如在去中心化交易所中定义订单簿时。

若要表示一种不带金额的 token，可将其写成 currency object，但省略 `value` 字段。例如：

```json
{
  "currency": "TST",
  "issuer": "rP9jPyP5kyvFRb6ZiRghAGw5u8SGAmU4bd"
}
```

若要表示不带金额的 XAH，则应使用一个**只包含** `currency` 字段的 JSON 对象。绝不要为 XAH 加上 `issuer` 字段。例如：

```json
{
  "currency": "XAH"
}
```

### 字符串数字

### XAH 精度

XAH 的精度与一个 64 位无符号整数相同，其中每个单位等于 0.000001 XAH。它使用整数计算，因此任何小于一个完整 drop 的数量都会向下取整。

### Token 精度

Tokens 可以表示非常广泛的资产，包括那些通常以极小或极大单位计量的资产。这种格式使用有效数字与 10 的幂指数，类似于科学计数法。该格式支持在规定范围内的正负有效数字与指数。与典型的非整数浮点数表示不同，这种格式在所有计算中都使用整数运算，因此始终保持 15 位十进制精度。乘法和除法还会做额外调整，以补偿最低有效位上的过度舍入。

当 token amount 在 Xahau 的点对点网络中发送时，服务器会将该数量序列化为一个 64 位二进制值。

**提示：** 对于那些完全不应可分割的 token，请参见 Non-Fungible Tokens（NFTs）。

### 货币代码

#### 标准货币代码

货币代码的标准格式是一个三字符字符串，例如 `USD`。它原本是为 [ISO 4217 Currency Codes](https://www.xe.com/iso4217.php) 设计的。适用规则如下：

* 货币代码长度必须**严格为 3 个 ASCII 字符**。允许使用的字符包括：所有大小写字母、数字，以及符号 `?`、`!`、`@`、`#`、`$`、`%`、`^`、`&`、`*`、`<`、`>`、`(`、`)`、`{`、`}`、`[`、`]` 和 `|`。
* 货币代码区分大小写。
* 全大写的货币代码 `XAH` 不允许使用。真正的 XAH 在 Xahau 协议中通常不会用货币代码表示。

在协议层面，这种格式会被序列化为一个以 `0x00` 开头的 160 位二进制值。

#### 非标准货币代码

你也可以使用一个 160 位（40 个字符）的十六进制字符串作为货币代码，例如 `015841551A748AD2C1F76FF6ECB0CCCD00000000`。为了防止它被识别为“标准”货币代码，其前 8 位**不能**是 `0x00`。

**已弃用：** 早期某些版本的 [ripple-lib](https://github.com/XRPLF/xrpl.js) 曾支持一种 “interest-bearing” 或 “demurraging” 货币代码类型。这类代码的前 8 位是 `0x01`。Demurraging / interest-bearing currencies 已不再受支持，但你仍可能在账本数据中看到它们。更多信息请参见 Demurrage。
