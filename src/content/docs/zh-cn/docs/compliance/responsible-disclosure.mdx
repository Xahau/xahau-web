### 负责任披露政策

在 Xahau，我们认为系统安全至关重要。

尽管我们在产品开发与维护期间始终高度重视系统安全，但仍然有可能会有人发现某些我们需要改进、更新、变更或修复的问题。

如果你发现了我们某个系统中的薄弱点，我们非常感谢你尽快通知我们，以便我们能够立即采取措施来保护客户及其数据。

### 如何报告

如果你认为自己在我们的某个系统中发现了安全问题，请尽快在 Github 上发布一段**高层级描述**以及你的联系方式（以便我们联系你）：
[**https://github.com/Xahau/xahaud/issues**](https://github.com/Xahau/xahaud/issues)

### 规则

这份负责任披露政策并不意味着我们公开邀请任何人主动扫描我们的网络和应用以寻找漏洞。我们的持续监控很可能会发现此类扫描，并对此进行调查。

#### 我们希望你：

* 在问题解决前，不要向其他人分享有关该安全问题的信息，并立即删除你获取到的任何机密数据
* 不要进一步滥用这个问题，例如下载超过演示泄露所必需的数据，或查看、删除、修改第三方数据
* 提供足够详细的信息，以便我们尽快复现、验证并解决问题。请包含你的测试数据、时间戳，以及涉及系统的 URL
* 留下你的联系方式（电子邮件地址和/或电话号码），以便我们在修复进展方面与你联系。我们也接受匿名报告
* 不要使用针对物理安全、社会工程学、分布式拒绝服务、垃圾信息或第三方应用的攻击

### 负责任披露流程

#### 当你报告安全问题后，我们会按以下方式处理：

* 在你提交报告后的 4 个工作日内，我们会确认收到报告
* 在发送确认后的 4 个工作日内，我们会提供对该安全问题的评估以及预计解决日期
* 如果你遵守上述条件，我们不会就该报告对你采取法律行动
* 我们会对你的报告保密，除非为履行法律义务所必需，否则未经你许可不会将你的信息分享给第三方

#### 本负责任披露方案不适用于：

* 投诉
* 网站不可用报告
* 钓鱼报告
* 欺诈报告

对于这些投诉或报告，请在 Github 上发布一段**高层级问题描述**以及你的联系方式（以便我们联系你）：
[**https://github.com/Xahau/xahaud/issues**](https://github.com/Xahau/xahaud/issues)

### Bug bounty program

Xahau 鼓励报告安全问题或漏洞。对于那些以保密方式披露、且此前我们尚未知晓、可能危及用户数据机密性或完整性的设计或实现问题，我们可能会给予适当奖励。是否符合资格以及奖励金额由我们决定。

### 排除项

#### 以下类型的安全问题不在范围内

* (D)DOS 攻击
* 不包含敏感数据的错误消息或错误页面
* Github 仓库中已公开的测试数据和示例数据
* 通过漏洞扫描识别出的常见问题，例如浏览器头部警告或 DNS 配置
* 针对我们公开使用的软件所生成的漏洞扫描报告
* 与过时操作系统、浏览器或插件相关的安全问题
* 对我们已经收到通知的问题再次提交报告

请注意：缺乏任何证明（例如截图或其他数据）、缺乏详细信息，或缺乏复现说明的报告仍会被调查，但将不具备获得奖励的资格。

本政策基于荷兰国家网络安全中心（National Cyber Security Centre）的 Responsible Disclosure Guidelines，以及 [Floor Terra 的示例](https://responsibledisclosure.nl/)。
