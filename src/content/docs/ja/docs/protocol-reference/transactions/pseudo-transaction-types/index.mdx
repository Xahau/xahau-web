---
title: 擬似トランザクションタイプ
---
擬似トランザクションは、ユーザーが送信することも、ネットワークを通じて伝播することもありません。代わりに、サーバーは特定のプロトコルルールに従って、提案中のレジャーに擬似トランザクションを直接挿入する場合があります。十分な数のサーバーがまったく同じ擬似トランザクションを提案すれば、コンセンサス処理で承認され、その擬似トランザクションはそのレジャーのトランザクションデータに含まれます。

### 共通フィールドの特殊値

通常のトランザクションで必要な \[common fields]\[]の一部は、擬似トランザクションでは意味を持ちません。擬似トランザクションでは、これらの共通フィールドに次の特殊値を使います。

| フィールド      | JSON型   | \[内部型]\[]        | 値                  |
| --------------- | --------- | ------------------- | ------------------- |
| `Account`       | String    | AccountID           | ACCOUNT_ZERO       |
| `Fee`           | String    | Amount              | `0`                 |
| `Sequence`      | Number    | UInt32              | `0`                 |
| `SigningPubKey` | String    | Blob                | `""` (Empty string) |
| `TxnSignature`  | String    | Blob                | `""` (Empty string) |

擬似トランザクションでは、次の共通フィールドは通常どおり使います。

* `TransactionType`
* `Flags`

| フィールド       | JSON型   | \[内部型]\[]        | 説明                                                                                                               |
| ----------------- | --------- | ------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `TransactionType` | String    | UInt16              | _(必須)_トランザクションの種類。                                                                                     |
| `Flags`           | Number    | UInt32              | _(任意)_このトランザクションのビットフラグの集合。個々のフラグの意味は、トランザクションタイプによって異なります。 |
