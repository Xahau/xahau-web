---
title: ClaimReward
description: >-
  ClaimRewardトランザクションを使うと、アカウントは蓄積した報酬を請求できます。報酬はアカウント所有者または指定したissuerが請求できます。アカウントは報酬の受け取りを拒否することもできます。
---
\[[ソース](https://github.com/Xahau/xahaud/blob/dev/src/ripple/app/tx/impl/ClaimReward.cpp)]

_(\[BalanceRewards amendment]\[]により追加。)_

### オプトイン + 請求

```json
{
    "TransactionType": "ClaimReward",
    "Account": "rWYkbWkCeg8dP6rXALnjgZSjjLyih5NXm",
    "Issuer": "rHb9CJAWyB4rj91VRWn96DkukG4bwdtyTh"
}
```

### オプトアウト

```json
{
    "TransactionType": "ClaimReward",
    "Account": "rWYkbWkCeg8dP6rXALnjgZSjjLyih5NXm",
    "Flags": 1
}
```

### フィールド

| フィールド     | JSON型 | \[Internal Type]\[] | 説明                                              |
| --------- | --------- | ------------------- | -------------------------------------------------------- |
| `Account` | String    | AccountID           | 報酬を請求しているアカウントのアドレス。               |
| `Flags`   | Number    | UInt32              | _(任意)_ フラグ1を設定すると報酬をオプトアウトできます。 |
| `Issuer`  | String    | AccountID           | _(任意)_ genesis account。                           |

### ClaimRewardフラグ

ClaimRewardタイプのトランザクションは、`Flags` フィールドで次の追加値をサポートします。

| フラグ名  | Hex値    | 10進値 | 説明                                                                                                                                                                                                                      |
| ---------- | ------------ | ------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `tfOptOut` | `0x00000001` | 1             | ClaimRewardコードの `isOptOut` フラグは、トランザクションの `sfFlags` フィールドが1に設定されている場合に、レジャー内のアカウントオブジェクトから報酬関連フィールドを削除して、そのアカウントを報酬から除外するために使われます。 |

### 特別なトランザクションコスト

ClaimRewardトランザクションには標準のトランザクションコストが適用されます。これは、すべてのトランザクションに必要な最小トランザクションコストです。

### エラーケース

すべてのトランザクションで発生しうるエラーに加えて、ClaimRewardトランザクションでは次のトランザクション結果コードが返されることがあります。

| エラーコード        | 説明                                                                                             |
| ----------------- | ------------------------------------------------------------------------------------------------------- |
| `temDISABLED`     | 機能が有効でない場合に発生します。                                                                      |
| `temINVALID_FLAG` | フラグが1以外の値に設定されている場合に発生します。                                                   |
| `temMALFORMED`    | issuerが送信元アカウントと同じ場合、またはフラグとissuerが正しく設定されていない場合に発生します。 |
| `tecNO_ISSUER`    | issuerが存在しない場合に発生します。                                                                    |
| `terNO_ACCOUNT`   | 送信アカウントが存在しない場合に発生します。                                                             |
