### 例

```json
{
    "TransactionType": "AccountSet",
    "Account" : "rf1BiGeXwwQoi8Z2ueFYTEXSwuJYfV2Jpn",
    "Fee": "12",
    "Sequence": 5,
    "Domain": "6578616D706C652E636F6D",
    "SetFlag": 5,
    "MessageKey": "03AB40A0490F9B7ED8DF29D246BF2D6269820A0EE7742ACDD457BEA7C7D0931EDB",
    "HookStateScale": 10
}
```

| --------------- | ------ | ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| `Domain`        | String | Blob         | _(任意)_このアカウントを所有するドメイン。小文字ドメインのASCIIを表すhex文字列として指定します。[長さは256バイトを超えられません。](https://github.com/Xahau/xahaud/blob/55dc7a252e08a0b02cd5aa39e9b4777af3eafe77/src/ripple/app/tx/impl/SetAccount.h#L34) |
| `EmailHash`     | String | Hash128      | _(任意)_任意の128ビット値。慣例として、クライアントはこれを [Gravatar](http://en.gravatar.com/site/implement/hash/)画像表示に使うメールアドレスのmd5ハッシュとして扱います。                                                                                                     |
| `MessageKey`    | String | Blob         | _(任意)_このアカウントへ暗号化メッセージを送るためのpublic key。鍵を設定するには、正確に33バイトで、最初のバイトが鍵タイプを示す必要があります。secp256k1鍵では `0x02` または `0x03`、Ed25519鍵では `0xED` です。鍵を削除するには空の値を使います。      |
| `NFTokenMinter` | String | Blob         | _(任意)_あなたの代わりにNFTokensをミントできる別のアカウント。_(\[NonFungibleTokensV1_1 amendment]\[]により追加。)_                                                                                                                                                              |

| `TransferRate`  | Number | UInt32       | _(任意)_ユーザーがこのアカウントのトークンを転送するときに課す手数料。単位の10億分率で表します。手数料なしを意味する特別な値 `0` を除き、`2000000000` を超えたり `1000000000` 未満にしたりすることはできません。                                                   |

| `WalletLocator` | String | Hash256      | _(任意)_任意の256ビット値。指定すると、その値はアカウントの一部として保存されますが、固有の意味や要件はありません。                                                                                                                                                                |

### Domain

`Domain` フィールドは、ドメインを小文字ASCIIにしたhex文字列として表します。たとえば、ドメイン_example.com_は `"6578616D706C652E636F6D"` と表されます。

アカウントから `Domain` フィールドを削除するには、Domainに空文字列を設定したAccountSetを送信します。

アカウントの `Domain` フィールドには任意のドメインを設定できます。アカウントとドメインが同じ個人または事業者に属することを証明するには、「双方向リンク」が必要です。

* 自分が所有するアカウントには、自分が所有するドメインを `Domain` フィールドに設定します。
* そのドメインで、所有するアカウントと、任意でXahauの利用方法に関する追加情報を列挙した [xahau.toml](/ja/docs/infrastructure/identity)ファイルを公開します。

### AccountSetフラグ

アカウントには、有効化または無効化できるオプションが複数あります。アカウントオプションは、状況に応じて異なる種類のフラグで表されます。

* `AccountSet` トランザクションタイプには、`SetFlag` パラメーターで渡すとオプションを有効化し、`ClearFlag` パラメーターで渡すとオプションを無効化する、複数の「AccountSet Flags」(**`asf`** 接頭辞)があります。新しいオプションはこの形式のフラグのみを持ちます。1つのトランザクションで有効化できる `asf` フラグは最大1つ、無効化できる `asf` フラグも最大1つです。
* `AccountSet` トランザクションタイプには、`Flags` パラメーターで渡すと特定のアカウントオプションを有効化または無効化できる、複数のトランザクションフラグ (**`tf`** 接頭辞)があります。複数の `tf` フラグを使うことで、1つのトランザクションで設定の組み合わせを有効化/無効化できますが、すべての設定に `tf` フラグがあるわけではありません。
* `AccountRoot` レジャーオブジェクトタイプには、特定のレジャー内で特定のアカウントオプションの状態を表す、複数のledger-state-flags (**`lsf`** 接頭辞)があります。これらの設定は、トランザクションで変更されるまで適用されます。

Account Flagsを有効化または無効化するには、AccountSetトランザクションの `SetFlag` と `ClearFlag` パラメーターを使います。AccountSetフラグの名前は **`asf`** で始まります。

すべてのフラグはデフォルトで無効です。

利用可能なAccountSetフラグは次のとおりです。

#### 着信トランザクションのブロック

目的が不明確な着信トランザクションは、顧客のミスを認識し、そのミスに応じてアカウントへ返金したり残高を調整したりする必要がある金融機関にとって不便になる場合があります。`asfRequireDest` と `asfDisallowXRP` フラグは、資金が送られた理由が不明確な形でユーザーが誤って資金を送ってしまうことを防ぐ目的で用意されています。

たとえば、destination tagは通常、金融機関がpaymentを受け取る際に、どのホスト型残高へ入金すべきかを識別するために使われます。destination tagが省略されていると、どのアカウントへ入金すべきかが不明確になり、返金などが必要になる場合があります。`asfRequireDest` タグを使うことで、すべての着信paymentにdestination tagを必須にし、他者が誤って曖昧なpaymentを送ってしまうことを防ぎやすくできます。

非XAH通貨の不要な着信paymentは、その通貨のtrust linesを作成しないことで防げます。XAHはtrustを必要としないため、`asfDisallowXRP` フラグはユーザーがXAHをアカウントへ送ることを抑止するために使われます。ただし、このフラグはXahauプロトコルでは強制されません。XAHが尽きた場合にアカウントが使用不能になる可能性があるためです。代わりに、クライアントアプリケーションは `asfDisallowXRP` フラグが有効なアカウントへのXAH paymentを禁止または抑止すべきです。

_すべての_ 着信paymentをブロックしたい場合は、Deposit Authorizationを有効化できます。これにより、アカウントがリザーブ要件を下回っている場合を除き、XAHであっても、どのトランザクションもあなたへ資金を送れなくなります。

\[DisallowIncoming amendment]\[] :not_enabled:が有効な場合、すべての着信Checks、NFTokenOffers、Payment Channels、trust linesをブロックするオプションもあります。一般に、これらのオブジェクトの受信側になること自体は無害ですが、アカウント削除を妨げたり、自分が作成したオブジェクト一覧に予期しないオブジェクトが混ざって混乱を招いたりする場合があります。着信オブジェクトをブロックするには、次のアカウントフラグを1つ以上使います。

* `asfDisallowIncomingCheck` - Checkオブジェクト用
* `asfDisallowIncomingNFTOffer` - NFTokenOfferオブジェクト用
* `asfDisallowIncomingPayChan` - PayChannelオブジェクト用
* `asfDisallowIncomingTrustline` - RippleState (trust line)オブジェクト用

トランザクションがこれらのレジャーエントリのいずれかを作成しようとしたとき、宛先アカウントで対応するフラグが有効になっている場合、そのトランザクションは結果コード `tecNO_PERMISSION` で失敗します。Deposit Authorizationとは異なり、これらの設定は一般的なpaymentの受信を妨げません。また、この設定を有効にしても、これらの種類のオブジェクトを自分で作成することは止められません (もちろん、あなたのトランザクションの宛先も同じ設定を使っている場合は別です)。

### TransferRate

`TransferRate` フィールドは、自分が発行する通貨を相手方が転送するたびに課す手数料を指定します。

HTTP APIとWebSocket APIでは、転送手数料は整数として表されます。これは、10億単位を到着させるために送信しなければならない量です。たとえば、20%の転送手数料は `1200000000` という値で表されます。この値は1000000000未満にはできません。(それ未満だと、トランザクションを送ることで資金を与えることを意味し、悪用可能です。)手数料なしを意味する `1000000000` の短縮形として `0` を指定できます。

詳しくはTransfer Feesを参照してください。
