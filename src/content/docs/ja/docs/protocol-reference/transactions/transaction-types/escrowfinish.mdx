### EscrowIDを使って完了

```json
{
    "Account": "rf1BiGeXwwQoi8Z2ueFYTEXSwuJYfV2Jpn",
    "TransactionType": "EscrowFinish",
    "Owner": "rf1BiGeXwwQoi8Z2ueFYTEXSwuJYfV2Jpn",
    "EscrowID": "49647F0D748DC3FE26BDACBC57F251AADEFFF391403EC9BF87C97F67E9977FB0",
    "Condition": "A0258020E3B0C44298FC1C149AFBF4C8996FB92427AE41E4649B934CA495991B7852B855810100",
    "Fulfillment": "A0028000"
}
```

### フィールド

| フィールド           | JSON型 | \[Internal Type]\[] | 説明                                                                                                                                                                                         |
| --------------- | --------- | ------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Owner`         | String    | AccountID           | エスクローに資金を出した送信元アカウントのアドレス。                                                                                                                                         |
| `OfferSequence` | Number    | UInt32              | _(任意)_ 完了させるエスクローを作成した \[EscrowCreate transaction]\[]のトランザクションシーケンス。                                                                                        |
| `Condition`     | String    | Blob                | _(任意)_ 事前に指定されたエスクローの [PREIMAGE-SHA-256 crypto-condition](https://tools.ietf.org/html/draft-thomas-crypto-conditions-02#section-8.1)と一致するhexadecimal値。         |
| `Fulfillment`   | String    | Blob                | _(任意)_ エスクローの`Condition`と一致する [PREIMAGE-SHA-256 crypto-condition fulfillment](https://tools.ietf.org/html/draft-thomas-crypto-conditions-02#section-8.1.4)のhexadecimal値。 |
| `EscrowID`      | String    | Hash256             | _(任意)_ 完了させるEscrowレジャーオブジェクトのID。64文字のhexadecimal文字列です。                                                                                                    |

どのアカウントでもEscrowFinishトランザクションを送信できます。

* エスクローに`FinishAfter`時刻がある場合、その時刻より前には実行できません。具体的には、対応する \[EscrowCreate transaction]\[]で指定された`FinishAfter`時刻が、直近にクローズされたレジャーのclose timeより後である場合、EscrowFinishトランザクションは失敗します。
* エスクローに`Condition`がある場合、その条件に一致する`Fulfillment`を指定しない限り実行できません。
* エスクローは、期限切れ後には実行できません。具体的には、対応する \[EscrowCreate transaction]\[]で指定された`CancelAfter`時刻が、直近にクローズされたレジャーのclose timeより前である場合、EscrowFinishトランザクションは失敗します。

**Note:** EscrowFinishトランザクションの最小トランザクションコストは、fulfillmentを含む場合に増加します。fulfillmentを含まない場合、トランザクションコストは標準の10 dropsです。fulfillmentを含む場合、トランザクションコストは330 \[drops of XAH]\[]に加えて、preimageのサイズ16 bytesごとにさらに10 dropsかかります。

本番以外のネットワークでは、保留中のescrowの宛先アカウントを削除できる場合があります。この場合、escrowを完了しようとすると`tecNO_TARGET`で失敗しますが、通常どおり期限切れになるまではescrowオブジェクトは残ります。別の支払いによって宛先アカウントが再作成されれば、escrowは正常に完了できます。escrowの宛先アカウントが削除できるのは、そのescrowがfix1523 amendmentの有効化より前に作成された場合だけです。production Xahauにはそのようなescrowは存在しないため、このエッジケースはproduction Xahauでは起こりません。また、新しいgenesis ledgerを開始したときの既定値である、fix1523とEscrowの両amendmentを同時に有効にするテストネットでも、このエッジケースは起こりません。
