description: >-
  レジャー内のCheckオブジェクトを償還し、対応するCheckCreateトランザクションで承認された金額まで受け取ろうとします。
---
\[[ソース](https://github.com/Xahau/xahaud/blob/dev/src/ripple/app/tx/impl/URIToken.cpp)]

_(\[Checks amendment]\[]により追加。)_

### 例

```json
{
    "Account": "rfkE1aSy9G8Upk4JssnwBxhEv5p4mn2KTy",
    "TransactionType": "CheckCash",
    "Amount": "100000000",
    "CheckID": "838766BA2B995C00744175F69A1B11E32C3DBC40E64801A4056FCBD657F57334",
    "Fee": "12"
}
```

| フィールド        | JSON型                                                                                                                          | \[Internal Type]\[] | 説明                                                                                                                                                                                                                     |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------- | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CheckID`    | String                                                                                                                             | Hash256             | 現金化するCheckレジャーオブジェクトのID。64文字のhexadecimal文字列です。                                                                                                                                                 |
| `Amount`     | [Currency Amount](/ja/docs/protocol-reference/data-types/currency-formats/#specifying-currency-amounts) | Amount              | _(任意)_ 可能であれば、この金額ちょうどでCheckを償還します。通貨は、対応するCheckCreateトランザクションの `SendMax` と一致していなければなりません。このフィールドか `DeliverMin` のどちらかを指定する必要があります。 |
| `DeliverMin` | \[Currency Amount]\[]                                                                                                              | Amount              | _(任意)_ この金額以上を、可能な限り多く償還します。通貨は、対応するCheckCreateトランザクションの `SendMax` と一致していなければなりません。このフィールドか `Amount` のどちらかを指定する必要があります。                   |

トランザクションには `Amount` または `DeliverMin` のどちらかを _**必ず**_ 含める必要があります。両方は指定できません。

### エラーケース

* CheckCashトランザクションの送信者がcheckの `Destination` でない場合、トランザクションは `tecNO_PERMISSION` で失敗します。
* `CheckID` フィールドで指定されたCheckが存在しない場合、トランザクションは `tecNO_ENTRY` で失敗します。
* `CheckID` フィールドで指定されたCheckがすでに期限切れの場合、トランザクションは `tecEXPIRED` で失敗します。
* Checkの宛先が `RequireDest` フラグを有効にしているにもかかわらず、作成されたCheckにdestination tagがない場合、トランザクションは `tecDST_TAG_NEEDED` で失敗します。
* トランザクションが `Amount` と `DeliverMin` の両方を指定した場合、または両方を省略した場合、トランザクションは `temMALFORMED` で失敗します。
* `Amount` または `DeliverMin` が、Checkの通貨（およびXAHでない場合はissuer）と一致しない場合、トランザクションは `temBAD_CURRENCY` で失敗します。
