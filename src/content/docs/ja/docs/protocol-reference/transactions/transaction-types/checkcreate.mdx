---
title: CheckCreate
description: >-
  レジャー内にCheckオブジェクトを作成します。これは、意図された宛先が現金化できる繰延支払いです。このトランザクションの送信者はCheckの送信者です。
---
\[[ソース](https://github.com/Xahau/xahaud/blob/dev/src/ripple/app/tx/impl/URIToken.cpp)]

_(\[Checks amendment]\[]により追加。)_

### 例

```json
{
  "TransactionType": "CheckCreate",
  "Account": "rUn84CUYbNjRoTQ6mSW7BVJPSVJNLb1QLo",
  "Destination": "rfkE1aSy9G8Upk4JssnwBxhEv5p4mn2KTy",
  "SendMax": "100000000",
  "Expiration": 570113521,
  "InvoiceID": "6F1DFD1D0FE8A32E40E1F2C05CF1C15545BAB56B617F9C6C2D63A6B704BEF59B",
  "DestinationTag": 1,
  "Fee": "12"
}
```

| フィールド            | JSON型                                                                                                                          | \[Internal Type]\[] | 説明                                                                                                                                                                                                                                                                                                 |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Destination`    | String                                                                                                                             | AccountID           | Checkを現金化できるアカウントの固有アドレス。                                                                                                                                                                                                                                                            |
| `SendMax`        | [Currency Amount](/ja/docs/protocol-reference/data-types/currency-formats/#specifying-currency-amounts) | Amount              | Checkが送信者から引き落とせる送信元通貨の最大額。XAH以外では送金手数料を含みます。Checkは、宛先に同じ通貨（XAH以外は同じissuer）だけを入金できます。XAH以外の金額では、ネストされたフィールド名は必ず小文字にする必要があります。 |
| `DestinationTag` | Number                                                                                                                             | UInt32              | _(任意)_ Checkの理由、または支払い先のホスト型受取人を識別する任意のタグ。                                                                                                                                                                                                                           |
| `Expiration`     | Number                                                                                                                             | UInt32              | _(任意)_ この時刻以降、Checkが無効になる時刻。\[seconds since the Ripple Epoch]\[]で指定します。                                                                                                                                                                                                      |
| `InvoiceID`      | String                                                                                                                             | Hash256             | _(任意)_ このCheckの特定の理由または識別子を表す任意の256-bit hash。                                                                                                                                                                                                                               |

### エラーケース

* `Destination` アカウントがincoming Checksをブロックしている場合、トランザクションは `tecNO_PERMISSION` で失敗します。&#x20;
* `Destination` がトランザクションの送信者である場合、トランザクションは `temREDUNDANT` で失敗します。
* `Destination` アカウントがレジャー内に存在しない場合、トランザクションは `tecNO_DST` で失敗します。
* `Destination` アカウントで `RequireDest` フラグが有効なのに、トランザクションに `DestinationTag` フィールドが含まれていない場合、トランザクションは `tecDST_TAG_NEEDED` で失敗します。
* `SendMax` が凍結されたtokenを指定している場合、トランザクションは `tecFROZEN` で失敗します。
* トランザクションの `Expiration` が過去である場合、トランザクションは `tecEXPIRED` で失敗します。
* Checkを追加した後に送信者がowner reserveを満たすだけのXAHを持っていない場合、トランザクションは `tecINSUFFICIENT_RESERVE` で失敗します。
* Checkの送信者または宛先のいずれかが、レジャー内でこれ以上オブジェクトを所有できない場合、トランザクションは `tecDIR_FULL` で失敗します。
