---
title: TERコード
---
これらのコードは、トランザクションが失敗したものの、将来的には成功して適用できる可能性があることを示します。通常は、別の仮想的なトランザクションが先に適用された場合です。数値は-99から-1の範囲です。各エラーに対する正確なコードは変更される可能性があるため、依存しないでください。

**注意:** `ter` コードのトランザクションは現在のレジャーには適用されず、Xahauの状態を変更することもありません。ただし、暫定的に失敗したトランザクションでも、自動再適用後に別のコードで成功または失敗する可能性があります。詳細は、結果の確定性およびReliable Transaction Submissionを参照してください。

| コード           | 説明                                                                                                                                                                                                                      |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `terFUNDS_SPENT` | **非推奨。**                                                                                                                                                                                                                  |
| `terINSUF_FEE_B` | トランザクションを送信したアカウントに、トランザクションで指定された `Fee` を支払うだけの十分なXAHがありません。                                                                                                                      |
| `terLAST`        | 内部専用です。このコードが返されることはありません。                                                                                                                                                                        |
| `terNO_ACCOUNT`  | トランザクションを送信したアドレスは、まだレジャーで資金供給されていません。                                                                                                                                                           |
| `terNO_AUTH`     | このトランザクションでは、`lsfRequireAuth` が有効なアカウントが発行した通貨を、未認可のトラストラインへ追加することになります。たとえば、保持が認可されていない通貨を買うオファーを出した場合です。       |
| `terNO_LINE`     | 内部専用です。このコードが返されることはありません。                                                                                                                                                                        |
| `terNO_RIPPLE`   | 内部専用です。このコードが返されることはありません。                                                                                                                                                                        |
| `terOWNERS`      | このトランザクションが成功するには、送信元アカウントの "owners count" が0ではない必要があります。そのため、このトランザクションは成功できません。たとえば、アカウントにトラストラインや利用可能なOfferがある場合、`lsfRequireAuth` フラグを有効にすることはできません。 |
| `terPRE_SEQ`     | 現在のトランザクションの `Sequence` 番号が、トランザクションを送信したアカウントの現在のシーケンス番号より大きいです。                                                                                              |
| `terPRE_TICKET`  | トランザクションはTicketを使おうとしましたが、指定された `TicketSequence` 番号はレジャーに存在しません。ただし、そのTicketは別のトランザクションで作成できる可能性があります。                                            |
| `terRETRY`       | 原因不明の再試行可能なエラーです。                                                                                                                                                                                                     |
| `terQUEUED`      | トランザクションは負荷調整後のトランザクションコストを満たしましたが、open ledgerの要件を満たさなかったため、将来のレジャー向けにキューへ入れられました。                                                                       |
| `terNO_HOOK`     | トランザクションは、レジャーに存在しない `HookHash` を使おうとしました。                                                                                                                                                  |
