---
title: TEMコード
---
これらのコードは、トランザクションが不正であり、Xahauプロトコルに従う限り成功できないことを示します。数値は-299から-200の範囲です。各エラーに対する正確なコードは変更される可能性があるため、依存しないでください。

**ヒント:** `tem`コードのトランザクションはレジャーに適用されず、Xahauの状態を変更することもありません。`tem` の結果は、有効なトランザクションのルールが変わらない限り最終確定です。（たとえば、あるAmendmentが有効になる前にそのAmendmentの機能を使うと`temDISABLED`になります。そのトランザクションは、Amendmentが有効になって有効な内容になれば、後で成功する可能性があります。）

| コード                         | 説明                                                                                                                                                                                                                                                                            |
| ------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `temBAD_AMOUNT`                | トランザクションで指定された金額（たとえば \[Payment]\[]の宛先`Amount`や`SendMax`の値）が無効でした。負の数であった可能性があります。                                                                                                           |
| `temBAD_AUTH_MASTER`           | このトランザクションに署名した鍵が、送信元アカウントのmaster keyと一致しておらず、かつそのアカウントにRegular Keyが設定されていません。                                                                                                                                       |
| `temBAD_CURRENCY`              | トランザクションでcurrencyフィールドが不適切に指定されていました。正しい形式は \[Specifying Currency Amounts]\[Currency Amount]を参照してください。                                                                                                                                                    |
| `temBAD_EXPIRATION`            | トランザクションで有効期限の値が不適切に指定されていました。たとえば \[OfferCreate transaction]\[]の一部として指定された場合です。逆に、たとえば \[EscrowCreate transaction]\[]の一部として必要な有効期限が指定されていない場合もあります。                               |
| `temBAD_FEE`                   | トランザクションで`Fee`値が不適切に指定されていました。たとえば、XAH以外の通貨を指定したり、負のXAHを指定したりした場合です。                                                                                                                                                        |
| `temBAD_ISSUER`                | リクエストに含まれる通貨の`issuer`フィールドが不適切に指定されていました。                                                                                                                                                                                      |
| `temBAD_LIMIT`                 | \[TrustSet transaction]\[]で、トラストラインの`LimitAmount`値が不適切に指定されていました。                                                                                                                                                                                           |
| `temBAD_NFTOKEN_TRANSFER_FEE`  | \[NFTokenMint transaction]\[]で、トランザクションの`TransferFee`フィールドが不適切に指定されていました。_(NonFungibleTokensV1_1 amendmentにより追加。)_                                                                                                                            |
| `temBAD_OFFER`                 | \[OfferCreate transaction]\[]が、XAHを自分自身と交換するような内容や、負の金額などの無効なオファーを指定していました。                                                                                                                                                 |
| `temBAD_PATH`                  | \[Payment transaction]\[]が、1つ以上のPathsを不適切に指定していました。たとえばXAHに対するissuerを含めていたり、アカウントを別の形で指定していたりする場合です。                                                                                                                                   |
| `temBAD_PATH_LOOP`             | \[Payment transaction]\[]内のPathsの1つがループとしてフラグ付けされており、有限時間では処理できません。                                                                                                                                                        |
| `temBAD_SEND_NATIVE_LIMIT`     | \[Payment transaction]\[]が、XAHからXAHへの直接支払いで`tfLimitQuality`フラグを使っていました。XAH同士の支払いでは変換が発生しないため、これは不要です。                                                                                                                           |
| `temBAD_SEND_NATIVE_MAX`       | \[Payment transaction]\[]が、XAHからXAHへの直接支払いで`SendMax`フィールドを含んでいました。XAHを送るのに`SendMax`は必要ありません。（XAHは、宛先`Amount`がXAHでない場合にのみ`SendMax`で有効です。）                                                              |
| `temBAD_SEND_NATIVE_NO_DIRECT` | \[Payment transaction]\[]が、XAHからXAHへの直接支払いに`tfNoDirectRipple`フラグを使っていました。XAH同士の支払いは常に直接です。                                                                                                                                     |
| `temBAD_SEND_NATIVE_PARTIAL`   | \[Payment transaction]\[]が、XAHからXAHへの直接支払いに`tfPartialPayment`フラグを使っていました。XAH同士の支払いは常に全額を送るべきです。                                                                                                                 |
| `temBAD_SEND_NATIVE_PATHS`     | \[Payment transaction]\[]がXAHを送る際に`Paths`を含めていました。XAH同士の支払いは常に直接であるべきです。                                                                                                                                                             |
| `temBAD_SEQUENCE`              | トランザクションが、自身の`Sequence`番号よりも大きいシーケンス番号を参照していました。たとえば、そのオファーをキャンセルするトランザクションより後に置かれるはずのオファーをキャンセルしようとした場合です。                                                                               |
| `temBAD_SIGNATURE`             | このトランザクションを認可する署名がないか、署名の形式が正しい形式になっていません。（署名自体は正しいが、このアカウントでは認可されていない場合は`tecNO_PERMISSION`を参照してください。）                                          |
| `temBAD_SRC_ACCOUNT`           | このトランザクションを送っている`Account`（送信元アカウント）が正しい形式のアカウントアドレスではありません。                                                                                                                                                          |
| `temBAD_TRANSFER_RATE`         | AccountSetトランザクションの`TransferRate`フィールドの形式が正しくないか、許容範囲外です。                                                                                                                                                                        |
| `temCANNOT_PREAUTH_SELF`       | \[DepositPreauth transaction]\[]の送信者が、事前認可対象のアカウントとしても指定されていました。自分自身を事前認可することはできません。                                                                                                                                                |
| `temDST_IS_SRC`                | トランザクションが、送信元としての`Account`に宛先アドレスを不適切に指定していました。これにはトラストライン（宛先アドレスが`LimitAmount`の`issuer`フィールドである場合）とPayment Channel（宛先アドレスが`Destination`フィールドである場合）が含まれます。 |
| `temDST_NEEDED`                | トランザクションが宛先の指定を不適切に省略していました。これは \[Payment transaction]\[]の`Destination`フィールドである場合もあれば、TrustSetトランザクションの`LimitAmount`フィールド内の`issuer`サブフィールドである場合もあります。                                                                              |
| `temINVALID`                   | それ以外の理由でトランザクションが無効です。たとえば、トランザクションIDの形式が正しくない、署名が正しく形成されていない、その他トランザクションの解釈中に問題が発生したなどです。                                                                            |
| `temINVALID_COUNT`             | トランザクションに`TicketCount`フィールドがありますが、指定されたTicketの数が無効です。                                                                                                                                                                                        |
| `temINVALID_FLAG`              | トランザクションに存在しないFlagが含まれているか、矛盾するフラグの組み合わせが含まれています。                                                                                                                                                                                 |
| `temMALFORMED`                 | トランザクション形式に関する原因不明の問題です。                                                                                                                                                                                                                                |
| `temREDUNDANT`                 | このトランザクションは何もしません。たとえば、送信元アカウントへ直接支払いを送っている場合や、同じ発行者から同じ通貨を売買するオファーを作成している場合です。                                                                                                   |
| `temREDUNDANT_SEND_MAX`        | \[Removed in: rippled 0.28.0]\[]                                                                                                                                                                                                                                                       |
| `temRIPPLE_EMPTY`              | \[Payment transaction]\[]に空の`Paths`フィールドが含まれていますが、この支払いを完了するにはパスが必要です。                                                                                                                                                                       |
| `temBAD_WEIGHT`                | \[SignerListSet transaction]\[]に、無効な `SignerWeight`（たとえば0や負の値）が含まれています。                                                                                                                                                                   |
| `temBAD_SIGNER`                | \[SignerListSet transaction]\[]に無効なsignerが含まれています。たとえば、重複エントリがあるか、SignerListの所有者がメンバーにもなっている場合です。                                                                                                                |
| `temBAD_QUORUM`                | \[SignerListSet transaction]\[]の`SignerQuorum`値が無効です。値が0より大きくないか、リスト内のすべてのsignerの合計を上回っています。                                                                                                             |
| `temUNCERTAIN`                 | 内部専用です。このコードが返されることはありません。                                                                                                                                                                                                                              |
| `temUNKNOWN`                   | 内部専用です。このコードが返されることはありません。                                                                                                                                                                                                                              |
| `temDISABLED`                  | トランザクションが、無効になっているロジックを必要としています。通常、これは現在のレジャーで有効になっていないAmendmentを使おうとしていることを意味します。                                                                                                                                   |
| `temHOOK_DATA_TOO_LARGE`       | トランザクションの`CreateCode`フィールドに256バイトを超えるデータが含まれています。                                                                                                                                                                                                                       |
