---
title: TESコード
---
`tesSUCCESS` と `tesPARTIAL` は、トランザクションが成功したことを示す唯一のコードです。ただし、必ずしも期待どおりの結果になったとは限りません。（たとえば、キャンセル対象のOfferがなくても、\[OfferCancel]\[]は「成功」し得ます。）&#x20;

`tesPARTIAL` コードは、トランザクションが部分的に成功したことを示します。具体的には、Hookの状態レコード削除のような操作が、ある上限（たとえば512件や1024件）を超えた場合に使われます。その場合、トランザクションは上限まで削除し、その後、残りのレコードがすべて削除されるまで、新しいシーケンス番号を持つ追加トランザクションを送信する必要があります。これにより、大きな操作を扱いやすい単位に分割できます。

| コード       | 説明                                                                                                                                               |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `tesSUCCESS` | トランザクションは適用され、他のサーバーへ転送されました。これが検証済みレジャーに現れた場合、そのトランザクションの成功は最終確定です。               |
| `tesPARTIAL` | トランザクションは部分的に成功しており、所定の上限を超えるレコードの削除など、大きな操作を完了するには追加トランザクションが必要です。 |
