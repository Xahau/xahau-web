title: 通貨フォーマット
---
Xahauには2種類のデジタルアセットがあります。XAHとトークンです。どちらも高い精度を持ちますが、フォーマットは異なります。

### 比較

次の表は、XahauにおけるXAHとトークンの主な違いをまとめたものです。

| XAH                                                                                  | トークン                                                                                       |
| ------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------- |
| 発行者を持ちません。                                                                 | 常にXahauアカウントによって発行されます。                                                   |
| 文字列として指定します。                                                             | オブジェクトとして指定します。                                                                 |
| アカウントで追跡されます。                                                           | トラストラインで追跡されます。                                                                 |
| 作成されることはなく、破棄されるだけです。                                           | 自由に発行または償還できます。                                                                 |
| 最小値: `0`。（負の値にはできません。）                                              | 最小値: `-9999999999999999e80`。最小の非ゼロ絶対値: `1000000000000000e-96`。                    |
| 最大値は `100000000000` (1011) XAHです。これは `100000000000000000` (1017) "drops" です。 | 最大値は `9999999999999999e80` です。                                                          |
| "drop" 単位（0.000001 XAH）までの精度があります。                                    | 小数点以下15桁の精度があります。                                                             |
| 凍結できません。                                                                     | 発行者は残高を凍結できます。                                                                   |
| 転送手数料はありません。XAH同士の支払いは常に直接です。                             | 各発行者が一定割合の転送手数料を課す、間接的な経路を取ることがあります。                       |
| Payment ChannelsとEscrowで使用できます。                                          | Payment ChannelsまたはEscrowで使用できます。                                              |

詳細は、What is XAH? とTokensを参照してください。

### 通貨金額の指定 <a id="specifying-currency-amounts" />

指定したい通貨の種類に応じて、適切な形式を使います。

* XAH金額
* トークン金額

#### XAH金額

XAHの金額を指定するには、XAHの_drops_を示すString Numberを使います。1 dropは0.000001 XAHです。たとえば、13.1 XAHを指定するには次のようにします。

```
"13100000"
```

**XAHをオブジェクトとして指定しないでください。**

XAHの金額は負の値にできません。

#### トークン金額

（代替可能な）トークンの金額を指定するには、Amountオブジェクトを使います。これは3つのフィールドを持つJSONオブジェクトです。

| フィールド | 型                     | 説明                                                                                                                                                                                                                                                                                                              |
| ---------- | ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `currency` | String - Currency Code | トークンの任意の通貨コードです。XAHは使用できません。                                                                                                                                                                                                                                                         |
| `value`    | String Number          | トークン金額の引用付き10進数表現です。`1.23e11` のような科学記法も使え、これは123,000,000,000を意味します。`e` と `E` の両方を使用できます。残高を表示する場合は負の値にもできますが、送金額の指定など他の場面では負の値は許可されません。 |
| `issuer`   | String                 | 通常は、このトークンを発行するアカウントです。特別な場合には、代わりにトークンを保有しているアカウントを指すことがあります（たとえばClawbackトランザクション）。                                                                                                                                                |

**注意:** これらのフィールド名は大文字と小文字を区別します。

たとえば、アカウント `r9cZA1mLK5R5Am25ArfXFmqgNwjZgnfk59` が発行した153.75米ドルを表すには、次のように指定します。

```json
{
    "currency": "USD",
    "value": "153.75",
    "issuer": "r9cZA1mLK5R5Am25ArfXFmqgNwjZgnfk59"
}
```

#### 金額を指定しない場合

場合によっては、分散型取引所でオーダーブックを定義する場合のように、特定の金額を指定せずにアセット（XAHまたはトークン）を定義する必要があります。

金額なしでトークンを表すには、currencyオブジェクトとして指定し、`value` フィールドを省略します。たとえば次のようにします。

```json
{
  "currency": "TST",
  "issuer": "rP9jPyP5kyvFRb6ZiRghAGw5u8SGAmU4bd"
}
```

金額なしでXAHを表すには、`currency` フィールド_のみ_を持つJSONオブジェクトとして指定します。XAHに `issuer` フィールドを含めてはいけません。たとえば次のようにします。

```json
{
  "currency": "XAH"
}
```

### String Number

### XAHの精度

XAHの精度は、各単位が0.000001 XAHに相当する64ビット符号なし整数と同じです。整数演算を使うため、1 drop未満の金額は切り捨てられます。

### トークンの精度

トークンは、非常に小さい単位や非常に大きい単位で測定されるものを含め、さまざまなアセットを表現できます。このフォーマットは、科学記法に似た方法で、有効数字と10の累乗の指数を使います。指定範囲内の正負の有効数字と指数をサポートします。非整数の一般的な浮動小数点表現とは異なり、このフォーマットはすべての計算で整数演算を使うため、常に小数点以下15桁の精度を維持します。乗算と除算では、最下位桁での過剰な丸めを補正する調整が行われます。

Xahauのピアツーピアネットワークでトークン金額を送信するとき、サーバーはその金額を64ビットのバイナリ値としてシリアライズします。

**ヒント:** まったく分割できないトークンについては、Non-Fungible Tokens (NFTs)を参照してください。

### 通貨コード

#### 標準通貨コード

通貨コードの標準形式は、`USD` のような3文字の文字列です。これは [ISO 4217 Currency Codes](https://www.xe.com/iso4217.php)での使用を想定しています。次のルールが適用されます。

* 通貨コードはちょうど3文字のASCII文字でなければなりません。許可される文字は、大文字と小文字の英字、数字、および `?`, `!`, `@`, `#`, `$`, `%`, `^`, `&`, `*`, `<`, `>`, `(`, `)`, `{`, `}`, `[`, `]`, `|` です。
* 通貨コードは大文字と小文字を区別します。
* 通貨コードXAH（すべて大文字）は使用できません。実際のXAHは通常、Xahauのプロトコルで通貨コードを使用しません。

プロトコルレベルでは、この形式は `0x00` で始まる160ビットのバイナリ値としてシリアライズされます。

#### 非標準通貨コード

`015841551A748AD2C1F76FF6ECB0CCCD00000000` のような160ビット（40文字）の16進文字列を通貨コードとして使うこともできます。これが「標準」通貨コードとして扱われないようにするため、先頭8ビットは `0x00` であってはなりません。

**非推奨:** [ripple-lib](https://github.com/XRPLF/xrpl.js)の以前の一部バージョンでは、「利付き」または「デマラージュ」の通貨コード型がサポートされていました。これらのコードは先頭8ビットが `0x01` です。デマラージュ/利付き通貨は現在サポートされていませんが、レジャーデータ内に見つかることがあります。詳細は、Demurrageを参照してください。
