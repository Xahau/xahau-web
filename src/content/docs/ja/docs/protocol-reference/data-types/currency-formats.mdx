---
title: 通貨形式
---
Xahau には 2 種類のデジタル資産があります。XAH と tokens です。どちらも高い精度を持ちますが、形式は異なります。

### 比較

次の表は、Xahau における XAH と tokens の違いをまとめたものです。

| XAH                                                                                  | Tokens                                                                                         |
| ------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------- |
| issuer を持たない。                                                                  | 常に Xahau account によって発行される。                                                        |
| 文字列として指定する。                                                               | オブジェクトとして指定する。                                                                    |
| account に記録される。                                                               | trust line に記録される。                                                                       |
| 作成されることはなく、破棄されるのみ。                                               | 自由に発行または償還できる。                                                                    |
| 最小値: `0`。（負値は不可。）                                                        | 最小値: `-9999999999999999e80`。ゼロでない絶対値の最小値: `1000000000000000e-96`。              |
| 最大値 `100000000000` (1011) XAH。これは `100000000000000000` (1017) drops に相当。 | 最大値 `9999999999999999e80`。                                                                  |
| 最小単位は 1 drop（0.000001 XAH）                                                    | 10 進 15 桁精度。                                                                               |
| 凍結できない。                                                                        | issuer は残高を凍結できる。                                                                     |
| 転送手数料はなく、XAH から XAH の支払いは常に直接。                                  | 間接パスを取ることができ、各 issuer が割合ベースの transfer fee を課すことがある。              |
| Payment Channels と Escrow で使用できる。                                            | Payment Channels または Escrow で使用できる。                                                  |

詳細は What is XAH? および Tokens を参照してください。

### 通貨額の指定

指定したい通貨の種類に応じて、適切な形式を使ってください。

* XAH Amounts
* Token Amounts

#### XAH Amounts

XAH の量を指定するには、XAH の _drops_ 単位を表す文字列数値を使います。1 drop は 0.000001 XAH です。たとえば 13.1 XAH を指定する場合:

```
"13100000"
```

**XAH をオブジェクト形式で指定してはいけません。**

XAH の量は負値にできません。

#### Token Amounts

（代替可能な）token の量を指定するには、Amount object を使います。これは 3 つのフィールドを持つ JSON オブジェクトです。

| `Field`    | Type                   | Description                                                                                                                                                                                                                                                                                                       |
| ---------- | ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `currency` | String - Currency Code | token の任意通貨コードです。XAH は使用できません。                                                                                                                                                                                                                                                                |
| `value`    | String Number          | token 量の 10 進文字列表現です。`1.23e11`（123,000,000,000 を意味する）などの科学表記を含められます。`e` と `E` はどちらも使用できます。残高表示では負値を含められますが、送金量指定など他の文脈では負値は許可されません。                                                                                |
| `issuer`   | String                 | 通常はこの token を発行した account です。特別な場合には、代わりに token を保有している account を指すこともあります（たとえば Clawback transaction 内）。                                                                                                                                                       |

**注意:** これらのフィールド名は大文字小文字を区別します。

たとえば、account `r9cZA1mLK5R5Am25ArfXFmqgNwjZgnfk59` が発行した 153.75 米ドルを表すには、次のように指定します。

```json
{
    "currency": "USD",
    "value": "153.75",
    "issuer": "r9cZA1mLK5R5Am25ArfXFmqgNwjZgnfk59"
}
```

#### Amount を伴わずに指定する

場合によっては、分散型取引所の order book を定義する時のように、具体的な量なしで資産（XAH または token）を定義する必要があります。

量なしの token を表すには、currency object として指定し、`value` フィールドを省略します。たとえば:

```json
{
  "currency": "TST",
  "issuer": "rP9jPyP5kyvFRb6ZiRghAGw5u8SGAmU4bd"
}
```

量なしの XAH を表すには、`currency` フィールド _のみ_ を持つ JSON オブジェクトとして指定します。XAH に `issuer` フィールドを含めてはいけません。たとえば:

```json
{
  "currency": "XAH"
}
```

### String Numbers

### XAH Precision

XAH は、各単位が 0.000001 XAH に相当する 64 ビット符号なし整数と同じ精度を持ちます。整数演算を使うため、1 drop 未満の量は切り捨てられます。

### Token Precision

Tokens は、非常に小さい単位や非常に大きい単位で測られる資産を含む、幅広い資産を表現できます。この形式は、有効数字と 10 のべき指数を、科学表記に似た形で使います。この形式は、指定範囲内の正負の有効数字および指数をサポートします。一般的な浮動小数点表現と異なり、すべての計算で整数演算を使うため、常に 15 桁の 10 進精度を維持します。乗算と除算では、最下位桁の過剰丸めを補正するための調整が入ります。

Xahau のピアツーピアネットワークで token 量を送信する際、サーバーはその量を 64 ビットのバイナリ値へシリアライズします。

**ヒント:** まったく分割可能でない token が必要な場合は、Non-Fungible Tokens（NFTs）を参照してください。

### 通貨コード

#### 標準通貨コード

通貨コードの標準形式は、`USD` のような 3 文字文字列です。これは [ISO 4217 Currency Codes](https://www.xe.com/iso4217.php) を使うことを意図しています。次のルールが適用されます。

* 通貨コードは必ず長さ 3 の ASCII 文字でなければなりません。使用可能文字は、大文字・小文字の英字、数字、ならびに `?`, `!`, `@`, `#`, `$`, `%`, `^`, `&`, `*`, `<`, `>`, `(`, `)`, `{`, `}`, `[`, `]`, `|` です。
* 通貨コードは大文字小文字を区別します。
* 通貨コード `XAH`（すべて大文字）は使用禁止です。実際の XAH は通常、Xahau プロトコル内で通貨コードを使いません。

プロトコルレベルでは、この形式は `0x00` で始まる 160 ビットのバイナリ値へシリアライズされます。

#### 非標準通貨コード

`015841551A748AD2C1F76FF6ECB0CCCD00000000` のような 160 ビット（40 文字）の 16 進文字列を通貨コードとして使うこともできます。これが「標準」通貨コードとして扱われないよう、先頭 8 ビットは **必ず** `0x00` 以外でなければなりません。

**非推奨:** [ripple-lib](https://github.com/XRPLF/xrpl.js) の一部旧バージョンでは、「interest-bearing」または「demurraging」通貨コード型をサポートしていました。これらのコードは先頭 8 ビットが `0x01` です。Demurraging / interest-bearing currencies は現在サポートされていませんが、ledger data 内に見つかることはあります。詳細は Demurrage を参照してください。
