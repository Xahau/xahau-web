---
title: Base 58エンコーディング
---
XahauのAPIでは、アカウントアドレスや暗号鍵に関連する他の種類の値を表すために、チェックサム付きの "base58" エンコーディング（"Base58Check" とも呼ばれます）がよく使われます。このエンコーディングは [Bitcoinアドレスで使われているもの](https://en.bitcoin.it/wiki/Base58Check_encoding)と同じですが、Xahauでは次の辞書を使用します: `rpshnaf39wBUDNEGHJKLM4PQRST7VWXYZ2bcdeCg65jkm8oFqi1tuvAxyz`。

Xahauでは、異なるデータ型を区別するために、エンコードする前に値の種類ごとに特定の8ビット数を接頭辞として付けます。Xahauのbase58辞書の文字配置により、エンコードされた値の種類ごとのbase58表現は、型ごとに特定の文字で始まります。

次の表は、Xahauが使用するすべてのエンコーディングを示しています。

| データ型                                 | 先頭文字 | 型プレフィックス | 内容サイズ¹ | 最大文字数 |
| ---------------------------------------- | -------- | ---------------- | ----------- | ---------- |
| アカウントアドレス                      | r        | `0x00`           | 20 bytes    | 35         |
| アカウント公開鍵                         | a        | `0x23`           | 33 bytes    | 53         |
| Seed値（秘密鍵用）                      | s        | `0x21`           | 16 bytes    | 29         |
| 検証用公開鍵またはノード公開鍵           | n        | `0x1C`           | 33 bytes    | 53         |

¹ 内容サイズには1バイトの型プレフィックスは含まれません。
