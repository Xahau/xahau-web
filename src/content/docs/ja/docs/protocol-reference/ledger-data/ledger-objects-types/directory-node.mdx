`DirectoryNode` オブジェクトタイプは、レジャーの状態ツリーにある他のオブジェクトへのリンク一覧を提供します。概念上の1つの _Directory_ は二重連結リストとして表され、1つ以上のDirectoryNodeオブジェクトがそれぞれ最大32個の他のオブジェクトIDを含みます。最初のオブジェクトはディレクトリのrootと呼ばれ、root以外のすべてのオブジェクトは必要に応じて追加または削除できます。

Directoryには2種類あります。

* **Ownerディレクトリ** は、`RippleState`（トラストライン）や `Offer` オブジェクトなど、アカウントが所有する他のオブジェクトを一覧表示します。
* **Offerディレクトリ** は、分散型取引所で利用可能なOfferを一覧表示します。1つのOfferディレクトリには、同じトークン（通貨コードと発行者）に対して同じ為替レートを持つすべてのOfferが含まれます。

### フィールド

| フィールド         | JSON型   | \[内部型]\[]        | 必須?    | 説明                                                                                                                                                                                                                |
| ------------------- | --------- | ------------------- | --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Owner`             | String    | AccountID           | いいえ    | （Ownerディレクトリのみ）このディレクトリ内のオブジェクトを所有するアカウントのアドレス。                                                                                                                               |
| `TakerGetsCurrency` | String    | Hash160             | いいえ    | （Offerディレクトリのみ）このディレクトリ内のOfferにおける `TakerGets` 金額の通貨コード。                                                                                                                    |
| `TakerGetsIssuer`   | String    | Hash160             | いいえ    | （Offerディレクトリのみ）このディレクトリ内のOfferにおける `TakerGets` 金額の発行者。                                                                                                                           |
| `TakerPaysCurrency` | String    | Hash160             | いいえ    | （Offerディレクトリのみ）このディレクトリ内のOfferにおける `TakerPays` 金額の通貨コード。                                                                                                                    |
| `TakerPaysIssuer`   | String    | Hash160             | いいえ    | （Offerディレクトリのみ）このディレクトリ内のOfferにおける `TakerPays` 金額の発行者。                                                                                                                           |
| `ExchangeRate`      | String    | UInt64              | いいえ    | （Offerディレクトリのみ）**非推奨**。使用しないでください。                                                                                                                                                                       |
| `ReferenceCount`    | String    | UInt64              | いいえ    | レジャーでの参照カウントにより、同じHookの複数のインストールをレジャー上の同一オブジェクトへ向けられるため、保存負荷と手数料を削減できます。Hookがインストールされているアカウントがなくなったときだけ削除されます。 |
| `Indexes`           | Array     | Vector256           | はい       | このDirectoryの内容。ほかのオブジェクトIDの配列です。                                                                                                                                                          |
| `IndexNext`         | Number    | UInt64              | いいえ    | このDirectoryが複数ページで構成される場合、このIDは連鎖内の次のオブジェクトを指し、末尾では先頭へ戻ります。                                                                                                   |
| `IndexPrevious`     | Number    | UInt64              | いいえ    | このDirectoryが複数ページで構成される場合、このIDは連鎖内の前のオブジェクトを指し、先頭では末尾へ戻ります。                                                                                         |
| `NFTokenID`         | String    | Hash256             | いいえ    | 未使用です。                                                                                                                                                                                                                     |

### DirectoryのID形式

DirectoryNodeのIDを作成する式は、DirectoryNodeが表す内容に応じて3つあります。

* Ownerディレクトリの最初のページ（rootとも呼ばれます）
* Offerディレクトリの最初のページ
* どちらかの種類の後続ページ

**Ownerディレクトリの最初のページ** のIDは、次の値を順番に連結したものの \[SHA-512Half][]です。

* Ownerディレクトリ スペースキー (`0x004F`)
* `Owner` フィールドのAccountID

**Offerディレクトリの最初のページ** には特別なIDがあります。上位192ビットはオーダーブックを定義し、残りの64ビットはそのディレクトリ内のOfferの為替レートを定義します。（IDはビッグエンディアンなので、オーダーブックはより上位のビットにあり先頭に来て、レートはより下位のビットにあり末尾に来ます。）

これは、オーダーブックを最良のOfferから最悪のOfferへたどる方法を提供します。具体的には、最初の192ビットは、次の値を順番に連結したものの \[SHA-512Half][]の最初の192ビットです。

* Book Directoryスペースキー (`0x0042`)
* `TakerPaysCurrency` の160ビット通貨コード
* `TakerGetsCurrency` の160ビット通貨コード
* `TakerPaysIssuer` のAccountID
* `TakerGetsIssuer` のAccountID

OfferディレクトリのIDの下位64ビットは、そのディレクトリ内のOffer（群）における `TakerPays` 金額を `TakerGets` 金額で割ったものを、Xahauの内部金額形式で表した64ビット値です。

**DirectoryNodeがDirectoryの最初のページではない場合**（OwnerディレクトリかOfferディレクトリかに関係なく）、そのIDは次の値を順番に連結したものの \[SHA-512Half][]です。

* DirectoryNodeスペースキー (`0x0064`)
* root DirectoryNodeのID
* このオブジェクトのページ番号。（0がroot DirectoryNodeなので、この値は1以上の整数です。）
