`Offer`レジャーエントリは、Xahauの分散型取引所で通貨を交換するためのOfferを表します。（金融の文脈では、これはより一般的に _order_ と呼ばれます。）`OfferCreate` トランザクションは、レジャー内にある他のOfferを消費してもそのOfferを直ちに完全実行できない場合にのみ、レジャーに`Offer`エントリを作成します。

Offerは、レジャーに残ったままネットワーク内の別の活動によってunfundedになることがあります。トランザクションの処理中、ネットワークはそれらのトランザクションが見つけたunfunded Offerを自動的にpruneします。（そうでなければ、レジャーの状態を変更できるのは _トランザクション_ だけなので、unfunded Offerは残り続けます。）

### フィールド

`Offer`オブジェクトには次のフィールドがあります。

| 名前                | JSON型          | \[内部型]\[] | 必須? | 説明                                                                                                                                                                                                                                                              |
| ------------------- | ---------------- | ------------ | ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `Account`           | String           | AccountID    | はい  | このOfferを所有するアカウントのアドレスです。                                                                                                                                                                                                                         |
| `BookDirectory`     | String           | Hash256      | はい  | このOfferにリンクするOffer DirectoryのIDです。                                                                                                                                                                                                                  |
| `BookNode`          | String           | UInt64       | はい  | ディレクトリが複数ページで構成されている場合に、このオブジェクトへリンクするoffer directoryのどのページかを示すヒントです。                                                                                                                                              |
| `Expiration`        | Number           | UInt32       | いいえ | このOfferがunfundedと見なされる時刻を示します。詳細は \[時間の指定]\[]を参照してください。                                                                                                                                                                 |
| `Flags`             | Number           | UInt32       | はい  | このofferで有効になっている真偽値フラグのビットマップです。                                                                                                                                                                                                                       |
| `LedgerEntryType`   | String           | UInt16       | はい  | 文字列`Offer`に対応付けられる値`0x006F`は、このオブジェクトがOfferを表すことを示します。                                                                                                                                                                         |
| `OwnerNode`         | String           | UInt64       | はい  | ディレクトリが複数ページで構成されている場合に、このオブジェクトへリンクするowner directoryのどのページかを示すヒントです。**注:** このOfferは、それが含まれているowner directoryへの直接リンクを持ちません。`Account` からその値を導出できるためです。 |
| `PreviousTxnID`     | String           | Hash256      | はい  | このオブジェクトを最後に変更したトランザクションの識別ハッシュです。                                                                                                                                                                                         |
| `PreviousTxnLgrSeq` | Number           | UInt32       | はい  | このオブジェクトを最後に変更したトランザクションを含むレジャーの \[インデックス]\[Ledger Index]です。                                                                                                                                                         |
| `Sequence`          | Number           | UInt32       | はい  | この`Offer`オブジェクトを作成した \[OfferCreate]\[]トランザクションの`Sequence`値です。`Account` と組み合わせてこのOfferを識別するために使われます。                                                                                                               |
| `TakerPays`         | String or Object | Amount       | はい  | Offerの作成者が要求する、残りの金額と通貨の種類です。                                                                                                                                                                                                |
| `TakerGets`         | String or Object | Amount       | はい  | Offerの作成者が提供する、残りの金額と通貨の種類です。                                                                                                                                                                                           |

### Offerフラグ

`OfferCreate`トランザクションがofferオブジェクトを作成するときには、有効にも無効にもできる複数のオプションがあります。レジャーでは、フラグはbitwise-or演算で組み合わせられる二進数値として表されます。レジャー内のフラグのビット値は、トランザクションでそれらのフラグを有効化または無効化するために使う値とは異なります。レジャーのフラグ名は **`lsf`** で始まります。

`Offer`オブジェクトには、次のフラグ値を指定できます。

<table data-header-hidden><thead><tr><th width="135"></th><th></th><th></th><th></th><th></th></tr></thead><tbody><tr><td>フラグ名</td><td>16進値</td><td>10進値</td><td>対応するOfferCreateフラグ</td><td>説明</td></tr><tr><td><code>lsfPassive</code></td><td><code>0x00010000</code></td><td>65536</td><td><code>tfPassive</code></td><td>オブジェクトはpassive Offerとして配置されました。これはレジャー内のオブジェクトには影響しません。</td></tr><tr><td><code>lsfSell</code></td><td><code>0x00020000</code></td><td>131072</td><td><code>tfSell</code></td><td>オブジェクトはsell Offerとして配置されました。これはレジャー内のオブジェクトには影響しません（`tfSell`は、要求額よりも良いレートを得た場合にのみ意味がありますが、オブジェクトがレジャーに入った後ではそのようなことは起こりません）。</td></tr></tbody></table>

### OfferのID形式

`Offer`オブジェクトのIDは、次の値を順番に連結したものの \[SHA-512Half]\[]です。

* Offerスペースキー（`0x006F`）
* Offerを置いたアカウントのAccountID
*   Offerを作成した \[OfferCreateトランザクション]\[]のSequence番号。

    OfferCreateトランザクションでTicketを使用した場合は、代わりに`TicketSequence`の値を使用します。
