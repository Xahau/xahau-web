[\[出典\]](https://github.com/ripple/rippled/blob/master/src/ripple/protocol/impl/LedgerFormats.cpp#L104-L108)

（レジャーバージョンを一意に識別する \["レジャーハッシュ" 文字列データ型]\[Hash]と混同しないでください。この節では `LedgerHashes` レジャーオブジェクト型について説明します。）

`LedgerHashes` オブジェクト型には、このレジャーバージョンに至るまでの過去のレジャー履歴が、ハッシュの形で含まれます。このレジャー型のオブジェクトは、レジャーをクローズすると自動的に変更されます。（これは、トランザクションやpseudo-transactionなしでレジャーの状態データが変更される数少ない場面のひとつです。）`LedgerHashes` オブジェクトがあることで、現在のレジャーバージョンと、過去のレジャーバージョンを最大1回参照するだけで、前のレジャーのハッシュを調べられます。

`LedgerHashes` オブジェクトには2種類あります。どちらの型も同じフィールドを持ちます。各レジャーバージョンには次が含まれます。

* 「最近の履歴」の `LedgerHashes` オブジェクトがちょうど1つ
* 現在のレジャーインデックス（つまりレジャー履歴の長さ）に基づく、複数の「過去履歴」`LedgerHashes` オブジェクト。具体的には、Xahauは65536レジャーバージョンごとに新しい「過去履歴」オブジェクトを追加します。

**注:** 例外として、新しいジェネシスレジャーにはレジャー履歴がないため、`LedgerHashes` オブジェクトは1つもありません。

`LedgerHashes` オブジェクトの例（長さを省略）:

```json
{
  "LedgerEntryType": "LedgerHashes",
  "Flags": 0,
  "FirstLedgerSequence": 2,
  "LastLedgerSequence": 33872029,
  "Hashes": [
    "D638208ADBD04CBB10DE7B645D3AB4BA31489379411A3A347151702B6401AA78",
    "254D690864E418DDD9BCAC93F41B1F53B1AE693FC5FE667CE40205C322D1BE3B",
    "A2B31D28905E2DEF926362822BC412B12ABF6942B73B72A32D46ED2ABB7ACCFA",
    "AB4014846DF818A4B43D6B1686D0DE0644FE711577C5AB6F0B2A21CCEE280140",
    "3383784E82A8BA45F4DD5EF4EE90A1B2D3B4571317DBAC37B859836ADDE644C1",
    ... (up to 256 ledger hashes) ...
  ],
  "index": "B4979A36CDC7F3D3D5C31A4EAE2AC7D7209DDA877588B9AFC66799692AB0D66B"
}
```

`LedgerHashes` オブジェクトには次のフィールドがあります。

| 名前                 | JSON型          | \[内部型]\[] | 必須? | 説明                                                                                                                                                                                                                                                                                                                                                                                              |
| -------------------- | ---------------- | ------------ | ----- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `FirstLedgerSequence` | Number           | UInt32       | はい  | **非推奨**。使用しないでください。（本番Xahauの「最近のハッシュ」オブジェクトでは、以前の `rippled` ソフトウェアの結果としてこのフィールドの値が `2` になっています。その値は「最近のハッシュ」オブジェクトが更新されるたびに引き継がれます。新しい「過去履歴」オブジェクトにはこのフィールドがなく、より新しい `rippled` バージョンで開始された並列ネットワークの「最近のハッシュ」オブジェクトにもありません。） |
| `Flags`               | Number           | UInt32       | はい  | このオブジェクトで有効になっている真偽値フラグのビットマップです。現在、プロトコルは `LedgerHashes` オブジェクトに対するフラグを定義していません。値は常に `0` です。                                                                                                                                                                                                                          |
| `Hashes`              | Array of Strings | Vector256    | はい  | 最大256個のレジャーハッシュを含む配列です。内容は、この `LedgerHashes` オブジェクトのサブタイプによって異なります。                                                                                                                                                                                                                                                                          |
| `LastLedgerSequence`  | Number           | UInt32       | はい  | このオブジェクトの `Hashes` 配列内にある最後の項目の \[レジャーインデックス]\[]です。                                                                                                                                                                                                                                                                                                         |
| `LedgerEntryType`     | String           | UInt16       | はい  | `LedgerHashes` という文字列に対応付けられる値 `0x0068` は、このオブジェクトがレジャーハッシュの一覧であることを示します。                                                                                                                                                                                                                                                                    |

### 最新履歴のLedgerHashes

ジェネシスレジャーの後にある各レジャーには、「最近の履歴」サブタイプの `LedgerHashes` オブジェクトがちょうど1つあります。このオブジェクトは、最新の256レジャーバージョン（またはレジャー履歴全体が256レジャー未満ならそれより少ない数）の識別用ハッシュを `Hashes` 配列に含みます。新しいレジャーがクローズされるたびに、そのクローズ処理の一部として、そこから派生した前のレジャーバージョンのハッシュ（このレジャーバージョンの_親レジャー_とも呼ばれます）で「最近の履歴」オブジェクトが更新されます。ハッシュが256個を超えると、最も古いものが削除されます。

あるレジャーの「最近の履歴」`LedgerHashes` オブジェクトを使うと、そのレジャーバージョンより前の256レジャーバージョン以内にある任意のレジャーインデックスのハッシュを取得できます。

### 以前の履歴のLedgerHashes

「過去履歴」の `LedgerHashes` エントリは、レジャー全履歴における256レジャーバージョンごとの版（「フラグレジャー」とも呼ばれます）のハッシュをまとめて保持します。フラグレジャーの子がクローズされると、そのフラグレジャーのハッシュが、最も新しい「過去履歴」`LedgerHashes` オブジェクトの `Hashes` 配列に追加されます。`rippled` は65536レジャーごとに新しい `LedgerHashes` オブジェクトを作成し、各「過去履歴」オブジェクトが256個のフラグレジャーのハッシュを持つようにします。

**注:** 最も古い「過去履歴」の `LedgerHashes` オブジェクトには255エントリしかありません。これは、ジェネシスレジャーのレジャーインデックスが0ではなく1だからです。

「過去履歴」の `LedgerHashes` オブジェクトは [skip list](https://en.wikipedia.org/wiki/Skip_list)として機能するため、過去の任意のフラグレジャーのハッシュをそのインデックスから取得できます。そこから、そのフラグレジャーの「最近の履歴」オブジェクトを使って、他の任意のレジャーのハッシュを取得できます。

### LedgerHashesのID形式

`LedgerHashes` オブジェクトIDには、オブジェクトが「最近の履歴」サブタイプか「過去履歴」サブタイプかによって2つの形式があります。

**「最近の履歴」** の `LedgerHashes` オブジェクトのIDは、`LedgerHashes` スペースキー（`0x0073`）の \[SHA-512Half]\[]です。言い換えると、「最近の履歴」のIDは常に `B4979A36CDC7F3D3D5C31A4EAE2AC7D7209DDA877588B9AFC66799692AB0D66B` です。

**「過去履歴」** の `LedgerHashes` オブジェクトのIDは、次の値を順番に連結したものの \[SHA-512Half]\[]です。

* `LedgerHashes` スペースキー（`0x0073`）
*   このオブジェクトの `Hashes` 配列にあるフラグレジャーの32ビット \[レジャーインデックス]\[]を65536で割った値。

    **ヒント:** 65536で割ると、最上位16ビットだけが残ります。これは「過去履歴」オブジェクトに一覧されるすべてのフラグレジャーと、そのレジャーだけで共通です。この事実を使うと、任意のフラグレジャーのハッシュを含む `LedgerHashes` オブジェクトを見つけられます。
