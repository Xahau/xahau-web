---
title: レジャーデータ
---
XRP Ledgerの各レジャーバージョンは3つの部分で構成されます。

* **レジャーヘッダー**:このレジャーバージョン自体のメタデータ。
* **トランザクションセット**:このレジャーバージョンを作成するために実行されたすべてのトランザクション。
* **ステートデータ**:このレジャーバージョン時点のアカウント、設定、残高を表すオブジェクトの完全な記録。（これは「アカウント状態」とも呼ばれます。）

### ステートデータ

各レジャーバージョンのステートデータは、**レジャーオブジェクト** の集合です。これは_レジャーエントリ_とも呼ばれ、ある時点におけるすべての設定、残高、関係をまとめて表します。ステートデータ内でオブジェクトを保存または取得するために、プロトコルはそのオブジェクト固有の [**Ledger Object ID**](/ja/docs/protocol-reference/ledger-data/ledger-object-ids)を使用します。

ピアプロトコルでは、レジャーオブジェクトは [正規バイナリ形式](/ja/docs/protocol-reference/binary-format)を持ちます。`xahaud` APIでは、レジャーオブジェクトはJSONオブジェクトとして表されます。

レジャーオブジェクトのデータフィールドはオブジェクトの種類によって異なります。Xahau Networkは次の種類をサポートしています。

* [レジャーヘッダー](/ja/docs/protocol-reference/ledger-data/ledger-header)
* [レジャーエントリ共通フィールド](/ja/docs/protocol-reference/ledger-data/ledger-object-ids)
* [レジャーエントリタイプ](/ja/docs/protocol-reference/ledger-data/ledger-objects-types/)

### 拡張レジャー

Xahauは拡張レジャーをサポートしており、従来の上限を超えるより大きなレジャーサイズを可能にします。拡張レジャーにより、ネットワークはパフォーマンスとコンセンサスの安定性を維持しながら、より多いトランザクション量とより複雑なステートデータを扱えます。

拡張レジャーが有効な場合、レジャーは次の内容を収容できます。
- 1レジャーあたりのトランザクション数の増加
-より大きなステートデータセット
- Hook実行の複雑性の増加

拡張レジャーは、トランザクション量とステートサイズに基づいてネットワークが必要と判断したときに自動的に使用されます。ノードが拡張レジャーに参加するための特別な設定は不要です。
