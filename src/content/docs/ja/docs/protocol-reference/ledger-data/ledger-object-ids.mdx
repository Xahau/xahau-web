---
title: レジャーオブジェクトID
---
[\[出典\]](https://github.com/ripple/rippled/blob/master/src/ripple/protocol/impl/Indexes.cpp)

レジャーのステートデータに含まれる各オブジェクトには、一意のIDがあります。このIDは、オブジェクトの重要な内容と [名前空間識別子](https://github.com/ripple/rippled/blob/master/src/ripple/protocol/LedgerFormats.h#L99)をまとめてハッシュすることで導出されます。どの名前空間識別子を使い、どの内容をハッシュに含めるかは、レジャーオブジェクトの種類によって決まります。これにより、すべてのIDが一意になります。ハッシュの計算には、`xahaud`がSHA-512を使い、その結果を最初の256ビットに切り詰めます。このアルゴリズムは、通称 **SHA-512Half** と呼ばれ、SHA-256と同等の安全性を持ちながら、64ビットプロセッサーではより高速に動作します。

通常、レジャーオブジェクトのIDは、オブジェクトの内容と同じ階層にあるJSONの`index`フィールドとして返されます。トランザクションメタデータでは、レジャーオブジェクトのIDはJSONで`LedgerIndex`フィールドです。

**ヒント:** レジャー内のオブジェクトの`index`または`LedgerIndex`フィールドはレジャーオブジェクトIDです。これは \[ledger index]\[]とは異なります。

### フラグ <a href="#flags" id="flags"></a>

フラグはオン/オフ設定で、ビットごとのOR演算を使って1つの数値にまとめられるバイナリ値として表されます。レジャーエントリにおけるフラグのビット値は、トランザクション内でそれらのフラグを有効化または無効化するために使う値とは異なります。レジャー状態フラグの名前は **`lsf`** で始まります。

`flags`フィールドの取りうる値は、レジャーエントリの種類によって異なります。レジャーエントリの種類によっては、フラグが定義されていないものもあります。その場合、`Flags`フィールドの値は常に`0`です。
