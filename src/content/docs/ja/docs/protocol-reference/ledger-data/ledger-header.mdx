title: レジャーヘッダー
---
[\[出典\]](https://github.com/ripple/rippled/blob/master/src/ripple/ledger/ReadView.h#L71)

各レジャーバージョンには、その内容を説明する一意のヘッダーがあります。レジャーのヘッダー情報は \[ledger method]\[]で参照できます。レジャーヘッダーの内容は次のとおりです。

| フィールド              | JSON型   | \[内部型]\[] | 説明                                                                                                                                                                                                                                                             |
| ----------------------- | --------- | ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ledger_index`          | String    | UInt32              | レジャーの \[ledger index]\[Ledger Index]です。一部のAPIメソッドではこれを引用符付き整数として表示し、ほかのものではネイティブのJSON数値として表示します。                                                                                           |
| `ledger_hash`           | String    | Hash256             | このレジャーバージョンの \[SHA-512Half]\[]です。これは、このレジャーとそのすべての内容の一意の識別子として機能します。                                                                                                                                          |
| `account_hash`          | String    | Hash256             | このレジャーの状態ツリー情報の \[SHA-512Half]\[]です。                                                                                                                                                                                                       |
| `close_time`            | Number    | UInt32              | このレジャーバージョンがクローズしたおおよその時刻です。Ripple Epochの2000-01-01 00:00:00からの秒数で表されます。この値は`close_time_resolution`に基づいて丸められます。                                                                                |
| `closed`                | Boolean   | Boolean             | `true`の場合、このレジャーバージョンは新しいトランザクションを受け付けません。（ただし、このレジャーバージョンが検証されていない限り、異なるトランザクション集合を持つ別のレジャーバージョンに置き換わる可能性があります。）                         |
| `parent_hash`           | String    | Hash256             | このレジャーの直前の親である、前のレジャーバージョンの`ledger_hash`値です。前のレジャーインデックスに複数のバージョンがある場合、どのバージョンからレジャーが派生したかを示します。                                                                           |
| `total_coins`           | String    | UInt64              | レジャー内のアカウントが保有する \[drops of XAH]\[]の合計数です。これには、トランザクション手数料によって破棄されたXAHは含まれません。実際に流通しているXAH量は、鍵が誰にも分からない「ブラックホール」アカウントがあるため、これより少なくなります。 |
| `transaction_hash`      | String    | Hash256             | このレジャーに含まれるトランザクションの \[SHA-512Half]\[]です。                                                                                                                                                                                            |
| `close_time_resolution` | Number    | Uint8               | `close_time`が丸められる最大秒数を示す、\[2,120]の範囲の整数です。                                                                                                                                                                                          |
| `closeFlags`            | (Omitted) | UInt8               | このレジャーのクローズに関するフラグのビットマップです。                                                                                                                                                                                                       |

### レジャーインデックス

### クローズフラグ

`closeFlags`に定義されているフラグは1つだけです。**`sLCF_NoConsensusTime`**（値 `1`）です。このフラグが有効な場合、バリデーターごとにレジャーのクローズ時刻は異なっていたものの、それ以外は同じレジャーを構築していたため、クローズ時刻については「異なることで合意する」形でコンセンサスを宣言したことを意味します。この場合、レジャーの公式な`close_time`値は親レジャーより1秒後になります。

`closeFlags`フィールドはレジャーのJSON表現には含まれませんが、レジャーのバイナリ表現には含まれ、レジャーのハッシュを決定するフィールドの1つです。
