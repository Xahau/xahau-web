ノードを実行する際は、そのノードがテストネットで動作するのかメインネットで動作するのかに応じて設定を構成する必要があります。1つのノードをあるネットワークから別のネットワークへ切り替える場合は、サーバーのID資格情報を含む`wallet.db`を除き、データベースファイルを消去する必要があります。

xahaudをインストールする最も一般的な方法は、ローカルで`systemd`を使う方法か、Dockerのようなコンテナ内で実行する方法です。

<Aside type="tip">
このページのインストール手順は、[xahaud GitHubリポジトリ](https://github.com/xahau/xahaud/blob/dev/docs/build/install.md)にあるインストール手順を補足したものです。
</Aside>

## Linuxのインストール手順 <a id="linux-install-process" />
xahaudをDockerでインストールする場合でもシステム全体にインストールする場合でも、Linuxでは全体的な手順はほぼ同じです。
以下のセクション [Dockerコンテナ](#docker-container-install-script)と [ローカルインストール](#local-install-script)では、次の手順を自動化するbashスクリプトを利用します。

1. xahaudのバイナリを入手する:
    - [https://build.xahau.tech](https://build.xahau.tech)からコンパイル済みバイナリイメージをダウンロードする
    - [ソースからビルドする](/ja/docs/infrastructure/build-xahaud/)
2. 設定ファイルを入手する:
    - [xahaud.cfg](https://raw.githubusercontent.com/Xahau/xahaud/refs/heads/dev/cfg/xahaud-example.cfg)
    - [validators-xahau.txt](https://raw.githubusercontent.com/Xahau/xahaud/refs/heads/dev/cfg/validators-example.txt)
3. 必要に応じて、バイナリや関連ファイルの所有者にできる`xahaud`ユーザーとグループを作成する:
    - `sudo groupadd --system xahaud`
    - `sudo useradd --system --gid xahaud --no-create-home xahaud`
4. 保存先のディレクトリ構成を決めて作成する:
    - xahaudバイナリ。既定は通常 `/opt/xahaud/bin/xahaud`
    - 2つのxahaud設定ファイル`xahaud.cfg`と `validators-xahau.txt`。既定は`/etc/xahaud/`ですが、`/opt/xahaud/etc/`がよく使われます。この2つの場所はシンボリックリンクで結べます。
    - xahaudデータベース。既定は多くの場合 `/opt/xahaud/db/`
    -ログファイルの保存先。例: `/var/log/xahaud/default.log`
5. xahaudバイナリと設定ファイルを、ディレクトリ構成の所定位置へ移動する。
6. 設定内のパスがディレクトリ構成と一致するように`xahaud.cfg`を編集する。
7. 必要に応じて所有権と権限を変更する。xahaudをvalidatorとして実行する場合、`xahaud.cfg`にはvalidation tokenが含まれるため、アクセスを制限することが重要です。
    - `chown -R xahaud:xahaud /opt/xahaud /var/log/xahaud`
    - `chmod -R 750 /opt/xahaud /var/log/xahaud`
8. xahaudをデーモンとして実行するためのsystemdサービスファイルを作成する (例は [下記](#example-systemd-service-file))。
    -ファイルの場所: `/etc/systemd/system/xahaud.service`
    -ファイルを作成または編集した後に実行する: `systemctl daemon-reload`
    -システム起動時にxahaudを自動実行する: `systemctl enable --now xahaud`
    -必要であれば、xahaudはsystemdを使わずに実行できます: `/path/to/xahaud --conf=/path/to/xahaud.cfg`
9. フルパスを指定せずにxahaudコマンドを実行できるラッパーを作成する。
    -通常の保存先: `/usr/local/bin/xahaud`
	- Contents might be: `exec ${XAHAUD_DIR}xahaud --conf=${CONF_DIR}xahaud.cfg "$@"`

### systemdサービスファイルの例 <a id="example-systemd-service-file" />
以下のサービスファイルは、ユーザーが選択したディレクトリ構成と所有権/権限に合わせて変更してください。
```
[Unit]
Description=Xahaud Daemon
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/path/to/xahaud --silent --conf=/path/to/xahaud.cfg
Restart=on-failure
User=xahaud
Group=xahaud
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

### Linuxインストールスクリプトの例 <a id="linux-install-script-example" />
xahaudをインストールする基本的なbashスクリプトです。このスクリプトは、前述の手順に従っています。
```
### VARIABLES ###

#### Directory structure ####
XAHAUD_DIR="/opt/xahaud/bin/"	# Path where xahaud binary will be stored
CONF_DIR="/opt/xahaud/etc/"		# Path where the xahaud.cfg and validators-xahau.txt will be stored
DB_DIR="/opt/xahaud/db/"		# Path where xahaud will store databases
LOG_DIR="/var/log/xahaud/"		# Path where logfile(s) will be stored.

#### Ownership ####
XAHAUD_USER="xahaud"			# User that owns the xahaud process.

#### Download links ####
BINARY_URL="https://build.xahau.tech/2025.9.8-HEAD%2B2194"											# URL to the xahaud binary that will be downloaded.
CFG_URL="https://raw.githubusercontent.com/Xahau/xahaud/refs/heads/dev/cfg/xahaud-example.cfg"		# This will be renamed to 'xahaud.cfg'
VAL_URL="https://raw.githubusercontent.com/Xahau/xahaud/refs/heads/dev/cfg/validators-example.txt"	# This will be renamed to 'validators-xahau.txt'



########## SCRIPT BEGINS HERE. DO NOT ADJUST VARIABLES BELOW THIS LINE ##########

### CREATE DIRECTORIES ####
echo "Checking directory structure."
sudo mkdir -p ${XAHAUD_DIR} ${CONF_DIR} ${DB_DIR} ${LOG_DIR}

### CREATE xahaud GROUP AND USER ###
echo "Checking for user and group."
if ! getent group ${XAHAUD_USER} > /dev/null; then
	sudo groupadd --system ${XAHAUD_USER}
fi

if ! getent passwd ${XAHAUD_USER} > /dev/null; then
	sudo useradd --system --gid ${XAHAUD_USER} --no-create-home ${XAHAUD_USER}
fi

### DOWNLOAD CONFIGURATION FILES ###
echo "Downloading files."
if [[ ! -f "${CONF_DIR}xahaud.cfg" ]]; then
	curl -fsSL ${CFG_URL} -o ${CONF_DIR}xahaud.cfg
fi
if [[ ! -f "${CONF_DIR}validators-xahau.txt" ]]; then
	curl -fsSL ${VAL_URL} -o ${CONF_DIR}validators-xahau.txt
fi

### DOWNLOAD xahaud ###
if [[ ! -f "${XAHAUD_DIR}xahaud" ]]; then
	curl -fsSL ${BINARY_URL} -o ${XAHAUD_DIR}xahaud
elif [[ -f "${XAHAUD_DIR}xahaud" ]]; then
	echo "Existing xahaud binary found. It will be renamed to 'xahaud.old'."
	mv ${XAHAUD_DIR}xahaud ${XAHAUD_DIR}xahaud.old
	curl -fsSL ${BINARY_URL} -o ${XAHAUD_DIR}xahaud
fi

### CHANGE OWNERSHIP AND PERMISSIONS ###
echo "Checking ownership and permissions."
sudo chown -R ${XAHAUD_USER}:${XAHAUD_USER} ${XAHAUD_DIR} ${CONF_DIR} ${DB_DIR} ${LOG_DIR}
sudo chmod -R 0750 ${XAHAUD_DIR} ${CONF_DIR} ${DB_DIR} ${LOG_DIR}

### Install systemd SERVICE FILE ###
if [[ ! -f "/etc/systemd/system/xahaud.service" ]]; then
echo "Installing system service file."
sudo cat > /etc/systemd/system/xahaud.service <<EOF
[Unit]
Description=Xahaud Daemon
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=${XAHAUD_DIR}xahaud --silent --conf=${CONF_DIR}xahaud.cfg
Restart=on-failure
User=${XAHAUD_USER}
Group=${XAHAUD_USER}
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
EOF
fi

### RUN COMMANDS AS 'xahaud' WITHOUT SPECIFYING FULL PATHS ###
if [[ ! -f "/usr/local/bin/xahaud" ]]; then
echo "Creating xahaud wrapper."
sudo cat > /usr/local/bin/xahaud <<EOF
#!/usr/bin/env bash
set -euo pipefail

exec ${XAHAUD_DIR}xahaud --conf=${CONF_DIR}xahaud.cfg "\$@"
EOF
fi

sudo chmod 0755 /usr/local/bin/xahaud

### ENABLE AND START xahaud ###
sudo systemctl daemon-reload
sudo systemctl stop xahaud
sudo systemctl enable --now xahaud
sudo systemctl status xahaud
echo "The install script completed successfully."
```

## Dockerコンテナのインストールスクリプト <a id="docker-container-install-script" />

xahaudをDockerコンテナで実行するには:

1. 以下の表にあるURLを使って、適切な (メインネットまたはテストネット) Xahau Docker GitHubリポジトリをクローンする: <code>git clone [repository-url]</code>
2. リポジトリのディレクトリへ移動する: <code>cd [repository-name]</code>
3. Run: <code>./build</code>
4. Run: <code>./up</code>
5. コマンドや追加情報はDockerリポジトリのREADME.mdにあります。設定ファイルとデータベースはDockerディレクトリ内の`store`ディレクトリにあります。

## ローカルインストールスクリプト <a id="local-install-script" />
Xahau Dockerリポジトリ (メインネットとテストネット)には、それぞれxahaudをDocker外で実行できるようにローカルインストール手順を自動化するスクリプトがあります。これらのスクリプトは [Linuxのインストール手順](#linux-install-process)に似ており、systemdファイル、`xahaud`ユーザー、既定の設定ファイルを作成します。これらのスクリプトのURLはページ下部の表にあります。ローカルにインストールするには:

1. インストールスクリプトをダウンロードして実行する: <code>curl -sL [https://link-to-script] | bash</code>
2. 設定ファイルを編集する: <code>/opt/xahahud/etc/xahaud.cfg</code>
3. 信頼済みvalidatorファイルの内容を確認し、必要に応じて編集する: <code>/opt/xahaud/etc/validators-xahau.txt</code>
4. xahaudを起動して自動実行を有効にする: <code>systemctl enable --now xahaud</code>
5. xahaudが実行中であることを確認する: <code>/opt/xahaud/bin/xahaud server_info</code>
