## ソースコードからソフトウェアをビルドする
開発者は通常、人間が読みやすいプログラミング言語でソフトウェアを書きます。しかし、コンピューターがそのまま人間向けコードを実行するのは効率的ではありません。そのため、人が読めるコードを、コンピューターが高速に読み取り解釈できるバイトコードへ「コンパイル」する必要があります。

この過程をさらに複雑にするのが「依存関係」です。これは、あるソフトウェアが別のソフトウェアを必要とすることを意味します。依存関係を最終成果物に含めるには、コンパイル前にコードへ「リンク」する必要があります。元のソフトウェアの依存関係自体がさらに別の依存関係を持つこともあり、このリンク作業は非常に重くなることがあります。開発者は Conan のようなツールを使って、依存関係の探索、設定、リンクを簡素化します。

依存関係を満たすこと、ソフトウェアをマシン向け形式へコンパイルすること、そして最終成果物に対してテストを実行することを含む一連の工程全体を「ビルド」と呼びます。

`xahaud` は [GitHub 上でオープンソースとして公開](https://github.com/Xahau/xahaud)されているため、誰でもコードを閲覧、監査、変更、ビルド、実行できます。ソースコードからビルドすれば、実行しているものが公開されたソースコードに基づいていることを確実に確認できますし、完全な監査履歴も追えます。そのため、最大限の安全性を重視するバリデーター運用者はソースからのビルドを検討するべきです。

## 依存関係

互換性を保つため、ビルド工程で使うソフトウェアのバージョンには注意が必要です。`xahaud` は C++20 で書かれています。

現在の `xahaud` ビルド要件は以下の通りです。


| Dependency  | Version   | 
| :---------- | :-------- | 
| Python3     | >= 3.7    |
| GCC / G++   | >= 14.x   |
| CMake       | >= 3.16   |
| Conan2      | 2.x       |
| openssl     | 1.1.1     |
| boost       | 1.86.0    | 
| SQLite3     | 3.42.0    |
| Snappy*     | 1.1.10    |
| SOCI*       | 4.0.3     |
| WasmEdge*   | 0.11.2    |

\* Xahau/xahaud GitHub リポジトリに含まれています。
