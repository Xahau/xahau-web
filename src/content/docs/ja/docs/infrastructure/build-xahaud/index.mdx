xahaudのようなソフトウェアを公開ソースコードからビルドするのは、深い知識と経験を要する複雑な作業です。そのため、環境 (Linux、MacOS、Windowsなど)に応じて、開発者が採りうる手法は複数あります。以下のドキュメント章の手順は、[xahaud GitHubリポジトリ](https://github.com/Xahau/xahaud/blob/dev/BUILD.md)にある手順を簡略化し、補足することを目的としています。開発者は、特定の作業を行うため、または特定のオペレーティングシステムとの互換性を確保するために、これらの手順から逸脱する必要がある場合があります。 

以下の記事では、さまざまなオペレーティングシステム環境でビルド環境を構築し、xahaudをビルドする手順を詳しく説明します。

<Aside type="danger">
ソースコードからソフトウェアをビルドするのは、高度な技術知識と経験を必要とする複雑な作業です。オペレーティングシステムレベルで、簡単には元に戻せない変更を加える可能性があります。ビルド作業は自己責任で行ってください。このドキュメントは、厳密な正確性を保証するものではなく、ガイドとして設計されています。
</Aside>

## ソースからソフトウェアをビルドする
開発者は、人間が読みやすいように設計されたプログラミング言語でソフトウェアを書きます。しかし、その人間可読なコードをコンピューターがそのまま実行するのは効率的ではありません。そのため、ソフトウェアを人間可読コードから、コンピューターがすばやく読み取り解釈できるバイトコードへ「コンパイル」する必要があります。

このプロセスをさらに複雑にするのが、開発者が他の開発者によって書かれたソフトウェアに依存することです。これらは、あるソフトウェアパッケージが別のものに依存 (require)するため、「dependencies」と呼ばれます。これらのdependenciesを最終製品に含めるには、コードをコンパイルする前にコードへ「link」する必要があります。linkのプロセスは非常に負荷が高い場合があります。元のソフトウェアパッケージのdependency自体にもdependencyがあり、それがさらに別のdependencyを持つことがあるからです。開発者はConanのようなツールを使って、dependenciesの探索、設定、linkを簡略化します。

dependenciesを満たし、ソフトウェアをマシン向けに最適化された形式へコンパイルし、最終成果物に対してテストを実行する一連の作業全体を「ビルド」と呼びます。

xahaudは [GitHub上でオープンソースソフトウェアとして公開されています](https://github.com/Xahau/xahaud)ので、誰でもコードを閲覧、監査、修正、ビルド、実行できます。ソースコードからビルドすると、実行している製品が公開されているソースコードに基づいていることを確認でき、その完全な監査証跡も残ります。そのため、最大限のセキュリティを優先するvalidator運用者は、ソースからのビルドを検討すべきです。

## 依存関係

互換性のため、ビルドプロセスで使うソフトウェアの特定バージョンに注意することが重要です。xahaudはC++20で書かれています。

現在のxahaudビルド要件は次のとおりです。


| 依存関係  | バージョン   | 
| :---------- | :-------- | 
| Python3     | >= 3.7    |
| GCC / G++   | >= 14.x   |
| CMake       | >= 3.16   |
| Conan2      | 2.x       |
| openssl     | 1.1.1     |
| boost       | 1.86.0    | 
| SQLite3     | 3.42.0    |
| Snappy*     | 1.1.10    |
| SOCI*       | 4.0.3     |
| WasmEdge*   | 0.11.2    |

\* Xahau/xahaud GitHubリポジトリに含まれています。
