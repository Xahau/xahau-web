---
title: レスポンスフォーマットガイド
---
レスポンスは、リクエストがWebSocket、JSON-RPC、またはコマンドラインインターフェースのいずれを介して行われるかに基づいて、構造が異なります。コマンドラインインターフェースは内部的にJSON-RPCを使用しているため、JSON-RPCとコマンドラインインターフェースは同じ形式を共有します。

### フィールド

| フィールド          | 型         | 説明                                                                                                                                                              |
| ------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| id            | （多様）     | （WebSocket用）元のリクエストのID。                                                                                                                                         |
| status        | String   | （WebSocket用）リクエストが正常に受信・処理されたことを示す`success`。                                                                                                              |
| result.status | String   | （JSON-RPCおよびコマンドライン用）リクエストが正常に処理されたことを示す`success`。                                                                                                          |
| type          | String   | （WebSocket用）API リクエストへの直接返信には`response`値が使用されます。非同期通知には`ledgerClosed`や`transaction`などの他の値が使用されます。                                                           |
| result        | Object   | クエリ結果を含み、コマンドによって内容が異なります。                                                                                                                                    |
| warning       | String   | _（オプション）_ 存在する場合、値は`load`であり、クライアントがサーバーが切断する可能性のあるレート制限閾値に近づいていることを示します。                                                                                  |
| warnings      | Array    | _（オプション）_ 重要なサーバー警告を含む**Warningオブジェクト**のリスト。詳細はAPI警告を参照してください。                                                                                               |
| forwarded     | Boolean  | _（オプション）_ `true`はリクエストがレポートモードサーバーからP2Pサーバーに転送されてリクエストを履行したことを示します。デフォルトは`false`。                                                                          |

API警告

レスポンスに`warnings`配列が含まれている場合、各エントリはサーバーからの特定の警告を表します。各**Warningオブジェクト**には次のフィールドが含まれます：

| フィールド    | 型       | 説明                                                                                                                                                              |
| ------- | ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| id      | Number | この警告メッセージを識別するユニークな数値コード。                                                                                                                                       |
| message | String | 警告の人間が読める説明。このフィールドの内容に依存するコードは書かないでください；代わりに警告を解釈するために`id`（および利用可能な場合は`details`）を使用してください。 |
| details | Object | _（オプション）_ 警告に関する追加コンテキスト。内容は警告タイプによって異なります。                                                                                                                    |
