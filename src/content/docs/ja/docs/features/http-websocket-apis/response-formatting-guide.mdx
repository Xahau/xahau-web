---
title: レスポンスフォーマットガイド
---
レスポンスの構造は、リクエストがWebSocket、JSON-RPC、コマンドラインインターフェイスのどれを通じて行われたかによって異なります。コマンドラインインターフェイスは内部でJSON-RPCを使用するため、JSON-RPCとコマンドラインインターフェイスは同じ形式を共有します。

### フィールド

| フィールド    | タイプ   | 説明                                                                                                                                                                   |
| ------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| id            | (Varies) | (WebSocket用)元のリクエストのID。                                                                                                                                      |
| status        | String   | (WebSocket用)リクエストが正しく受信され処理された場合に`success`を示します。                                                                                          |
| result.status | String   | (JSON-RPCおよびコマンドライン用)リクエストが正常に処理された場合に`success`を示します。                                                                                |
| type          | String   | (WebSocket用) APIリクエストへの直接返信には値`response`が使用されます。非同期通知では`ledgerClosed`や`transaction`など、他の値が使用されます。                         |
| result        | Object   | クエリ結果を含み、内容はコマンドによって異なります。                                                                                                                   |
| warning       | String   | _(任意)_ 存在する場合、値は`load`で、サーバーが切断する可能性のあるレート制限しきい値にクライアントが近づいていることを示します。                                      |
| warnings      | Array    | _(任意)_ 重要なサーバー警告を含む**Warning Object**のリスト。詳細についてはAPI Warningsを参照してください。                                                          |
| forwarded     | Boolean  | _(任意)_ `true`は、リクエストを満たすためにReporting ModeサーバーからP2Pサーバーへリクエストが転送されたことを示します。デフォルトは`false`です。                      |

API警告

レスポンスに`warnings`配列が含まれる場合、各エントリはサーバーからの特定の警告を表します。各**Warningオブジェクト**には次のフィールドが含まれます。



| フィールド | タイプ | 説明                                                                                                                                                                                   |
| ------- | ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| id      | Number | この警告メッセージを識別する一意の数値コード。                                                                                                                                         |
| message | String | 警告の人間が読める説明。このフィールドの内容に依存するコードを書くことは避け、代わりに`id` (利用可能な場合は`details`も)を使用して警告を解釈してください。                           |
| details | Object | _(任意)_ 警告に関する追加コンテキスト。内容は警告タイプによって異なります。                                                                                                           |
