## マーカー

一部のメソッドは、単一のレスポンスに効率的に収まらないデータを返します。結果がレスポンスの制限を超えた場合、`marker`フィールドがレスポンスに含まれます。このフィールドを使用して、後続のリクエストを通じて追加のデータページを取得できます。データの取得を続けるには、前のレスポンスの`marker`値を次のリクエストに含めてください。レスポンスに`marker`が含まれていない場合、データセットの末尾に達したことを意味します。

`marker`フィールドの形式は意図的に未指定です。各サーバーは必要に応じて`marker`を定義でき、文字列、ネストされたオブジェクト、または別のタイプである可能性があります。`marker`形式はサーバー間、さらには同じサーバーのメソッド間でも異なる場合があります。各`marker`は一時的であり、約10分後に無効になる可能性があります。

## レート制限

`xahaud`サーバーは、過度なリクエストを防ぐためにパブリックAPIを使用するAPIクライアントにレート制限を適用します。レート制限はクライアントのIPアドレスに基づいて適用されます。つまり、[ネットワークアドレス変換（NAT）](https://en.wikipedia.org/wiki/Network_address_translation)を共有する複数のクライアントは、パブリックIPに関連付けられた同じレート制限を共有します。

クライアントがレート制限に近づいている場合、サーバーはAPIレスポンスのトップレベルに`"warning": "load"`フィールドを含めます。この警告はすべてのレスポンスに表示されるわけではありませんが、サーバーがクライアントを切断する前に数回送信される場合があります。管理者として接続されたクライアントはレート制限から免除されます。

クライアントがレート制限を超えると、サーバーはクライアントを切断し、そのIPアドレスからのさらなるリクエストを一時的にブロックします。WebSocketとJSON-RPC APIは以下で説明するように、切断を異なる方法で処理します。
