## マーカー

一部のメソッドは、1つのレスポンスに効率よく収まる量を超えるデータを返します。結果がレスポンス制限を超える場合、レスポンスには`marker`フィールドが含まれます。このフィールドにより、後続のリクエストで追加ページのデータを取得できます。データ取得を続けるには、前回のレスポンスの`marker`値を次のリクエストに含めます。レスポンスに`marker`が含まれていない場合は、データセットの末尾に到達したことを意味します。

`marker`フィールドの形式は意図的に未指定です。各サーバーは必要に応じて`marker`を定義できるため、文字列、ネストされたオブジェクト、または別の型である可能性があります。`marker`の形式はサーバー間で異なる場合があり、同じサーバー上のメソッド間でも異なることがあります。各`marker`は一時的なもので、およそ10分後に無効になる可能性があります。

## レート制限

`xahaud`サーバーは、過剰なリクエストを防ぐため、パブリックAPIを使用するAPIクライアントにレート制限を適用します。レート制限はクライアントのIPアドレスに基づいて適用されます。つまり、[network address translation (NAT)](https://en.wikipedia.org/wiki/Network_address_translation)を共有する複数のクライアントは、そのパブリックIPに関連付けられた同じレート制限を共有します。

クライアントがレート制限に近づくと、サーバーはAPIレスポンスのトップレベルに`"warning": "load"`フィールドを含めます。この警告はすべてのレスポンスに表示されるわけではありませんが、サーバーがクライアントを切断する前に複数回送信されることがあります。adminとして接続しているクライアントはレート制限の対象外です。

クライアントがレート制限を超えた場合、サーバーはクライアントを切断し、そのIPアドレスからの追加リクエストを一時的にブロックします。WebSocket APIとJSON-RPC APIでは、以下に説明するように切断の扱いが異なります。
