## 主な違い

1. Hooks対応ネットワークでは、ネットワーク定義を動的に取得できます。これにより、クライアントは利用可能なトランザクションタイプ、レジャーオブジェクト、プロパティ、値タイプに適応できます。正しく実装されていれば、ネットワークがトランザクション/オブジェクトタイプ/プロパティを追加しても、署名ライブラリやエンコードライブラリを更新する必要はありません。<mark style="color:blue;">**以下のライブラリはこれを実装しており、この処理を代わりに扱います。**</mark>
2. Hooks対応ネットワークでは、別のチェーンでのトランザクションリプレイを防ぐため、すべてのトランザクションに**NetworkID**が必要です。**NetworkID**は、`server_info` RPCコマンドの`network_id`フィールドにも返されます (例: Hooks V3テストネットでは**`21338`**)
3. Hooks対応ネットワーク上のトランザクションでは、送信元アカウントからの送信時および宛先アカウントでの受信時に実行されるHooksに応じて、別のアカウントへトランザクションを届けるためにより高い手数料が必要になる場合があります。Hooks実行を満たす妥当な手数料は、トランザクションを`tx_blob`として提供しながら`fee`コマンドを発行することで、ノードから動的に取得できます。<mark style="color:blue;">**以下のライブラリはこれを実装しており、この処理を代わりに扱います。**</mark>

## JavaScript/Typescript

[**npm package `xrpl-accountlib`**](https://www.npmjs.com/package/xrpl-accountlib)は、実行時にネットワーク定義を取得する完全な動的ネットワーク機能サポートを提供するため、Hooks対応ネットワーク向けのトランザクションに署名できます。

[**npm package `xrpl-client`**](https://www.npmjs.com/package/xrpl-client)は`xrpl-accountlib`と適切に統合され (依存関係として含まれます)、前述のネットワーク定義とアカウント値を動的に取得して、トランザクション送信を支援します。

```javascript
    Flags: 1,

  }
  }],
  ...networkInfo.txValues,
  // ^^ This adds autmatically fetched values for you:
  // Sequence, Account, LastLedgerSequence,
  // Fee (Hooks enabled: autodetect (from ledger))
}

/**
 * Note: the code above and `signAndSubmit` results in automatically
 * fetching and setting a fee for you. If you want to check the fee
 * for min/max/..., get your own fee (string in drops) using:
 *    utils.networkTxFee(wss, tx)
 * 
 * e.g.

 */

const submitted = await signAndSubmit(tx, wss, account)

console.log(submitted)
```
