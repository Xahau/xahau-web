### Hooksとは何か？ <a href="#what-are-hooks" id="what-are-hooks"></a>

Hooksにより、Xahau内でカスタマイズされたロジックと自動化を作成でき、トランザクションをよりスマートで便利にします。これらの小さく効率的なモジュールは、レジャー上の特定のイベントに対するカスタムトリガーの作成などのカスタムオンレジャー機能を追加します。

これらのトリガーは、指定されたイベントへの応答として、オンレジャーアクションを送信したり他のアクションを実行したりするために使用できます。HooksはHooksは現在[Xahauネットワーク](https://xahau.network/)で利用できます。

> _Hooksの理論的なコンセプトの詳細については、_[_コンセプト_](/docs/hooks/concepts/introduction)_セクションを参照してください。_

> _CでのHooksのコーディングとその関数については、_[_Hooks C-Functions_](/docs/hooks/functions/overview/hook-api-conventions/)_を参照してください。_

> _[Hooks Builderサイト](https://builder.xahau.network/)では、ブラウザ上でテストネットに独自のHooksを開発、テスト、デバッグ、デプロイできます。用意されたサンプルを利用することも、ゼロから独自のものを構築することもできます。_

> _[Hookstore](https://hookstore.xahau.network/)はHooksの発見、共有、管理のためのプラットフォームを提供します。[Hookstore Console](https://console.hookstore.xahau.network/)を使用してHooksを管理し、エコシステムを探索してください。_

> JSHooksとも呼ばれるJavaScriptでHooksを書けるようにする開発が進行中です。現在は[JSHooks](https://github.com/Xahau/jshooks-alpha)リポジトリを使用してテストできます。

### なぜHooksが重要なのか？ <a href="#why-are-hooks-a-big-deal" id="why-are-hooks-a-big-deal"></a>

端的に言えば、HooksはXahauに堅牢なスマートコントラクト機能を追加し、特定のニーズと要件に合ったカスタム機能のアプリケーションを構築・展開できるようにします。

Hooksはビジネスロジックとスマートコントラクトパラダイムの幅広いスペクトルを実装するために使用できるため、汎用性の高いプラットフォームを提供します。アカウントにHookが設定されると、以下のことが可能になります：

* アカウントへの/からのトランザクションのブロックまたは許可。
* Hookの内部状態とロジックを変更・追跡してプログラム的な選択を通知。
* アカウントの代わりに新しいトランザクションを自律的に開始。

HooksはCまたは他の好みの言語で書き、WebAssemblyにコンパイルできます。

### 特定のHooksとユースケースの例 <a href="#some-examples-of-specific-hooks" id="some-examples-of-specific-hooks"></a>

Xahauでのスマートコントラクト機能のユニークなアプリケーションをそれぞれ示した具体的な例でHooksの可能性を紹介します：

* **自動貯蓄Hook**: レジャー上の別の貯蓄アカウントに一定額のXAHを転送するようにHookを設定することで、貯蓄を自動化します。これにより、XAHの一部を節約し、日次、週次、月次などの指定された間隔で貯蓄を積み上げることができます。この定期転送メカニズムは、個人ファイナンスアプリケーションやサブスクリプションベースのモデルの開発の基盤となります。
* **カーボンオフセットHook**: 各トランザクションは、NGO（信頼できる非政府組織）が善意のために管理するカーボンオフセットアカウントへの追加の1%転送をトリガーします。この機能は、行われたすべてのトランザクションで環境の持続可能性に貢献するアプリケーションの構築の基盤として使用できます。
* **ファイアウォールHook**: 着信・発信トランザクションをフィルタリングすることで、ファイアウォールHookは既知の詐欺アカウントからの悪意のあるトランザクションや疑わしいメモを含むトランザクションをブロックできます。別のアカウントのHookから更新されたブロックリストを取得することで、ファイアウォールは手動介入なしに詐欺に対する強固な防御を維持します。さらに、高額の不正出金を拒否する支出制限の実装は、金融アプリケーションにとって重要な機能となる可能性があります。

### **HooksとEthereum Virtual Machine（EVM）の違い**

Xahau HooksとEVMにより、開発者はプラットフォーム内でカスタムロジックと自動化を構築・展開できます。ただし、これら2つの技術を区別するいくつかの主な違いがあります。

* **プラットフォーム互換性**: HooksはXahau向けに調整されており、EVMスマートコントラクトはEthereumベースのブロックチェーン向けに設計されています。
* **実行効率**: HooksはWebAssembly（WASM）を利用しており、速度と効率の面でEVMが使用するバイトコードより優れています。
* **予測可能な実行時間**: Xahau HooksはGuardを使用して最大実行時間が事前に適切に制限・把握されることを保証し、効率性を向上させます。

### XRPLエコシステムにおけるHooksの代替手段

RippleとPeersystは、EVMと互換性のあるサイドチェーンが[同社のdevnet](https://opensource.ripple.com/docs/evm-sidechain/intro-to-evm-sidechain/)で稼働を開始したことを発表しました。このサイドチェーンは、独自のコンセンサスプロトコルとトランザクションルールを持つ自律的なブロックチェーンとして機能します。EVMサイドチェーンはHooksの代替手段であり、エコシステムにスマートコントラクト機能を追加します。

ただし、EVMサイドチェーンコントラクトはレイヤー2で機能することに注意が必要です。つまり、コントラクト実行のためにXRPをサイドチェーンに移行し、その後メインレジャーに戻す2段階プロセスが必要です。サイドチェーン上では、再びXRPL上でも、レイヤー2スマートコントラクトはフローに影響を与えることができません。Hooksは最初にトランザクションを許可するかどうかを決定できます。レイヤー2は遡及的な決定を行うことができますが、最初のトランザクションはすでに発生しています。

HooksはXRPLとより密接に統合されており、レイヤー1 XRPLコアフォークであるXahau上で直接動作するため、EVMと互換性のあるサイドチェーンよりもXRPLプラットフォームの特定の機能と能力を活用するために、基盤となるブロックチェーン技術とより緊密に統合されています。WebAssemblyの固有のスケーラビリティとパフォーマンスにより、HooksはXahauの機能強化に最適です。

### HooksはXahauのオンレジャー機能を拡大し、成長に貢献します

HooksはXahauにネイティブなスマートコントラクト機能を追加し、ユーザーのユニークなニーズを満たすカスタムアプリケーションの構築を可能にし、新しい機能をもたらし、機能の全く新しい領域を開きます。Hooksを使えば、可能性は事実上無限です。

Xahauが成長し続ける中、Hooksが小売・エンタープライズユーザーによるプラットフォームのさらなるイノベーションと採用を推進する上で重要な役割を果たすことは疑いの余地がありません。
