<Aside type="note">
Hooksは、Xahau専用に設計された小さく効率的なWebAssemblyモジュールです。&#x20;

Hooksは任意の言語（WebAssemblyにコンパイル可能な言語）で書くことができ、ほとんどのビジネスロジックとスマートコントラクトのコンセプトをHookに実装できます。

### Hooksとは？ <a href="#what-are-hooks" id="what-are-hooks"></a>

Hooksにより、Xahau内でカスタマイズされたロジックと自動化を作成でき、トランザクションをよりスマートで便利にできます。これらの小さく効率的なモジュールは、レジャー上の特定イベントに対するカスタムトリガーの作成など、カスタムのオンレジャー機能を追加します。

これらのトリガーは、指定されたイベントに応じてオンレジャーアクションを送信したり、他のアクションを実行したりするために使用できます。Hooksは現在、[Xahauネットワーク](https://xahau.network/)で利用できます。

> _Hooksの理論的なコンセプトについて詳しく知るには、 _[_コンセプト_](/ja/docs/hooks/concepts/introduction)_ セクションをご覧ください。_

> _CでHooksをコーディングする方法とその関数について知るには、 _[_Hooks C関数_](/ja/docs/hooks/functions/overview/hook-api-conventions/)_ をご覧ください。_

> _[Hooks Builderサイト](https://builder.xahau.network/)では、ブラウザ内でテストネット上の自分のHooksを開発、テスト、デバッグ、デプロイできます。例を使用することも、ゼロから構築することもできます。_

> _[Hookstore](https://hookstore.xahau.network/)は、Hooksを発見、共有、管理するためのプラットフォームを提供します。[Hookstore Console](https://console.hookstore.xahau.network/)を使用して、Hooksを管理し、エコシステムを探索できます。_

> JavaScriptでHooksを書けるようにする今後の開発があり、これはJSHooksとしても知られています。現時点では[JSHooks](https://github.com/Xahau/jshooks-alpha)リポジトリを使用してテストできます。

### Hooksが重要である理由 <a href="#why-are-hooks-a-big-deal" id="why-are-hooks-a-big-deal"></a>

簡単に言えば、HooksはXahauに堅牢なスマートコントラクト機能を追加し、特定のニーズと要件に合った独自機能を持つアプリケーションを構築し、デプロイできるようにします。

Hooksは幅広いビジネスロジックとスマートコントラクトのパラダイムの実装に使用できるため、多用途なプラットフォームを提供します。Hookがアカウントに設定されると、次のことが可能になります。

* アカウントへの、またはアカウントからのトランザクションをブロックまたは許可する。
* プログラム上の判断に役立てるため、Hookの内部状態とロジックを変更し追跡する。
* アカウントに代わって自律的に新しいトランザクションを開始する。

HooksはCまたは他の任意の言語で書き、その後WebAssemblyにコンパイルできます。

### 具体的なHooksとユースケースの例 <a href="#some-examples-of-specific-hooks" id="some-examples-of-specific-hooks"></a>

これらの具体例を通じてHooksの可能性を示します。それぞれがXahau上のスマートコントラクト機能の独自の応用を表しています。

* **Auto-Savings Hook**: Hookを設定して、一定額のXAHをレジャー上の別の貯蓄アカウントへ転送することで、貯蓄を自動化します。これは、XAHの一部を貯蓄し、日次、週次、月次など指定された間隔で貯蓄を積み上げるために行えます。この定期的な転送メカニズムは、個人向け金融アプリケーションやサブスクリプションベースのモデルを開発する基盤になり得ます。
* **Carbon-Offset Hook**:各トランザクションは、金額の1%を、善良な目的に資金を使用する信頼できる非政府組織（NGO）が管理するカーボンオフセットアカウントへ追加転送します。この機能は、行われるすべてのトランザクションを通じて環境持続可能性に貢献するアプリケーションを構築する基盤として使用できます。
* **Firewall Hook**:受信および送信トランザクションをフィルタリングします。Firewall Hookは、既知の詐欺アカウントから発信された、または疑わしいメモを含む悪意のあるトランザクションをブロックできます。別のアカウント上のHookから更新されたブロックリストを取得することで、ファイアウォールは手動介入なしに詐欺に対する堅牢な防御を維持します。さらに、高額な不正出金を拒否する支出制限の実装は、金融アプリケーションにとって重要な機能になり得ます。

### **Ethereum Virtual Machine（EVM）とHooksの違い**

Xahau HooksとEVMはいずれも、開発者がそれぞれのプラットフォーム内でカスタムロジックと自動化を構築しデプロイできるようにします。しかし、これら2つの技術にはいくつかの重要な違いがあります。&#x20;

* **プラットフォーム互換性**: HooksはXahau向けに調整されていますが、EVMスマートコントラクトはEthereumベースのブロックチェーン向けに設計されています。
* **実行効率**: HooksはWebAssembly（WASM）を利用し、速度と効率の点でEVMが使用するbytecodeを上回ります。
* **予測可能な実行時間**: Xahau Hooksはguardsを使用して最大実行時間が十分に境界付けられ、事前に既知であることを保証し、効率を向上させます。

### XRPLエコシステムにおけるHooksの代替

RippleとPeersystは、EVM互換サイドチェーンが[同社のdevnet](https://opensource.ripple.com/docs/evm-sidechain/intro-to-evm-sidechain/)で稼働中であることを発表しました。このサイドチェーンは、自律的なブロックチェーンとして機能し、独自のコンセンサスプロトコルとトランザクションルールを備えています。EVMサイドチェーンはHooksの代替であり、エコシステムにスマートコントラクト機能を追加します。&#x20;

ただし、EVMサイドチェーンのコントラクトはLayer 2上で機能する点に注意が必要です。つまり、XRPをサイドチェーンへ移行してコントラクトを実行し、その後メインレジャーへ戻す2段階のプロセスが必要です。いったんサイドチェーン上に移り、その後再びXRPL上に戻るため、Layer 2スマートコントラクトは流れに影響を与えることができません。Hooksは、そもそもトランザクションが許可されるかどうかを判断できます。Layer 2は事後的な判断を下せますが、最初のトランザクションはすでに発生しています。

HooksはXRPLとより密接に統合されており、Layer 1のXRPL-coreフォークであるXahau上で直接動作します。そのため、XRPLプラットフォームの特定の機能と能力を活用するために、基盤となるブロックチェーン技術とEVM互換サイドチェーンよりも緊密に統合されています。WebAssembly固有のスケーラビリティと性能により、HooksはXahauの機能を強化するのに最適です。

### Hooksはオンレジャー機能を拡張し、Xahauの成長を支援します

HooksはXahauにネイティブなスマートコントラクト機能を追加し、ユーザー固有のニーズを満たすカスタムアプリケーションの作成を可能にし、新しい機能をもたらし、まったく新しい機能領域を開きます。Hooksがあれば、可能性は事実上無限です。

Xahauが成長し続けるにつれて、Hooksがリテールユーザーとエンタープライズユーザーによるプラットフォームのさらなるイノベーションと採用を推進するうえで重要な役割を果たすことは間違いありません。
