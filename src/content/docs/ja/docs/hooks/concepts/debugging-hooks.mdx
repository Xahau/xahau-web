### Hookをデバッグするには？

Hook APIは`trace`名前空間に一連の関数を提供します。xrpldが _trace_ ログレベルで構成されている場合、これらの関数は`xrpld`ログファイルに出力を書き込みます。一般的に、これらの関数を使うと、変数やバッファの値を確認したり、ランタイムにおけるHookの実行と状態をトレースしたりできます。

<Aside type="tip">
執筆時点では対話型のHookデバッガーはありません。trace関数を使用する必要があります。
</Aside>

### Trace API

Hooks APIでは、次の`trace`関数を利用できます。

| Hook API                                                                | 機能                                                                              |
| ----------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| [trace](/ja/docs/hooks/functions/trace-debug/trace)              | utf-8メッセージを出力し、その後にユーザー指定のバッファを出力します（この最後の部分は任意でhexとして出力できます）。 |
| [trace_num](/ja/docs/hooks/functions/trace-debug/trace_num)     | utf-8メッセージを出力し、その後に整数を出力します。                                            |
| [trace_float](/ja/docs/hooks/functions/trace-debug/trace_float) | utf-8メッセージを出力し、その後にXFL浮動小数点数を出力します。                          |

### 例

次のコードは1行のtraceを出力し、その後発信トランザクションをacceptします。

```c
#include "../hookapi.h"
int64_t hook(int64_t reserved)
{
  trace_num(SBUF("A number"), 10);
  accept(0,0,0);
  return 0;
}
```

PaymentがHookアカウントから送信された場合、またはHookアカウントに送信された場合に`xahaud`が生成するログ行の例です。

```
2021-Apr-13 13:59:11.083700726 UTC View:TRC
    HookTrace[rHb9CJAWyB4rj91VRWn96DkukG4bwdtyTh-rE3SfnjwfzZFL3JK9cLVfJuy8Ar1XnCqPw]:
        A number 10
```

上記はログ内では1行として表示されますが、ここでは見やすさのために分割しています。

<Aside type="tip" title="テストネットを使用する">
[Xahau Testnet](https://xahau-test.net/)はHooksをテストするのに最適な場所です。
</Aside>

### ログ形式

ログ形式の内訳を以下の表に示します。

| 部分                                        | 説明                                                                                                                                                            | # |
| ------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- | - |
| 2021-Apr-13 13:59:11.083700726 UTC View:TRC | ログ行に付く`xahaud`のプレフィックスです。                                                                                                                                      | 1 |
| HookTrace                                   | これはHook自体によって開始されたtraceであり、Hookに関する他の情報ではありません。他の情報は`HookError`、`HookEmit`、`HookInfo`タグで利用できます。 | 2 |
| \[rHb9CJAWyB4rj91VRWn96DkukG4bwdtyTh        | 角括弧内の最初のアカウントはHookアカウントです。                                                                                                          | 3 |
| -rE3SfnjwfzZFL3JK9cLVfJuy8Ar1XnCqPw]:       | 角括弧内の2番目のアカウントは発信アカウントです。                                                                                                  | 4 |
| A number                                    | これは、traceペイロードの前に配信するようHookに指示されたメッセージです。                                                                                              | 5 |
| 10                                          | これはtraceペイロードです。                                                                                                                                              | 6 |

<Aside type="tip">
`Xahaud`は大量の出力を生成します。そのため一般的には、関心のあるアカウントについてログをgrepすることをお勧めします。

例: `tail -f log | grep HookTrace | grep <account>`
</Aside>
