### 制約

すべてのHooksは、Xahauアカウントに設定できるようになる前に、単一の[WebAssemblyモジュール](https://webassembly.github.io/spec/core/syntax/modules.html)へコンパイルされます。

Hookは常に、次の関数のうち正確に1つ、または両方を実装してエクスポートします。

`int64_t hook(uint32_t ctx) { ... }` _必須_

* Hookが設定されているアカウントにトランザクションが入ってくる、またはそのアカウントから出ていくたびに実行される（`ctx = 0`）、または
* [Weak Transactional Stakeholder](/ja/docs/hooks/concepts/weak-and-strong)として実行されるときに実行される（`ctx > 0`）。

`int64_t cbak(uint32_t ctx) { ... }` _任意_

* 発行トランザクションがレジャーに正常に受け入れられたときに実行される（`ctx = 0`）、または
* 発行トランザクションがどのレジャーにも受け入れられない場合に実行される（`ctx = 1`）。

Hooksは他の関数を指定できません。代わりに、これら2つの関数内ですべての計算を行うため、マクロをうまく利用する必要があります。これは、Hooksのランタイムを予測可能に保つための計算上の制約の一部です。

さらに、Hooksには_ヒープ_メモリが提供されません。必要なすべてのメモリはスタック上に予約して使用する必要があります。

### 例

以下はCで書かれたHookの例です。このHookは発信トランザクションをacceptする前に、traceログへ0...3を出力します。

```c
#include <stdint.h>
#define GUARD(maxiter) _g(__LINE__, (maxiter)+1)
extern int32_t _g (uint32_t id, uint32_t maxiter);
extern int64_t accept (uint32_t read_ptr, uint32_t read_len, int64_t error_code);
extern int64_t trace_num (uint32_t read_ptr, uint32_t read_len, int64_t number);

int64_t hook(uint32_t ctx)
{
    for (int i = 0; GUARD(3), i < 3; ++i)
    {
        trace_num("test", 4, i);
    }
    accept (0,0,0); 
    return 0;
}
```

<Aside type="tip">
教育目的のため、上記の例では開発者が通常使用する`hookapi.h`を意図的に含めていません。
</Aside>

通常、Hook開発者はWebAssemblyを直接調べる必要はありません。しかし、サンプルHookの内容を確認することは、概念を理解するための有益な練習になります。

上記から次のことがわかります。

* Hooks APIから3つの関数（`_g`、`accept`、`trace_num`）がインポートされています。
* Hookによって2つの関数（`cbak`、`hook`）が定義されています。
* Hookによって2つの関数（再び`cbak`、`hook`）がエクスポートされています。
* いくつかの静的（定数）データがHookに記録されています（末尾の`data`を参照）。

HookはHooks APIから利用可能な関数のみをインポートしなければならず、`cbak`関数と`hook`関数のみをエクスポートしなければならない点は非常に重要です。加えて、すべてのHooksはHooks APIから`_g`をインポートしなければなりません。これは`guard`関数です。

<Aside type="tip">
WebAssemblyはプラットフォーム非依存の汎用計算`bytecode`言語です。人間が読める同等表現と一対一に対応しています。これらは相互に使用されます。
</Aside>

### 不要なエクスポート

ほとんどのWebAssemblyコンパイラ（上記のものを含む）は、独自のリンク用途のために追加のエクスポートを生成します。多くの場合、これらの生成を無効にするのは困難、または不可能です。

不要なエクスポートがあると、それ以外は有効なHookでも拒否されます。したがって、コンパイル後、開発者は[Hook Cleaner Utility](https://github.com/XRPLF/hook-cleaner-c)を使用してこれらを取り除くべきです。そうしないと、Hookは拒否されます。


<Aside type="caution">
[Hook Cleaner Utility](https://github.com/XRPLF/hook-cleaner-c)を使用することを忘れないでください。使用しないとHooksは拒否されます。
</Aside>
