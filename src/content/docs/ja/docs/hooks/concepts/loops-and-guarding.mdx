### ガードとは？

Hooksは意図的に[チューリング完全](https://en.wikipedia.org/wiki/Turing_completeness)ではありません。これは、任意のループが禁止されていることを意味します。代わりに、ループを厳格な「最大反復回数」の境界で _ガード_ する必要があります。

ガードは、各ループの先頭でコード内に置かれるマーカーです。このマーカーは、あらゆる可能なシナリオにおいてループの上限が何になるかをXahauに知らせます。したがって、通常はループが2回実行されるだけでも、ときどき _500_ 回実行される場合、ガードには500を指定します。

ガードは、Xahauが実行前にHookの _最悪ケースの実行時間_ （命令数）を決定するために使用されます。これはXahauがHookの実行に対して請求する手数料の基礎であり、実行時間を予測可能かつ制御可能にします。

<Aside type="tip">
  他のスマートコントラクトプラットフォームから移行してきた既存の開発者は、最初はガードを面倒に感じるかもしれませんが、慣れれば通常のforループと同じくらい簡単に使えます。
</Aside>

### guard関数

guard関数は、ループが行う**最大反復回数**をレジャーに伝えます。具体的には、この関数は2つの引数を取ります。

```c
int32_t _g (uint32_t id, uint32_t maxiter);
```

最初の引数`id`は、このガードの識別子です。これは開発者が選ぶ一意の定数で、通常はソースファイル内の行番号が使用されます。

2番目の引数`maxiter`は、このガードがHookの実行中に`maxiter`回を超えて _ヒット_ しないことを開発者がレジャーに約束するものです。ガード呼び出しがこの回数を超えて実行された場合、Hookは`GUARD_VIOLATION`（[Hook APIの戻りコード](/ja/docs/hooks/functions/overview/return-codes)）で自動的にrollbackします。ガードはループ条件がチェックされる _前_ にヒットするため、想定される総反復回数に1を足すことが重要です。（注: GUARD()マクロはすでに1を足します）。

<Aside type="caution">
  ガードは数値リテラルを使用して設定する必要があります。Guard内で変数やランタイム値を使用することはできません。
</Aside>

### ガードの強制

Cの次のforループを考えます。

```c
#define GUARD(maxiter) _g(__LINE__, (maxiter)+1)
for (int i = 0; GUARD(3), i < 3; ++i)
{
  ...
}
```

Cでは、カンマ演算子は式のリスト（例: `A, B, C`）内の各式を実行し、最後の式（例: `C`）を返します。したがって上記の条件は依然として`i < 3`ですが、条件がチェックされる前にガードが呼び出されます。これは、Cでforループを使用する際に _ガードルール_ を満たす唯一の方法です。

<Aside type="tip" title="ガードルール">
  <code>_g</code>（guard関数）の呼び出しは、loop命令の後にある最初の分岐命令でなければなりません。
</Aside>

以下は、上記をコンパイルしたときのWebAssembly出力です。ループの開始時にguard関数が呼び出されていることに注意してください。この呼び出しの前に許可される命令は、非分岐命令（通常は定数の操作）のみです。

```wat
    block  ;; label = @1
      loop  ;; label = @2
        i32.const 3
        i32.const 14
=====>  call $_g          <=====
        drop
        ...
```

### ネストされたループ

ネストされたループを使用する場合、`maxiter`引数はガードが _ヒット_ される合計回数を反映している必要があります。つまり、ネストを掛け合わせる必要があります。

次の例を考えます。

```c
#define GUARD(maxiter) _g(__LINE__, (maxiter)+1)
for (int i = 0; GUARD(3), i < 3; ++i)
{
  for (int j = 0; GUARD(15), j < 5; ++j)
  {
    ...
  }
}
```

内側のループのガードが**15**に設定されていることに注目してください。Hook実行中に内側のガードがヒットされる最大回数を計算するには、ループ同士を掛け合わせる必要があります。

### 再帰なし

Hooks Amendmentでは、Hook API関数ではない関数の呼び出しは禁止されています。すべてのユーザーコードは、許可された2つのHook関数`cbak`と`hook`の中に収まらなければなりません。

<Aside type="caution">
  ガードを正しく使用しないと、試行された<code>SetHook</code>トランザクションは拒否されます。
</Aside>
