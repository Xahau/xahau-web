<Card title="Hook設計哲学">
_トランザクションの影響を受けるすべての当事者には、自分のHooksを実行する機会があるべきです。_
</Card>

HooksがStrong実行されない場合、それらの実行費用を発信トランザクションに請求するのは公平ではありません。たとえば、DEX上で20件のOfferと交差する_OfferCreate_が、それぞれのアカウントのHooksの実行費用まで支払うよう強制されるべきではありません。

したがって通常のWeak実行では、その実行の手数料はHookの所有者から徴収されます。これを有効にするには、次の条件が必要です。

* Hook所有者がAccountSetトランザクションを使用して、自分のXahauアカウントに`asfTshCollect`を設定していること。
* Hook所有者が、Weak TSHとして呼び出されることを望む特定のHookに`hsfCollect`を設定していること。

### 手数料責任表

| Weak実行の種類                                                                                                                                                   | 手数料                                                                                                                                                                                    |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <p><strong>Again As Weak</strong><br />- Strong実行されたHookが<a href="/ja/docs/hooks/functions/hook-context/hook_again">hook_again</a>を呼び出したときに発生します</p>  | 無料（Strong実行によってすでに支払われています）。                                                                                                                                           |
| <p><strong>Callback</strong><br />-発行トランザクションがレジャーに取り込まれた場合、または今後どのレジャーにも取り込まれることが不可能としてフラグ付けされた場合に発生します。</p> | 無料（発行時にすでに支払われています）。                                                                                                                                                   |
| <p><strong>Weak Transactional Stakeholder</strong><br />-トランザクションが何らかの形であなたのアカウントに軽微な影響を与える場合に発生します。</p>                                            | あなたのアカウントに`asfTshCollect`フラグが付いており、かつあなたのHookに`hsfCollect`フラグが付いている場合に限り、あなたのアカウントが支払います（発信トランザクションではありません）。 |

<Aside type="caution">
これはほとんどのHook開発者が使用しない可能性が高い高度な機能です。
</Aside>
