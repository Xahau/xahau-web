<Card title="Hook設計哲学">
_トランザクションによって影響を受けるすべての当事者は、自分のHooksを実行する機会を持つべきです。_
</Card>

HooksがStrongly Executedでない場合、実行費用を発信トランザクションに請求することは不公平です。例えば、DEXで20のオファーをクロスする_OfferCreate_は、それらのアカウントのHooks実行のために支払いを強制されるべきではありません。

したがって、通常のWeak実行中は、実行費用はHookのオーナーから収集されます。これを有効にするには：

* HookオーナーはAccountSetトランザクションを使用してXahauアカウントに`asfTshCollect`を設定する必要があります。
* HookオーナーはWeak TSHとして呼び出されたい特定のHookに`hsfCollect`を設定する必要があります。

### 手数料責任テーブル

| Weak実行のタイプ                                                                                                                                                            | 手数料                                                                                                                                                                                 |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <p><strong>Again As Weak</strong><br />- Strongly ExecutedされたHookが<a href="/docs/hooks/functions/hook-context/hook_again">hook_again</a>を呼び出した時に発生</p>  | 無料（Strong実行時にすでに支払い済み）。                                                                                                                                               |
| <p><strong>Callback</strong><br />- Emittedトランザクションがレジャーになるかレジャーに入ることが不可能とフラグ立てされた時に発生。</p> | 無料（Emission時にすでに支払い済み）。                                                                                                                                                 |
| <p><strong>弱いトランザクション利害関係者</strong><br />- トランザクションが何らかの形でアカウントに軽度の影響を与えた場合に発生。</p>                                            | アカウントが`asfTshCollect`フラグでマークされており、かつHookが`hsfCollect`フラグでマークされている場合のみ、（発信トランザクションではなく）自分のアカウントが支払います。 |

<Aside type="caution">
これはほとんどのHook開発者が使用しない高度な機能です。
</Aside>
