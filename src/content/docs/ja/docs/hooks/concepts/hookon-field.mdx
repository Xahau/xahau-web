### HookOnフィールドの理解

この符号なし256ビット整数の各ビットは、Hookが特定のトランザクションタイプで実行されるかどうかを示します。すべてのビットは_アクティブロー_ですが、**ビット22は**_アクティブハイ_です。22はttHOOK_SETであるため、デフォルト値のすべて0はSetHookトランザクションでは発火しませんが、他のすべてのトランザクションタイプでは発火します。これは、誤動作するHookでXahauアカウントをブリックしないように人々を助けるための意図的な設計上の選択です。

ビットは右から左に番号が付けられています：

* ビット0 - 右端、つまり最下位ビット。
* ビット63 - 左端、つまり最上位ビット。

例（256ビット符号なし整数型を想定）：

1. Hookを完全に無効にしたい場合：

```c
~(1ULL << 22) /* ビット22を除くすべてのビットは1 */
```

2. ttPAYMENT以外のすべてでHookを無効にしたい場合：

```c
~(1ULL << 22) & ~(1ULL)
```

3. ttHOOK_SET以外のすべてでHookを有効にしたい場合：

```c
0
```

4. ttHOOK_SET（危険）と他のすべてのトランザクションタイプでHookの発火を有効にしたい場合：

```c
(1ULL << 22)
```
