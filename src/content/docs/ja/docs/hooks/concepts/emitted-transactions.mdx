### 背景

Xahauへの**すべての**変更は、レジャーへの有効なトランザクションの適用の結果でなければなりません。したがって、変更_X_が行われた場合、いくつかのトランザクション_Y_が責任を持ちます。

Hooks APIを設計する際、Hooksがトランザクションの単純な受け入れまたは拒否を_超えて_レジャーに変更を加える方法が必要でした。しかし、これらの変更を発信トランザクションに添付することは混乱を招き、システムの全体的な複雑さを大幅に増加させました。

例えば、Hookがあなたに資金を送る必要があると仮定すると、送金操作はアカウントセットトランザクションのような全く無関係のものである可能性のある発信トランザクションによってレジャーに効果的に適用されることになります。さらに、この送金操作は支払いの受信側の別のHookをトリガーできる必要があります。

解決策：**Emittedトランザクション**。発信トランザクションがトランザクションの内容が示すことを正確に実行できるようにします。Hookがレジャーへの追加変更（支払いの送信など）を行う必要がある場合、まったく新しいトランザクションを作成して_emit_します。

### Emittedトランザクションとは？

EmittedトランザクションはHookの実行によって作成された_新しい_トランザクションであり、次のレジャーで処理するためにコンセンサスに入力されます。トランザクションは任意のトランザクションタイプにできますが、厳格なEmissionルールに従う必要があります。

トランザクションをEmitするには、HookはまずシリアライズされたトランザクションをPrepareし、次に[emit](/docs/hooks/functions/emitted-transaction/emit-1)を呼び出します。

EmittedトランザクションはHooksをトリガーする可能性があり、それによってさらにトランザクションがEmitされる可能性があるため、すべてのEmittedトランザクションは`EmitDetails`ブロックに`burden`と`generation`フィールドを持ちます。`EmitDetails`ブロックは従来のトランザクションの署名フィールドに取って代わります。

`burden`と`generation`フィールドは、指数関数的に拡張するEmittedトランザクションのコストを指数関数的に増加させることで、レジャーへの[フォークボム](https://en.wikipedia.org/wiki/Fork_bomb)攻撃を集合的に防止します。

Hooks APIが_リライトなし_の厳格なルールに従うことに注意することが重要です。Emitされるために完全かつ有効で正規の形式でEmittedトランザクションをxahaudに提示_しなければ_なりません。そうしないと拒否されます。トランザクションを代わりに構築することはxahaudの仕事ではありません。Hookはこれを自分で行う必要があります。

### コールバック

[はじめに・用語集](/docs/hooks/concepts/terminology)で紹介されているように、EmittedトランザクションはレジャーにAcceptされた時にコールバックをトリガーします。コンセンサスの分散性により、EmittedトランザクションのレジャーへのAcceptanceは**保証されない**ですが、通常はほぼ保証されます。

EmittedトランザクションがレジャーにAcceptされる前に期限切れになった場合（レジャーが満杯、手数料がEmittedトランザクションに対して高すぎる、またはEmittedトランザクションが何らかの理由で無効など、様々な理由により）、Emittedトランザクションをクリーンアップするためにレジャーに_擬似トランザクション_が作成されます。この擬似トランザクションはHookのコールバックも呼び出し、`parameter = 1`でEmittedトランザクションが確かに失敗したことを示します。

### Emissionルール

[emit](/docs/hooks/functions/emitted-transaction/emit-1) Hook APIは、提案された（Emitされる予定の）トランザクションに以下のルールを適用します。

| # | Emissionルール                                          | 説明                                                                                                                                                                                                                                                              |
| - | ------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1 | `sfSequence` = 0                                       | EmittedトランザクションはHookアカウントのシーケンス番号を増加させません。これは常にゼロに設定する必要があります。                                                                                                                                         |
| 2 | `sfPubSigningKey` = 0                                  | Emittedトランザクションは署名されていませんが、xrpld処理には必須フィールドです。すべてゼロに設定する必要があります。                                                                                                                                      |
| 3 | `sfEmitDetails` 存在かつ有効                              | Emittedトランザクションには`sfEmitDetails`ブロックが必要で、正しく記入されている必要があります。以下のEmitDetailsセクションを参照してください。                                                                                                                 |
| 4 | `sfSignature` 不在                                      | このフィールドはEmittedトランザクションにあってはなりません。存在する場合、トランザクションが曖昧になります。                                                                                                                                                |
| 5 | `LastLedgerSequence` 有効かつ未来                         | すべてのEmittedトランザクションは最後のレジャーシーケンスを設定する必要があります。これにより、HookはEmittedトランザクションが失敗したかどうかを把握できます（コールバックを受け取らなかったため）。現在、これは現在のレジャーから最大5レジャー後に設定されています。              |
| 6 | `FirstLedgerSequence` 有効かつ次のレジャーに設定          | すべてのEmittedトランザクションは次のレジャー（現在のレジャーの後）に最初のレジャーシーケンスを設定する必要があります。Hooksが1つのレジャー内で再帰的にカスケードしないようにするためです。現在、現在のレジャーの次のレジャーに強制されています。 |
| 7 | 手数料が適切に計算・設定                                    | 手数料はEmittedトランザクションのサイズとネットワークへの負担（このEmittedトランザクションが別のEmittedトランザクションの結果である場合）によります。                                                                                                    |
| 8 | 世代上限を超えない                                        | EmittedトランザクションはEmittedトランザクションを生成でき、これらはチェーンを形成できます。チェーンの長さは`sfEmitGeneration`です。現在、10が上限です。                                                                                                  |

### EmitDetailsブロック

すべてのEmittedトランザクションには、以下のテーブルのフィールドで正しく入力された`sfEmitDetails`オブジェクトを含む必要があります。

| フィールド             | 必須値                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    | 説明                                                                                                                                                                                                                                                                     |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| sfEmitGeneration  | 発信トランザクション自体がEmittedトランザクションである場合、そのトランザクションの`sfEmitGeneration`より1多い。発信トランザクションがEmittedトランザクションでない場合は`1`。[etxn_generation](/docs/hooks/functions/emitted-transaction/etxn_generation)を使用して入力する必要があります。 | このフィールドは、他のトランザクションのEmitを引き起こすEmittedトランザクションのチェーンを追跡します。 |
| sfEmitBurden      | 発信トランザクション自体がEmittedトランザクションである場合、[etxn_reserve](/docs/hooks/functions/emitted-transaction/etxn_reserve)を使用してHookがEmitすると宣言した最大トランザクション数を掛けた発信トランザクションの`burden`。発信トランザクションがEmittedトランザクションでない場合は`1`。[etxn_burden](/docs/hooks/functions/emitted-transaction/etxn_burden)を使用して入力する必要があります。 | このフィールドはフォークボムを検出するためのヒューリスティックです。手数料はburdenに基づき、チェーン反応が開始された時にネットワークが自己強化するEmittedトランザクションに圧倒されないように指数関数的に増加します。 |
| sfEmitParentTxnID | 発信トランザクションのトランザクションID                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     | トランザクションをEmitしたHook実行は発信トランザクションに接続されています。したがってこのフィールドは動作の効率的なトレースのために常に必要です。 |
| sfEmitNonce       | [nonce](/docs/hooks/functions/emitted-transaction/etxn_nonce)の呼び出しによって生成された特別な決定論的ノンス                                                                                                                                                                                                                                                                                                                                                                                                                | ノンスが使用されない場合、EmittedトランザクションはFull同一フィールドで同一のトランザクションハッシュを持ちます。ただし、ネットワーク上のすべてのノードがノンスに合意する必要があるため、決定論的なノンスを生成する特別なHook APIが提供されています。 |
| sfEmitCallback    | 20バイトのHookアカウントID                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   | このフィールドはxahaudがコールバックを開始する必要がある時に使用され、どのHookとアカウントでコールバックを開始するかを把握します。コールバックはEmittedトランザクションがレジャーにAcceptされた時に発生します。 |

[サンプルHooks](https://github.com/XRPL-Labs/xrpld-hooks/tree/hooks-ssvm/hook-api-examples)、特にPeggy、Carbon、Doublerは、シンプルなものとより複雑なトランザクションの両方をEmitする方法を示しています。
