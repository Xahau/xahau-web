### 背景

Xahauに加えられる**すべての**変更は、レジャーに有効なトランザクションを適用した結果でなければなりません。したがって、何らかの変更_X_が行われた場合、何らかのトランザクション_Y_がその責任を負います。

Hooks APIを設計する際、Hooksが単にトランザクションをacceptまたはrejectするだけでなく、レジャーに変更を加える方法が必要でした。しかし、これらの変更を発信トランザクションに紐づけると混乱を招き、システム全体の複雑さが大幅に増加しました。

たとえば、Hookがあなたにいくらかの資金を送る必要があるとします。その送金操作は、Account Setトランザクションのようにまったく関係のないものかもしれない発信トランザクションによって、実質的にレジャー上で実行されることになります。さらに、この送金操作は、Paymentの受信側で別のHookをトリガーできる必要もあります。

解決策が**発行トランザクション**です。発信トランザクションには、そのトランザクションの内容が示すことだけを正確に実行させます。HookがPaymentの送信など、レジャーに追加の変更を加える必要がある場合、Hookはまったく新しいトランザクションを作成し、それを _発行_ します。

### 発行トランザクションとは？

発行トランザクションは、Hookの実行によって作成され、次のレジャーで処理されるようコンセンサスに投入される_新しい_トランザクションです。トランザクションタイプは任意ですが、厳密な発行ルールに従う必要があります。

トランザクションを発行するには、Hookはまずシリアライズ済みトランザクションを準備し、その後[emit](/ja/docs/hooks/functions/emitted-transaction/emit-1)を呼び出します。

発行トランザクションは次のレジャーでHooksをトリガーでき、そのHooksがさらにトランザクションを発行する場合があるため、すべての発行トランザクションは`EmitDetails`ブロック内に`burden`フィールドと`generation`フィールドを持ちます。`EmitDetails`ブロックは、従来のトランザクションの署名フィールドを置き換えます。

`burden`フィールドと`generation`フィールドは、指数関数的に拡大する発行トランザクションのコストを指数関数的に増加させることで、レジャーに対する[Fork bomb](https://en.wikipedia.org/wiki/Fork_bomb)攻撃を共同で防ぎます。

Hooks APIは _書き換えなし_ という厳格なルールに従うことに注意してください。発行するトランザクションは、完全で有効かつ正規に形成された形でxahaudに提示しなければならず、そうでなければ拒否されます。あなたの代わりにトランザクションを構築することはxahaudの仕事ではありません。Hook自身がこれを行う必要があります。

### コールバック

[はじめにと用語](/ja/docs/hooks/concepts/terminology)で紹介したように、発行トランザクションはレジャーに受け入れられるとコールバックをトリガーします。コンセンサスの分散的な性質により、発行トランザクションがレジャーに受け入れられることは**保証ではありません**が、通常はほぼ保証されています。

発行トランザクションがレジャーに受け入れられる前に期限切れになった場合（理由はさまざまです。レジャーが満杯、手数料が発行トランザクションにとって高すぎる、発行トランザクションが何らかの形で無効など）、発行トランザクションをクリーンアップするための _擬似トランザクション_ がレジャー内に作成されます。この擬似トランザクションもあなたのhookのコールバックを呼び出し、`parameter = 1`によって発行トランザクションが実際に失敗したことを示します。

### 発行ルール

[emit](/ja/docs/hooks/functions/emitted-transaction/emit-1) Hook APIは、提案された（発行予定の）トランザクションに対して次のルールを強制します。

| # | 発行ルール                                          | 説明                                                                                                                                                                                                                                              |
| - | ------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1 | `sfSequence` = 0                                       | 発行トランザクションはHookアカウントのシーケンス番号を _増加させません_。これは常にゼロに設定する必要があります。                                                                                                                                         |
| 2 | `sfPubSigningKey` = 0                                  | 発行トランザクションは署名されませんが、これはxrpld処理に必要なフィールドです。すべてゼロに設定する必要があります。                                                                                                                                      |
| 3 | `sfEmitDetails`が存在し有効                      | 発行トランザクションには`sfEmitDetails`ブロックが必要であり、これは正しく入力されていなければなりません。以下のEmitDetailsセクションを参照してください。                                                                                                                                  |
| 4 | `sfSignature`が存在しない                                   | このフィールドは発行トランザクション内に存在してはなりません。存在するとトランザクションが曖昧になるためです。                                                                                                                                     |
| 5 | `LastLedgerSequence`が有効で将来を指す           | すべての発行トランザクションにはlast ledger sequenceを設定する必要があります。これにより、Hookは発行トランザクションが失敗したかどうかを知ることができます（時間内にコールバックを受け取らなかったため）。現在、これは現在のレジャーから最大5レジャー後に設定されます。              |
| 6 | `FirstLedgerSequence`が有効で次のレジャーに設定されている | すべての発行トランザクションには、Hooksが単一のレジャー内で再帰的に連鎖しないよう、first ledger sequenceを次のレジャー（現在のレジャーの後）に設定する必要があります。現在、これは現在のレジャーの次のレジャーであることが強制されます。 |
| 7 | Feeが適切に計算され設定されている                     | 手数料は、発行トランザクションのサイズとネットワーク上のburden（つまり、この発行トランザクションが別の発行トランザクションの結果であったかどうか）に依存します。                                                                         |
| 8 | Generation上限を超えていない                            | 発行トランザクションは他の発行トランザクションを生成でき、それらはチェーンを形成できます。そのチェーンの長さが`sfEmitGeneration`です。現在、これは10に制限されています。                                                                            |

### EmitDetailsブロック

すべての発行トランザクションには、以下の表のフィールドが正しく設定された`sfEmitDetails`オブジェクトが含まれていなければなりません。

| フィールド             | 必須値                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        | 説明                                                                                                                                                                                                                                                                     |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| sfEmitGeneration  | <p>発信トランザクション自体が発行トランザクションだった場合、そのトランザクションの<code>sfEmitGeneration</code>より1大きい値。<br /><br />発信トランザクションが発行トランザクションではなかった場合は<code>1</code>。<br /><br />これは<a href="/ja/docs/hooks/functions/emitted-transaction/etxn_generation">etxn_generation</a>を使用して設定する必要があります。</p>                                                                                                                                                                  | このフィールドは、さらに他のトランザクションを発行させる発行トランザクションのチェーンを追跡します。                                                                                                                                                                  |
| sfEmitBurden      | <p>発信トランザクション自体が発行トランザクションだった場合、発信トランザクションの<code>burden</code>に、Hookが<a href="/ja/docs/hooks/functions/emitted-transaction/etxn_reserve">etxn_reserve</a>を使用して発行すると宣言した最大トランザクション数を掛けた値。<br /><br />発信トランザクションが発行トランザクションではなかった場合は<code>1</code>。<br /><br />これは<a href="/ja/docs/hooks/functions/emitted-transaction/etxn_burden">etxn_burden</a>を使用して設定する必要があります。</p> | このフィールドはforkbombを検出するためのヒューリスティックです。手数料はburdenに基づき、連鎖反応が始まると指数関数的に増加します。これは自己増殖する発行トランザクションによってネットワークが過負荷になるのを防ぐためです。                                                   |
| sfEmitParentTxnID | 発信トランザクションのトランザクションID                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     | トランザクションを発行したHook実行は発信トランザクションに接続されています。したがって、このフィールドは挙動を効率的に追跡するために常に必要です。                                                                                                    |
| sfEmitNonce       | [nonce](/ja/docs/hooks/functions/emitted-transaction/etxn_nonce)の呼び出しによって生成される特別な決定論的nonce                                                                                                                                                                                                                                                                                                                                                                                                                           | nonceが使用されなければ、同じフィールドを持つ発行トランザクションは同一になり、同一のトランザクションハッシュを持ちます。しかし、ネットワーク上のすべてのノードがnonceについて合意する必要があるため、決定論的nonceを生成する特別なHook APIが提供されています。 |
| sfEmitCallback    | 20バイトのHookアカウントID                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           | このフィールドは、xahaudがコールバックを開始する必要があるときに使用されます。どのHookとアカウント上でコールバックを開始すべきかを知るためです。コールバックは、発行トランザクションがレジャーに受け入れられたときに発生します。                                                                |
