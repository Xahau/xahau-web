<Aside type="caution">
  ほとんどのHook開発者はHookGrantsをほとんど使用する必要はなく、外部のHooksとアカウントにステート変更権限を付与する際には極めて注意を払うべきです。

  HookGrantは資金を直接盗むために使用できませんが、HookのStateを意図的に外部から変更すると、Hookが意図しない動作をする可能性があり、場合によっては盗難につながる可能性があります。

  Grantを使用する必要があると思う場合は、まず設計を再確認し、実際に使用する必要があるかどうかを確認してから続けてください。
</Aside>

### グラント

GrantはHookインストーラーが他のXahauアカウントの_外部_Hookに[ステート管理](/docs/hooks/concepts/state-management)権限を割り当てる方法を提供します。

[SetHookトランザクション](/docs/hooks/concepts/sethook-transaction)は、その`Hooks`配列内の任意の`Hook`オブジェクト内に`HookGrants`配列を指定することができます。`HookGrants`配列には1つ以上の`HookGrant`オブジェクト（最大8つ）が含まれます。

[パラメーター](/docs/hooks/concepts/parameters)とは異なり、`HookGrants`配列は常に[SetHookトランザクション](/docs/hooks/concepts/sethook-transaction)で指定された通りに設定されます。したがって、以前に設定された複数の他の`HookGrant`エントリを保持しながら特定の`HookGrant`を更新したい場合は、まず古い`HookGrants`配列を取得し、変更してから、[_Update_操作](/docs/hooks/concepts/sethook-transaction)で配列全体を再送信する必要があります。

すべてのGrantを削除するには、空の`HookGrants`配列を送信します。

<Aside type="caution">
  [パラメーター](/docs/hooks/concepts/parameters)とは異なり、`HookGrants`配列は常に[SetHookトランザクション](/docs/hooks/concepts/sethook-transaction)で指定された通りに設定されます。
</Aside>

Grantは、外部のXRPLアカウントまたはHookが、Grantが定義されている特定のHookの名前空間内のHookステートを変更することを許可します。

HookGrantには少なくとも以下を指定する必要があります：

* `HookHash`\
  さらにアカウントを指定することもできます：
* `Authorize`

HookHashで指定されたHookのみが、HookGrantが指定されたHookの名前空間内のHookステートを変更できます。`Authorize`が指定された場合、この権限はさらに`Authorize`で指定されたアカウントにインストールされた時のHookHashで指定されたHookのみに絞られます。

<Aside type="tip">
  GrantはHook外部にのみ適用され、インストールされているアカウントのHookステートに関するHooksの操作を制限することはありません。
</Aside>

### 例

```json
Account: "rALicebv3hMYNBWtu1VEEWkToArgYsYERs",
TransactionType: "SetHook",
Hooks:
[   
    {   
        Hook: {
            ...,
            HookNamespace: "3963ADEB1B0E8934C0963680531202FD511FF1E16D5864402C2DA63861C420A8",
            HookGrants:
            [   
                {   
                    HookGrant:    // 最初のgrant
                    {   
                        HookHash:  "78CAF69EEE950A6C55A450AC2A980DE434D624CD1B13148E007E28B7B6461CC8"
                    },
                    HookGrant:    // 2番目のgrant
                    {   
                        Authorize: "rCLairev2ma2gNZdcHJeTk7fCQ1ki84vr9",
                        HookHash:  "A5B8D62154DA1C329BE13582086B52612476720CEBD097EB85CEE1455E1C70A6"
                    }
                },  
            ]   
        }   
    }   
],  
... 
```

上記の_最初のgrant_は以下を許可します：

* `78CAF69EEE950A6C55A450AC2A980DE434D624CD1B13148E007E28B7B6461CC8`にハッシュするコードを持つHookの任意のインスタンス
* **任意のアカウント**で実行中
* アカウント`rALicebv3hMYNBWtu1VEEWkToArgYsYERs`のHookステートを変更する
* 名前空間`3963ADEB1B0E8934C0963680531202FD511FF1E16D5864402C2DA63861C420A8`の中で

上記の_2番目のgrant_は以下を許可します：

* `A5B8D62154DA1C329BE13582086B52612476720CEBD097EB85CEE1455E1C70A6`にハッシュするコードを持つHookの任意のインスタンス
* ただしアカウント`rCLairev2ma2gNZdcHJeTk7fCQ1ki84vr9`で実行中の場合のみ
* アカウント`rALicebv3hMYNBWtu1VEEWkToArgYsYERs`のHookステートを変更する
* 名前空間`3963ADEB1B0E8934C0963680531202FD511FF1E16D5864402C2DA63861C420A8`の中で

### Grantの使用

Grantを利用するには、Hookが[state_foreign_set](/docs/hooks/functions/state/state_foreign_set)を呼び出すことで、外部アカウントのStateオブジェクトを変更します。
