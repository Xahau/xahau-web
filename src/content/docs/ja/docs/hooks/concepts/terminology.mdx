### Hooksとは？

Hooksは、Xahau専用に設計された小さく効率的なWebAssemblyモジュールです。Hooksは任意の言語（WebAssemblyにコンパイル可能な言語）で書くことができ、ほとんどのビジネスロジックと多くのスマートコントラクトのコンセプトをHookに実装できます。通常、HooksはCで書かれます。

Hooksは`SetHook`トランザクションを使用してXahauアカウントに設定されます。アカウントにインストールされると、Hookは次のことができます。

1. アカウントへの受信トランザクションとアカウントからの送信トランザクションをブロックまたは許可する。
2. そのアカウント上のHookに固有の内部状態とロジックを変更し、維持する。
3. アカウントに代わって新しいトランザクションを発行する。

### 用語集

このHooksドキュメントとHooks APIでは、なじみのない用語を使用します。迷った場合は、以下の参照表を使用してください。

| 用語                     | 説明                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Hook                     | <p><em>この用語は文脈に応じてさまざまなものを指します</em><br />1. <a href="/ja/docs/hooks/concepts/sethook-transaction">SetHookトランザクション</a>タイプでXahauにアップロードできるWebAssemblyバイナリ。<br />2. すでにXahauにアップロードされ、Xahauアカウントに設定または構成されたWebAssemblyバイナリ。<br />3. そのようなバイナリの<a href="/ja/docs/hooks/concepts/compiling-hooks">ソースコード</a>。</p> |
| Originating Transaction  | Hookの発火をトリガーしたトランザクションです。これは、Hookが設定されたアカウントから送信された、またはそのアカウントに送信されたトランザクションのいずれかです。                                                                                                                                                                                                                                                                 |
| Originating Account      | 発信トランザクションを送信したアカウントです。                                                                                                                                                                                                                                                                                                                                                         |
| Hook Account             | 現在実行中のHookが存在するアカウントです。これはHookを所有するアカウント、Hookを作成した[SetHookトランザクション](/ja/docs/hooks/concepts/sethook-transaction)を実行したアカウント、そして現在実行中のHookのHook状態が属するアカウントです。                                                                                                                        |
| Installer                | SetHookトランザクションを使用して現在Hookをインストールしているアカウントです。                                                                                                                                                                                                                                                                                                                           |
| Emitted Transaction      | Hookの実行中にHookによって作成される、発信トランザクションではない新しいトランザクションです。通常、発信アカウントへ資金を送り返すために使用されます。参照: [発行トランザクション](/ja/docs/hooks/concepts/emitted-transactions)。                                                                                                                                                                |
| State                    | 32バイトキーから任意データへの、アカウントごとのKey-Valueマップです。アカウント上に存在するすべてのHooksは同じHook状態にアクセスでき、それを変更できます。Hook状態は発信アカウントではなくHookアカウント上に存在することに注意してください。参照: [状態管理](/ja/docs/hooks/concepts/state-management)。                                                                                                                 |
| SetHook                  | Hooks Amendmentで導入された新しいトランザクションタイプで、XahauアカウントにHookを設定します。参照: [SetHookトランザクション](/ja/docs/hooks/concepts/sethook-transaction)。                                                                                                                                                                                                                                                    |
| Guards                   | Hook内にループを書く場合に使用する必要がある特別な制御機構です。参照: [ループとガード](/ja/docs/hooks/concepts/loops-and-guarding)。                                                                                                                                                                                                                                                                         |
| Grants                   | Hook Installerが、別のアカウントまたは特定のHook（どこにインストールされているかに関係なく）に対して、Installerのアカウント上のHook状態を変更する権限を与える特別な許可です。                                                                                                                                                                                                                                    |
| Namespace                | ある状態キーの集合を別の集合から区別する一意の32バイトコードです。Hooksに設定された名前空間が異なっていれば、同じ状態キーを2つの異なるHooksが使用しても互いに干渉しません。                                                                                                                                                                                         |
| Parameters               | Hookに任意で設定できるインストール時パラメーターです。                                                                                                                                                                                                                                                                                                                                             |
| Reference Counting       | レジャー上の所有者のいないオブジェクトは参照カウントされる場合があります。これは、そのオブジェクトを参照（使用）していた最後のアカウントが参照を削除したときに、そのオブジェクトが削除されることを意味します。                                                                                                                                                                                                                                          |
| XFL or Floating Point    | 為替レート計算など、Hooksで高精度な数学計算を行う方法です。参照: [浮動小数点数（XFL）](/ja/docs/hooks/concepts/floating-point-numbers-xfl)。                                                                                                                                                                                                                                                       |
| Serialized Objects (STO) | xahaudがレジャーオブジェクトを送信し保存する方法です。参照: [シリアライズ済みオブジェクト](/ja/docs/hooks/concepts/serialized-objects)。                                                                                                                                                                                                                                                                                                     |
| Slots and Keylets        | Slotsはレジャーオブジェクトを保持でき、Keyletsはそれらのオブジェクトを識別します。参照: [SlotsとKeylets](/ja/docs/hooks/concepts/slots-and-keylets)。                                                                                                                                                                                                                                                                                      |
| Trace                    | Hookからxrpld出力にログ行を出力する方法です。参照: [Hooksのデバッグ](/ja/docs/hooks/concepts/debugging-hooks)。                                                                                                                                                                                                                                                                                                    |
