### Hooksとは何か？

HooksはXahau向けに特別に設計された、小さく効率的なWebアセンブリモジュールです。Hooksは任意の言語（WebAssemblyにコンパイル可能なもの）で書くことができ、ほとんどのビジネスロジックとスマートコントラクトの概念をHookに実装できます。通常、HooksはCで書かれます。

HooksはXahauアカウントに`SetHook`トランザクションを使用して設定されます。アカウントにインストールされると、Hookは以下のことができます：

1. アカウントへの受信および送信トランザクションをブロックまたは許可する
2. そのアカウントのHookに固有の内部ステートとロジックを変更および維持する
3. アカウントを代表して新しいトランザクションを発行する

### 用語集

このHooksドキュメントとHooks APIは、なじみのない用語のセットを使用しています。迷ったときは以下のルックアップテーブルを使用してください。

| 用語                     | 説明                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Hook                     | <p><em>この用語はコンテキストに応じてさまざまなことを指します</em><br />1. <a href="/docs/hooks/concepts/sethook-transaction">SetHookトランザクション</a>タイプでXahauにアップロード可能なWebアセンブリバイナリ。<br />2. Xahauにアップロード済みで、Xahauアカウントに設定または構成されたWebアセンブリバイナリ。<br />3. そのようなバイナリの<a href="/docs/hooks/concepts/compiling-hooks">ソースコード</a>。</p> |
| Originating Transaction  | Hookを起動させたトランザクション。これはHookが設定されたアカウントへの、またはアカウントからのトランザクションのいずれかです。                                                                                                                                                                                                                                                                 |
| Originating Account      | Originating Transactionを送信したアカウント。                                                                                                                                                                                                                                                                                                                                                         |
| Hook Account             | 現在実行中のHookが存在するアカウント。これはHookを所有するアカウント、Hookを作成した[SetHookトランザクション](/docs/hooks/concepts/sethook-transaction)を実行したアカウント、および現在実行中のHookのHookステートが属するアカウントです。                                                                                                                                                                                                                        |
| Installer                | SetHookトランザクションを使用してHookを現在インストールしているアカウント。                                                                                                                                                                                                                                                                                                                           |
| Emitted Transaction      | OriginalトランザクションではなくHookの実行中にHookによって作成された新しいトランザクション。これらは通常、Originating Accountに資金を送り返すために使用されます。参照：[Emitted Transactions](/docs/hooks/concepts/emitted-transactions)。                                                                                                                                                                                |
| State                    | 32バイトキーから任意データへの、アカウントごとのキーと値のマップ。アカウントに存在するすべてのHooksは同じHookステートにアクセスして変更できます。HookステートはHook Account上に存在し、**Originating Account上ではない**ことに注意してください。参照：[ステート管理](/docs/hooks/concepts/state-management)。                                                                                                                                 |
| SetHook                  | Hooks amendmentで導入された新しいトランザクションタイプで、XahauアカウントにHookを設定します。参照：[SetHookトランザクション](/docs/hooks/concepts/sethook-transaction)。                                                                                                                                                                                                                                                    |
| Guards                   | HookにループとGlobal書く場合に使用する必要がある特別な制御メカニズム。参照：[ループとガーディング](/docs/hooks/concepts/loops-and-guarding)。                                                                                                                                                                                                                                                                         |
| Grants                   | Hook InstallerがHookステートをInstallerのアカウントで変更する権限を、別のアカウントまたは特定のHook（インストール場所に関わらず）に与えることができる特別な権限。                                                                                                                                                                                                                                    |
| Namespace                | 1セットのステートキーを別のセットから区別する一意の32バイトコード。Hookに設定された名前空間が異なれば、同じステートキーを2つの異なるHookで互いに干渉なく使用できます。                                                                                                                                                                                                                                                                                         |
| Parameters               | Hookにオプションで設定できるインストール時パラメーター。                                                                                                                                                                                                                                                                                                                                             |
| Reference Counting       | レジャー上の所有者なしオブジェクトは参照カウントされる場合があります。つまり、それを参照（使用）していた最後のアカウントが参照を削除すると削除されます。                                                                                                                                                                                                                                          |
| XFL または浮動小数点    | 為替レート計算などのHooksでの高精度数学の方法。参照：[浮動小数点数（XFL）](/docs/hooks/concepts/floating-point-numbers-xfl)。                                                                                                                                                                                                                                                                       |
| Serialized Objects (STO) | xahaudがレジャーオブジェクトを送信および保存する方法。参照：[シリアライズされたオブジェクト](/docs/hooks/concepts/serialized-objects)。                                                                                                                                                                                                                                                                                                     |
| Slots and Keylets        | スロットはレジャーオブジェクトを含むことができ、Keyletはそれらのオブジェクトを識別します。参照：[スロットとKeylet](/docs/hooks/concepts/slots-and-keylets)。                                                                                                                                                                                                                                                                                      |
| Trace                    | Hookからxrpldへのログ行の出力方法。参照：[Hooksのデバッグ](/docs/hooks/concepts/debugging-hooks)。                                                                                                                                                                                                                                                                                                                    |
