### コンセプト

<LinkButton href="/ja/docs/hooks/concepts/floating-point-numbers-xfl">浮動小数点数 (XFL)</LinkButton>
<LinkButton href="/ja/docs/hooks/concepts/serialized-objects">シリアライズ済みオブジェクト</LinkButton>

### 動作

<Tabs>
<TabItem label="C">
* XFL浮動小数点数と、任意でフィールドコードおよび通貨コードを読み取ります
* 指定されたパラメーターに従って、シリアライズ済み金額を`write_ptr`に書き込みます
</TabItem>

<TabItem label="JavaScript">
* 浮動小数点数表現を指定されたフィールドに保存します。
* エラーコード、または更新済みの値を数値の配列として返します。
</TabItem>
</Tabs>

### 定義

<Tabs>
<TabItem label="C">
```c
int64_t float_sto (
    uint32_t write_ptr,
    uint32_t write_len,
    uint32_t cread_ptr,
    uint32_t cread_len,  
    uint32_t iread_ptr,
    uint32_t iread_len,  
    int64_t float1,
    uint32_t field_code
);
```


</TabItem>

<TabItem label="JavaScript">
```javascript
function float_sto(
    currency: ByteArray | HexString | undefined,
    issuer: ByteArray | HexString | undefined,
    f1: bigint,
    field_code: number
  ): ErrorCode | ByteArray
```
</TabItem>
</Tabs>



### 例

<Tabs>
<TabItem label="C">
```c
#define SBUF(str) (uint32_t)(str), sizeof(str)
uint8_t amt_out[48];
if (float_sto(SBUF(amt_out),
    SBUF(currency), SBUF(hook_accid), pusd_to_send, -1) < 0)
        rollback(SBUF("Peggy: Could not dump pusd amount into sto"), 1);
```


</TabItem>

<TabItem label="JavaScript">
```javascript
const amt_out = float_sto(currency, hook_accid, pusd_to_send, -1)
if (typeof amt_out === 'number')
        rollback("Peggy: Could not dump pusd amount into sto", 1)
```
</TabItem>
</Tabs>



### パラメーター

<Tabs>
<TabItem label="C">
| 名前        | 型        | 説明                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ----------- | --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| write_ptr  | uint32_t | シリアライズ済み金額フィールドを保存するのに十分なサイズのバッファへのポインター。少なくとも48バイトを推奨します。                                                                                                                                                                                                                                                                                                                                                                            |
| write_len  | uint32_t | 出力バッファの長さ。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| cread_ptr  | uint32_t | 出力にシリアライズする通貨コードを含むバッファへのポインター。_nullでもかまいません。_                                                                                                                                                                                                                                                                                                                                                                                                       |
| cread_len  | uint32_t | 通貨コードの長さ。20、3、または0（null）である必要があります。                                                                                                                                                                                                                                                                                                                                                                                                                                |
| iread_ptr  | uint32_t | 出力にシリアライズする発行者のAccount IDを含むバッファへのポインター。_nullでもかまいません。_                                                                                                                                                                                                                                                                                                                                                                                               |
| iread_len  | uint32_t | 発行者のAccount IDの長さ。20または0（null）のいずれかである必要があります。                                                                                                                                                                                                                                                                                                                                                                                                                   |
| float1      | int64_t  | シリアライズするXFL浮動小数点格納数。                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| field_code | uint32_t | <p>シリアライズ済み金額の前に付ける<code>sf</code>フィールドコード。例: <code>sfAmount</code>。<br />このフィールドが<code>0xFFFFFFFFU</code>（つまり<code>(uint32_t)(-1)</code>）の場合、出力の前にフィールドコードは付加されず、発行者や通貨コードも追加されませんが、シリアライズは浮動小数点金額として進行します。<br />このフィールドが0の場合、出力の前にフィールドコードは付加されず、発行者や通貨コードも追加されませんが、シリアライズは金額が浮動小数点ではなくXRPネイティブ金額であるかのように進行します。</p> |

<Aside type="tip">
`XAH`金額を出力するには、出力バッファにフィールドコードを事前に設定し、新しい開始位置まで進めた出力バッファと`0`をfield_codeとして渡します
</Aside>
</TabItem>

<TabItem label="JavaScript">


| 名前        | 型                                  | 説明                                                       |
| ----------- | ----------------------------------- | ---------------------------------------------------------- |
| currency    | ByteArray \| HexString \| undefined | 保存先の現在値。                                           |
| issuer      | ByteArray \| HexString \| undefined | 保存する値。                                               |
| f1          | bigint                              | 浮動小数点数を保存する場所を示すフィールドコード。         |
| field_code | number                              | エラーコード、または更新済みの値を数値の配列として返します。 |
</TabItem>
</Tabs>



### 戻り値

| 型       | 説明                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |

| int64_t | <p>出力バッファに書き込まれたバイト数。<br /><br />負の場合はエラー:<br /><code>INVALID_FLOAT</code><br />-指定された浮動小数点数が有効なXFL格納数ではありません<br /><br /><code>OUT_OF_BOUNDS</code><br />-ポインター/長さがHookメモリ外を指定しています。<br /><br /><code>INVALID_ARGUMENT</code><br />- <code>XRP</code>として、または<code>field code</code>なしで出力するよう指示された場合、書き込み用以外のすべてのポインターと長さは0（null）である必要があります。<br /><br /><code>TOO_SMALL</code><br />-出力バッファが小さすぎてシリアライズ済みオブジェクトを受け取れませんでした。<br /><br /><code>XFL_OVERFLOW</code><br />-出力の表現により正規化中にオーバーフローが発生しました。</p> |


</TabItem>
