### 概念

<LinkButton href="/ja/docs/hooks/concepts/introduction/">はじめに</LinkButton>
<LinkButton href="/ja/docs/hooks/concepts/execution-metadata">実行メタデータ</LinkButton>

### 動作

Hookの実行を成功で終了します。

* トランザクションメタデータに戻り文字列と戻りコードを記録します。
* すべての状態変更を確定します。
* すべての`emit()`トランザクションを送信します。
* 発信トランザクションの継続を許可します。

<Aside type="caution">
発信トランザクションが別の理由で停止した場合、このacceptはrollbackになります。参照:実行順序。
</Aside>

### 定義

<Tabs>
<TabItem label="C">
```c
int64_t accept (
    uint32_t read_ptr,
    uint32_t read_len,
    uint64_t error_code
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function accept(msg: string, code: number): number
```
</TabItem>
</Tabs>



### 例

<Tabs>
<TabItem label="C">
```c
accept("Success", 7, 100);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
accept('Success', 100)
```
</TabItem>
</Tabs>



### パラメーター

<Tabs>
<TabItem label="C">
| 名前       | 型        | 説明                                                                                                                                                                               |
| ---------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| read_ptr   | uint32_t | <p>実行メタデータに格納される戻り文字列へのポインターです。<br />これは、Hook開発者が受理時に返したい任意の文字列です。<em>nullにしてもよい。</em></p>                      |
| read_len   | uint32_t | 戻り文字列の長さです。最大32です。_nullにしてもよい。_                                                                                                                        |
| error_code | uint64_t | <p>このHook固有の戻りコードで、実行メタデータに格納されます。<br /><br />*nixシステム上のアプリケーションの戻りコードに似ています。慣例上、成功は0です。</p> |
</TabItem>

<TabItem label="JavaScript">
| 名前 | 型      | 説明                                                                                                                                                                               |
| ---- | ------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| msg  | string  | <p>実行メタデータに格納される文字列です。<br />これは、Hook開発者が受理時に返したい任意の文字列です。<em>nullにしてもよい。</em></p>                               |
| code | number  | <p>このHook固有の戻りコードで、実行メタデータに格納されます。<br /><br />*nixシステム上のアプリケーションの戻りコードに似ています。慣例上、成功は0です。</p> |
</TabItem>
</Tabs>



### 戻りコード

<Tabs>
<TabItem label="C">
| 型       | 説明                                                                                                                                                         |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| int64_t | AcceptはHookを終了するため、呼び出し元には値は返されません。慣例としてすべてのHook APIは`int64_t`を返しますが、この場合は何も返されません。 |
</TabItem>

<TabItem label="JavaScript">
| 型     | 説明                                                                                                                                                  |
| ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| number | AcceptはHookを終了するため、呼び出し元には値は返されません。慣例としてすべてのHook APIは`number`を返しますが、この場合は何も返されません。 |
</TabItem>
</Tabs>
