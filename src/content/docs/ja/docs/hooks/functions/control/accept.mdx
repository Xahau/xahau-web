### 動作

Hook の実行を成功状態で終了します。

* 戻り文字列と戻りコードをトランザクションメタデータに記録します。
* すべての状態変更を確定します。
* すべての `emit()` トランザクションを送信します。
* 発信元トランザクションの続行を許可します。

<Aside type="caution">
発信元トランザクションが別の理由で停止した場合、この accept は rollback になります。参照: Execution Order。
</Aside>

### 定義

<Tabs>
<TabItem label="C">
```c
int64_t accept (
    uint32_t read_ptr,
    uint32_t read_len,
    uint64_t error_code
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function accept(msg: string, code: number): number
```
</TabItem>
</Tabs>

### 例

<Tabs>
<TabItem label="C">
```c
accept("Success", 7, 100);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
accept('Success', 100)
```
</TabItem>
</Tabs>

### パラメーター

<Tabs>
<TabItem label="C">
| Name       | Type     | Description                                                                                                                                              |
| ---------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| read_ptr   | uint32_t | <p>実行メタデータに保存する戻り文字列へのポインタです。<br />受け入れ時に Hook 開発者が返したい任意の文字列を指定できます。 <em>null でも構いません。</em></p> |
| read_len   | uint32_t | 戻り文字列の長さです。最大 32。 _null でも構いません。_                                                                                                  |
| error_code | uint64_t | <p>この Hook 固有の戻りコードで、実行メタデータに保存されます。<br /><br />*nix システム上のアプリケーションの終了コードに似ています。慣例上、成功は 0 です。</p> |
</TabItem>

<TabItem label="JavaScript">
| Name | Type   | Description                                                                                                                                          |
| ---- | ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| msg  | string | <p>実行メタデータに保存する文字列です。<br />受け入れ時に Hook 開発者が返したい任意の文字列を指定できます。 <em>null でも構いません。</em></p>         |
| code | number | <p>この Hook 固有の戻りコードで、実行メタデータに保存されます。<br /><br />*nix システム上のアプリケーションの終了コードに似ています。慣例上、成功は 0 です。</p> |
</TabItem>
</Tabs>

### 戻りコード

<Tabs>
<TabItem label="C">
| Type    | Description                                                                                                                                    |
| ------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| int64_t | Accept は Hook を終了するため、呼び出し元には値を返しません。慣例としてすべての Hook API は `int64_t` を返しますが、この場合は何も返しません。 |
</TabItem>

<TabItem label="JavaScript">
| Type   | Description                                                                                                                                |
| ------ | ------------------------------------------------------------------------------------------------------------------------------------------ |
| number | Accept は Hook を終了するため、呼び出し元には値を返しません。慣例としてすべての Hook API は数値を返しますが、この場合は何も返しません。 |
</TabItem>
</Tabs>
