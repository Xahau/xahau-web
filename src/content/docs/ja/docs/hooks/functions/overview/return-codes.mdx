### 戻りコードの設計

WebAssemblyには例外（trap）がありますが、この言語機能はHookでは使われません。代わりに、どのHook APIからも戻る方法は1つだけです（すべてのHook APIは `noexcept` であると考えても構いません）。

効率的なエラー処理のために:

1. すべてのHook API関数は _符号付き_ 整数を返します。
2. すべての負の戻りコードはエラーです。
3. `0` 以上の戻りコードは、関数固有の出力であり、通常は読み書きしたバイト数ですが、必ずしもそうとは限りません。

### エラーコード

エラーコードはすべてのHook APIで共通であり、下の表にあります。

| 名前                               | 値      | 説明                                                                                                                                                                                                                                                                     |
| ---------------------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| SUCCESS                            | >= 0   | 負でない戻りコードは常に成功を意味し、通常は具体的なAPIに応じて、書き込まれたバイト数または実行されたイベント数を示します。                                                                                                                                      |
| OUT_OF_BOUNDS                      | -1     | パラメーターとして与えられたポインターまたはバッファ長が、Hookの許可されたメモリ領域の外を指しています。                                                                                                                                                           |
| INTERNAL_ERROR                     | -2     | 内部の不変条件違反用に予約されています。通常は入力とは関係ありません。これらはissueとして報告してください。                                                                                                                                                      |
| TOO_BIG                            | -3     | 許可された範囲より大きいパラメーターまたは値を設定しようとしました。                                                                                                                                                                                                   |
| TOO_SMALL                          | -4     | 指定された `write_len` が小さすぎたため、APIは `write_ptr` に出力を生成できませんでした。                                                                                                                                                                           |
| DOESNT_EXIST                       | -5     | 要求されたオブジェクトまたは項目が見つかりませんでした。                                                                                                                                                                                                                 |
| NO_FREE_SLOTS                      | -6     | Hookが項目をSlotに割り当てようとしましたが、空きSlotがありませんでした。既存のSlotを再利用するようにしてください。最大Slot数は255です。                                                                                                            |
| INVALID_ARGUMENT                   | -7     | APIへの1つ以上のパラメーターが、そのAPI個別の仕様に照らして無効でした。                                                                                                                                                                                           |
| ALREADY_SET                        | -8     | 一部のAPIでは、1回の実行につき1回だけ設定できるパラメーターがあります。1回目以降の設定試行はこのエラーになります。                                                                                                                                                |
| PREREQUISITE_NOT_MET               | -9     | APIの呼び出しが許可される前に、Hookが何かを実行している必要があります。APIのドキュメントを確認してください。                                                                                                                                                    |

| TOO_MANY_NONCES                    | -12    | Hookは1回の実行につき `nonce()` を最大256回までしか使えません。それ以上呼び出すとこのエラーコードになります。                                                                                                                                                   |

| NOT_IMPLEMENTED                    | -14    | Hooksが開発中の間、このAPIの一部または全部が計画済みだがまだ実装されていない場合に、この値を返すことがあります。                                                                                                                                              |
| INVALID_ACCOUNT                    | -15    | 20バイトのAccount IDを受け付けるAPIは、Account IDが何らかの理由で有効でないと判断した場合にこの値を返すことがあります。                                                                                                                                     |
| GUARD_VIOLATION                    | -16    | Hook内のすべてのループは、ループの先頭で、最初の非自明な命令として、分岐命令の前に、そのループの約束された最大反復回数を宣言しなければなりません。この約束が破られると、Hookはこのエラーコードで直ちに終了します。 |
| INVALID_FIELD                      | -17    | 要求されたシリアライズ済みフィールドが、指定したオブジェクト内に見つかりませんでした。                                                                                                                                                                                 |
| PARSE_ERROR                        | -18    | シリアライズ済みコンテンツの解析中にエラーが発生しました（通常は、シリアライズが無効なオブジェクトを示します）。                                                                                                                                                    |
| RC_ROLLBACK                        | -19    | ロールバックイベントを伝えるために内部で使用されます。                                                                                                                                                                                                                |

| EXPONENT_OVERSIZED                 | -28    | XFLを作成しようとしたとき、指数は80を超えてはなりません。                                                                                                                                                                                                          |
