---
title: slot_set
description: Localiza um objeto com base em seu keylet e o coloca em um slot
---
import { Tabs, TabItem } from '@astrojs/starlight/components';

### Comportamento

<Tabs>
<TabItem label="C">
* Localiza um objeto a partir do Keylet fornecido em `read_ptr`
* Posiciona o objeto localizado no slot especificado ou em um novo slot, caso nenhum slot (zero) seja especificado
</TabItem>

<TabItem label="JavaScript">
* Define os dados para o slot especificado.
* Retorna um código de erro ou o resultado da operação de definição.
</TabItem>
</Tabs>



### Definição

<Tabs>
<TabItem label="C">
```c
int64_t slot_set (
    uint32_t read_ptr,
    uint32_t read_len,
    uint32_t slot_no
);
```


</TabItem>

<TabItem label="JavaScript">
```javascript
function slot_set(
    kl: ByteArray | HexString,
    slotno: number
  ): ErrorCode | number
```
</TabItem>
</Tabs>



### Exemplo

<Tabs>
<TabItem label="C">
```c
int64_t slot_no = 
  slot_set(keylet, 34, 0);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
const slot_no = slot_set(keylet, 0)
```
</TabItem>
</Tabs>



### Parâmetros

<Tabs>
<TabItem label="C">
| Nome      | Tipo      | Descrição                                                                                     |
| --------- | --------- | ----------------------------------------------------------------------------------------------- |
| read_ptr | uint32_t | Ponteiro para um buffer contendo o keylet do objeto a ser localizado. Também pode ser um hash de transação. |
| read_len | uint32_t | Tamanho do buffer de leitura. Deve ser sempre 32 ou 34.                                           |
| slot_no  | uint32_t | O número do slot no qual posicionar o objeto, ou 0 caso deseje que o próximo disponível seja escolhido. |
</TabItem>

<TabItem label="JavaScript">
| Nome   | Tipo                   | Descrição                                               |
| ------ | ---------------------- | --------------------------------------------------------- |
| kl     | ByteArray \| HexString | Os dados a serem definidos no slot, podendo ser um array ou uma string. |
| slotno | number                 | O número do slot para o qual definir os dados.                          |
</TabItem>
</Tabs>



### Código de Retorno

<Tabs>
<TabItem label="C">
| Tipo     | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| int64_t | <p>O número do slot em que o objeto foi inserido<br /><br />Se negativo, um erro:<br /><code>OUT_OF_BOUNDS</code><br />- ponteiros/tamanhos especificados fora da memória do hook.<br /><br /><code>INVALID_ARGUMENT</code><br />- <code>read_len</code> deve ser 32 ou 34 bytes, dependendo se um hash de transação ou um keylet está sendo usado em <code>read_ptr</code><br />- o hash ou keylet era inválido<br /><br /><code>DOESNT_EXIST</code><br />- o objeto solicitado não foi encontrado</p> |
</TabItem>

<TabItem label="JavaScript">
| Tipo                | Descrição                                               |
| ------------------- | --------------------------------------------------------- |
| ErrorCode or number | Retorna um código de erro ou o resultado da operação de definição. |
</TabItem>
</Tabs>
