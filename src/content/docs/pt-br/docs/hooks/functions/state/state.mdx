---
title: state
description: >-
  Recupera os dados apontados por uma chave do Estado do Hook e os escreve em
  um buffer de saída
---
import { Tabs, TabItem, Aside } from '@astrojs/starlight/components';

### Comportamento

<Tabs>
<TabItem label="C">
* Lê uma chave do Estado do Hook de 32 bytes a partir de `kread_ptr`
* Escreve os dados (valor) dessa chave no buffer apontado por `write_ptr`
</TabItem>

<TabItem label="JavaScript">
* Recupera o valor do Estado do Hook associado à chave especificada.
* Retorna um código de erro ou o valor do Estado do Hook para a chave.
</TabItem>
</Tabs>

### Definição

<Tabs>
<TabItem label="C">
```c
int64_t state (
    uint32_t write_ptr,
    uint32_t write_len,
    uint32_t kread_ptr,
    uint32_t kread_len  
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function state(key: ByteArray | HexString): ErrorCode | ByteArray
```
</TabItem>
</Tabs>



### Exemplo

<Tabs>
<TabItem label="C">
```c
#define SBUF(str) (uint32_t)(str), sizeof(str)
if (state(SBUF(vault), SBUF(vault_key)) != 16)
    rollback(SBUF("Error: could not read state!"), 1);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
const value = state(SBUF(vault), SBUF(vault_key))
if (typeof value === 'number' || value.length != 16)
    rollback("Error: could not read state!", 1);
```
</TabItem>
</Tabs>



### Parâmetros

<Tabs>
<TabItem label="C">
| Nome       | Tipo      | Descrição                                                       |
| ---------- | --------- | ----------------------------------------------------------------- |
| write_ptr | uint32_t | Um ponteiro para o buffer no qual escrever os dados do Estado do Hook. |
| write_len | uint32_t | O tamanho do buffer de escrita.                                   |
| kread_ptr | uint32_t | Ponteiro para um buffer contendo a chave do Estado do Hook.                |
| kread_len | uint32_t | O tamanho da chave do Estado do Hook. (Deve ser 32.)                 |

<Aside type="tip">
Certifique-se de verificar o valor de retorno. Uma busca de estado pode falhar por uma série de motivos, e o buffer então conterá o que continha antes da chamada (normalmente tudo zeros).
</Aside>
</TabItem>

<TabItem label="JavaScript">
| Nome | Tipo                   | Descrição                                           |
| ---- | ---------------------- | ----------------------------------------------------- |
| key  | ByteArray or HexString | A chave do Estado do Hook da qual recuperar o valor. |
</TabItem>
</Tabs>



### Código de Retorno

<Tabs>
<TabItem label="C">
| Tipo     | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| int64_t | <p>O número de bytes escritos no buffer de escrita.<br /><br />Se negativo, um erro:<br /><code>OUT_OF_BOUNDS</code><br />- ponteiros/tamanhos especificados fora da memória do hook.<br /><br /><code>DOESNT_EXIST</code><br />- a chave do Estado do Hook especificada não possui um valor associado no ledger no momento da chamada.<br /><br /><code>TOO_BIG</code><br />- a chave especificada por <code>read_ptr</code> e <code>read_len</code> era maior que 32 bytes.<br /><br /><code>TOO_SMALL</code><br />- o buffer de saída era muito pequeno para armazenar os dados do Estado do Hook.</p> |
</TabItem>

<TabItem label="JavaScript">
| Tipo                   | Descrição                                                |
| ---------------------- | ---------------------------------------------------------- |
| ByteArray or ErrorCode | Retorna um código de erro ou o valor do Estado do Hook para a chave. |
</TabItem>
</Tabs>
