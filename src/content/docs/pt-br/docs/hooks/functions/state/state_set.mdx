---
title: state_set
description: Define o Estado do Hook para uma determinada chave e valor
---
import { Tabs, TabItem, Aside } from '@astrojs/starlight/components';

### Comportamento

<Tabs>
<TabItem label="C">
* Lê uma chave do Estado do Hook de 32 bytes a partir de `kread_ptr`
* Lê uma quantidade arbitrária de dados a partir de `read_ptr` (o valor)
* Atualiza a chave do Estado do Hook com o valor
</TabItem>

<TabItem label="JavaScript">
* Define o Estado do Hook com o valor e chave especificados.
* Retorna o número de bytes escritos no Estado do Hook (o tamanho dos dados), negativo em caso de erro.
</TabItem>
</Tabs>

### Definição

<Tabs>
<TabItem label="C">
```c
int64_t state_set (
    uint32_t read_ptr,
    uint32_t read_len,
    uint32_t kread_ptr,
    uint32_t kread_len  
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function state_set(
    value: ByteArray | HexString | undefined | null,
    key: ByteArray | HexString
  ): ErrorCode | number
```
</TabItem>
</Tabs>



### Exemplo

<Tabs>
<TabItem label="C">
```c
#define SBUF(str) (uint32_t)(str), sizeof(str)
if (state_set(SBUF(vault), SBUF(vault_key)) < 0)
		rollback(SBUF("Error: could not set state!"), 1);
```

<Aside type="tip">
Para excluir o estado, use `state_set(0, 0, SBUF(key);`.
</Aside>
</TabItem>

<TabItem label="JavaScript">
```javascript
const result = state_set(vault, vault_key)
if (result < 0)
    rollback("Error: could not set state!", 1);
```

<Aside type="tip">
Para excluir o estado, use `state_set(null, key);`.
</Aside>
</TabItem>
</Tabs>



### Parâmetros

<Tabs>
<TabItem label="C">
| Nome       | Tipo      | Descrição                                                                                                                                              |
| ---------- | --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| read_ptr  | uint32_t | <p>Ponteiro para os dados (valor) a serem escritos no Estado do Hook.<br />Se for <code>0</code> (nulo), então os dados nessa chave são excluídos. <em>Pode ser nulo.</em></p> |
| read_len  | uint32_t | <p>O tamanho dos dados.<br />Se for <code>0</code> (nulo), então os dados nessa chave são excluídos. <em>Pode ser nulo.</em></p>                               |
| kread_ptr | uint32_t | Um ponteiro para a chave do Estado do Hook na qual armazenar o valor.                                                                                             |
| kread_len | uint32_t | O tamanho da chave. (Deve ser sempre 32.)                                                                                                            |

<Aside type="caution">
O Xahaud define internamente um tamanho máximo de dados do hook. No momento em que este texto foi escrito, e para a mainnet, esse valor está fixado em 256 bytes; no entanto, no futuro será um número votável pelos validadores.
</Aside>
</TabItem>

<TabItem label="JavaScript">
| Nome  | Tipo                   | Descrição                                     |
| ----- | ---------------------- | ----------------------------------------------- |
| value | ByteArray or HexString | O valor dos dados a persistir.                   |
| key   | ByteArray or HexString | A chave do Estado do Hook para a qual definir o valor. |
</TabItem>
</Tabs>



### Código de Retorno

<Tabs>
<TabItem label="C">
| Tipo     | Descrição                                                                                                                                                                                                                                                                                                                                                                                                             |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| int64_t | <p>O número de bytes escritos no Estado do Hook (o tamanho dos dados.)<br /><br />Se negativo, um erro:<br /><code>OUT_OF_BOUNDS</code><br />- ponteiros/tamanhos especificados fora da memória do hook.<br /><br /><code>TOO_BIG</code><br />- <code>kread_len</code> era maior que 32, ou<br />- <code>read_len</code> era maior que o tamanho máximo de dados do hook.<br /><br /><code>TOO_SMALL</code><br />- <code>kread_len</code> era 0.</p> |
</TabItem>

<TabItem label="JavaScript">
| Tipo                | Descrição                                                                                    |
| ------------------- | ---------------------------------------------------------------------------------------------- |
| number or ErrorCode | Retorna o número de bytes escritos no Estado do Hook (o tamanho dos dados), negativo em caso de erro. |
</TabItem>
</Tabs>
