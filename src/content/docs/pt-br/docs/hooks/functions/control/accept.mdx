### Conceitos

<LinkButton href="/pt-br/docs/hooks/concepts/introduction/">Introdução</LinkButton>
<LinkButton href="/pt-br/docs/hooks/concepts/execution-metadata">Metadados de Execução</LinkButton>

### Comportamento

Encerra a execução do hook com status: sucesso.

* Registra uma string de retorno e um código de retorno nos metadados da transação.
* Confirma todas as alterações de estado.
* Envia todas as transações `emit()`.
* Permite que a transação de origem continue.

<Aside type="caution">
Se a transação de origem for interrompida por algum outro motivo, este accept se torna um rollback. Veja: Ordem de Execução.
</Aside>

### Definição

<Tabs>
<TabItem label="C">
```c
int64_t accept (
    uint32_t read_ptr,
    uint32_t read_len,
    uint64_t error_code
);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
function accept(msg: string, code: number): number
```
</TabItem>
</Tabs>



### Exemplo

<Tabs>
<TabItem label="C">
```c
accept("Sucesso", 7, 100);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
accept('Sucesso', 100)
```
</TabItem>
</Tabs>



### Parâmetros

<Tabs>
<TabItem label="C">
| Nome        | Tipo      | Descrição                                                                                                                                                                         |
| ----------- | --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| read_ptr   | uint32_t | <p>Ponteiro para uma string de retorno a ser armazenada nos metadados de execução.<br />Esta é qualquer string que o desenvolvedor do hook desejar retornar com a aceitação. <em>Pode ser nulo.</em></p>          |
| read_len   | uint32_t | O tamanho da string de retorno. No máximo 32. _Pode ser nulo._                                                                                                                         |
| error_code | uint64_t | <p>Um código de retorno específico deste hook a ser armazenado nos metadados de execução.<br /><br />Semelhante ao código de retorno de uma aplicação em um sistema *nix. Por convenção, sucesso é zero.</p> |
</TabItem>

<TabItem label="JavaScript">
| Nome | Tipo   | Descrição                                                                                                                                                                         |
| ---- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| msg  | string | <p>String a ser armazenada nos metadados de execução.<br />Esta é qualquer string que o desenvolvedor do hook desejar retornar com a aceitação. <em>Pode ser nulo.</em></p>                              |
| code | number | <p>Um código de retorno específico deste hook a ser armazenado nos metadados de execução.<br /><br />Semelhante ao código de retorno de uma aplicação em um sistema *nix. Por convenção, sucesso é zero.</p> |
</TabItem>
</Tabs>



### Código de Retorno

<Tabs>
<TabItem label="C">
| Tipo     | Descrição                                                                                                                                             |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| int64_t | Accept encerra o hook, portanto nenhum valor é retornado ao chamador. Por convenção, todas as APIs de Hook retornam `int64_t`, mas neste caso nada é retornado. |
</TabItem>

<TabItem label="JavaScript">
| Tipo   | Descrição                                                                                                                                              |
| ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| number | Accept encerra o hook, portanto nenhum valor é retornado ao chamador. Por convenção, todas as APIs de Hook retornam `number`, mas neste caso nada é retornado. |
</TabItem>
</Tabs>
