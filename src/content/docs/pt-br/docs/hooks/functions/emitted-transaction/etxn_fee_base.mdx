<Aside type="caution">
As taxas em um ledger com Hooks habilitados não são triviais. Veja: [Taxas de Hook](/pt-br/docs/hooks/concepts/hook-fees) para detalhes.
</Aside>

### Conceitos

<LinkButton href="/pt-br/docs/hooks/concepts/emitted-transactions">Transações Emitidas</LinkButton>

### Comportamento

* Retorna o valor da taxa em drops recomendada para uma transação a ser emitida.

### Definição

<Tabs>
<TabItem label="C">
```c
int64_t etxn_fee_base (
    uint32_t read_ptr,
    uint32_t read_len
);
```


</TabItem>

<TabItem label="JavaScript">
```javascript
function etxn_fee_base(txblob: ByteArray | HexString): ErrorCode | number
```
</TabItem>
</Tabs>



### Exemplo

<Tabs>
<TabItem label="C">
```c
int64_t fee_to_pay =
    etxn_fee_base(tx_blob, tx_blob_len);
```


</TabItem>

<TabItem label="JavaScript">
```javascript
const fee_to_pay = etxn_fee_base(tx_blob)
```
</TabItem>
</Tabs>



### Parâmetros

<Tabs>
<TabItem label="C">
<table><thead><tr><th>Nome</th><th width="118">Tipo</th><th>Descrição</th></tr></thead><tbody><tr><td>read_ptr</td><td>uint32_t</td><td>Ponteiro para o buffer contendo a transação serializada que você pretende emitir. O campo fee é obrigatório, mas ignorado (você pode usar zero). Use a saída desta função para preencher o campo fee corretamente.</td></tr><tr><td>read_len</td><td>uint32_t</td><td>O tamanho do tx blob.</td></tr></tbody></table>


</TabItem>

<TabItem label="JavaScript">


<table><thead><tr><th>Nome</th><th width="118">Tipo</th><th>Descrição</th></tr></thead><tbody><tr><td>txblob</td><td>ByteArray | HexString</td><td>O blob da transação, que pode ser um array de números ou uma string.</td></tr></tbody></table>
</TabItem>
</Tabs>



### Código de Retorno

<Tabs>
<TabItem label="C">


| Tipo     | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| int64_t | <p>O menor número de drops que uma txn emitida precisaria para ser aceita.<br /><br />Se negativo, um erro:<br /><code>OUT_OF_BOUNDS</code><br />- O buffer fornecido não está validamente dentro da memória do hook.<br /><br /><code>PREREQUISITE_NOT_MET</code><br />- <code>etxn_reserve</code> não foi chamado primeiro.<br /><br /><code>INVALID_TXN</code><br />- O buffer fornecido não contém uma transação serializada válida. (A desserialização falhou, ou um campo obrigatório estava faltando.)</p> |
</TabItem>

<TabItem label="JavaScript">


| Tipo   | Descrição                                                               |
| ------ | ------------------------------------------------------------------------- |
| number | Um ErrorCode se houver um erro, ou a taxa base calculada em caso de sucesso. |
</TabItem>
</Tabs>
