### Conceitos

<LinkButton href="/pt-br/docs/hooks/concepts/serialized-objects">Objetos Serializados</LinkButton>

### Comportamento

<Tabs>
<TabItem label="C">
* Analisa um STObject apontado por `read_ptr`
* Encontra o campo especificado por `field_id`
* Se o campo for encontrado, e:
*
  1. For um array, então retorna o início e o tamanho do array incluindo os bytes de leadin/leadout, ou
*
  2. **Não** for um array, então retorna o início e o tamanho do PAYLOAD do campo (excluindo os bytes de leadin).

<Aside type="caution" title="Codificação do Field ID">
As APIs `sto_` aceitam um parâmetro `field_id` codificado da seguinte forma: `(type << 16U) + field`
Assim, o tipo 1 campo 2 seria `0x10002U`.
</Aside>
</TabItem>

<TabItem label="JavaScript">
* Analisa um STObject na variável `sto`
* Encontra o campo especificado por `field_id`
</TabItem>
</Tabs>



### Definição

<Tabs>
<TabItem label="C">
```c
int64_t sto_subfield (
    uint32_t read_ptr,
    uint32_t read_len,
    uint32_t field_id
);
```


</TabItem>

<TabItem label="JavaScript">
```javascript
function sto_subfield(
    sto: ByteArray | HexString,
    field_id: number
  ): ErrorCode | bigint
```
</TabItem>
</Tabs>



### Exemplo

<Tabs>
<TabItem label="C">
```c
#define SUB_OFFSET(x) ((int32_t)(x >> 32))
#define SUB_LENGTH(x) ((int32_t)(x & 0xFFFFFFFFULL))
int64_t memos_lookup =
    sto_subfield(txn_ptr, txn_len, sfMemos);
if (memos_lookup < 0)
{
    // sfMemos was not found in the STObject pointed at by memo_ptr
}
else
{
    // sfMemos was found and its location is as follows:
    uint8_t* memos_ptr = SUB_OFFSET(memos_lookup) + memos_ptr;
    int64_t  memos_len = SUB_LENGTH(memos_lookup);
}
```

<Aside type="tip">
O hookmacro.h já contém as macros `SUB_OFFSET` e `SUB_LENGTH`.
</Aside>
</TabItem>

<TabItem label="JavaScript">
```javascript
const SUB_OFFSET = (x) => Number(x >> 32n)
const SUB_LENGTH = (x) => Number(x & 0xFFFFFFFFn)
const memos_lookup = sto_subfield(txn, sfMemos);
if (typeof memos_lookup === 'number')
{
    // sfMemos was not found in the STObject pointed at by memo_ptr
}
else
{
    // sfMemos was found and its location is as follows:
    const memo_start = SUB_OFFSET(memos_lookup)
    const memo_len = SUB_LENGTH(memos_lookup)
    const memo = txn.slice(memo_start, memo_len)
}
```
</TabItem>
</Tabs>



### Parâmetros

<Tabs>
<TabItem label="C">
| Nome      | Tipo      | Descrição                                                                                                                                                                                                                                                                                                                                                                                                |
| --------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| read_ptr | uint32_t | Ponteiro para o buffer contendo o STObject                                                                                                                                                                                                                                                                                                                                                              |
| read_len | uint32_t | Tamanho do STObject                                                                                                                                                                                                                                                                                                                                                                                         |
| field_id | uint32_t | <p>O código <code>sf</code> do campo que você está procurando.<br /><br />Para calcular isso manualmente, pegue o <code>type</code> serializado e desloque-o para os 16 bits mais altos de um uint32_t, depois pegue o <code>field</code> e coloque-o nos 16 bits mais baixos.<br /><br />Por exemplo:<br /><code>sfEmitNonce</code> tem <code>type</code> 5 e <code>field</code> 11, portanto seu valor é <code>0x050BU</code></p> |


</TabItem>

<TabItem label="JavaScript">


| Nome      | Tipo                   | Descrição                                                                     |
| --------- | ---------------------- | ------------------------------------------------------------------------------- |
| sto       | ByteArray or HexString | O objeto STO (dados de ledger codificados em binário) do qual o subcampo será extraído. |
| field_id | number                 | O ID do campo a ser extraído.                                            |
</TabItem>
</Tabs>



### Código de Retorno

<Tabs>
<TabItem label="C">
| Tipo     | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| int64_t | <p>A localização do campo dentro do buffer especificado:<br />- Os 32 bits superiores são o deslocamento (offset).<br />- Os 32 bits inferiores são o tamanho.<br /><br />Se negativo, um erro:<br /><code>OUT_OF_BOUNDS</code><br />- ponteiros/tamanhos especificados fora da memória do hook.<br /><br /><code>TOO_SMALL</code><br />- O buffer de entrada não é grande o suficiente para possivelmente conter um STObject válido.<br /><br /><code>DOESNT_EXIST</code><br />- O campo buscado não está presente no STObject fornecido.<br /><br /><code>PARSE_ERROR</code><br />- O STObject fornecido está malformado ou não é um STObject.</p> |


</TabItem>

<TabItem label="JavaScript">


| Tipo               | Descrição                                                                                                                                                                                              |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| bigint / ErrorCode | <p>A localização do campo dentro do buffer especificado:<br />- Os 32 bits superiores são o deslocamento (offset).<br />- Os 32 bits inferiores são o tamanho.</p><p></p><p>ou um código de erro se a extração falhar.</p> |
</TabItem>
