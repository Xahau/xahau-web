El tipo de objeto `DirectoryNode` proporciona una lista de enlaces a otros objetos en el ﾃ｡rbol de estado del ledger. Un solo _Directorio_ conceptual toma la forma de una lista doblemente enlazada, con uno o mﾃ｡s objetos DirectoryNode conteniendo cada uno hasta 32 IDs de otros objetos. El primer objeto se llama la raﾃｭz del directorio, y todos los demﾃ｡s objetos pueden agregarse o eliminarse segﾃｺn sea necesario.

Hay dos tipos de Directorios:

* Los **directorios de propietario** listan otros objetos que posee una cuenta, como objetos `RippleState` (lﾃｭnea de confianza) u `Offer`.
* Los **directorios de oferta** listan las ofertas disponibles en el intercambio descentralizado. Un solo directorio de oferta contiene todas las ofertas que tienen la misma tasa de cambio para el mismo token (cﾃｳdigo de moneda y emisor).

### Formatos de ID de Directorio

Hay tres fﾃｳrmulas diferentes para crear el ID de un DirectoryNode, dependiendo de cuﾃ｡l de las siguientes represente el DirectoryNode:

* La primera pﾃ｡gina (tambiﾃｩn llamada raﾃｭz) de un Directorio de Propietario
* La primera pﾃ｡gina de un Directorio de Oferta
* Pﾃ｡ginas posteriores de cualquier tipo

**La primera pﾃ｡gina de un Directorio de Propietario** tiene un ID que es el \[SHA-512Half]\[] de los siguientes valores, concatenados en orden:

* La clave de espacio del Directorio de Propietario (`0x004F`)
* El AccountID del campo `Owner`.

**La primera pﾃ｡gina de un Directorio de Oferta** tiene un ID especial: los 192 bits superiores definen el libro de ﾃｳrdenes, y los 64 bits restantes definen la tasa de cambio de las ofertas en ese directorio. (El ID es big-endian, por lo que el libro estﾃ｡ en los bits mﾃ｡s significativos, que van primero, y la calidad estﾃ｡ en los bits menos significativos que van al final.)

Especﾃｭficamente: los primeros 192 bits son los primeros 192 bits del \[SHA-512Half]\[] de los siguientes valores, concatenados en orden:

* La clave de espacio del Directorio de Libros (`0x0042`)
* El cﾃｳdigo de moneda de 160 bits de `TakerPaysCurrency`
* El cﾃｳdigo de moneda de 160 bits de `TakerGetsCurrency`
* El AccountID de `TakerPaysIssuer`
* El AccountID de `TakerGetsIssuer`

Los 64 bits inferiores del ID de un Directorio de Oferta representan el monto `TakerPays` dividido por el monto `TakerGets` de las ofertas en ese directorio como un nﾃｺmero de 64 bits en el formato de cantidad interna de Xahau.

**Si el DirectoryNode no es la primera pﾃ｡gina del Directorio** (independientemente de si es un Directorio de Propietario o de Oferta), entonces tiene un ID que es el \[SHA-512Half]\[] de los siguientes valores, concatenados en orden:

* La clave de espacio DirectoryNode (`0x0064`)
* El ID del DirectoryNode raﾃｭz
* El nﾃｺmero de pﾃ｡gina de este objeto. (Como 0 es el DirectoryNode raﾃｭz, este valor es un entero 1 o mayor.)
