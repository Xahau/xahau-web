---
title: otxn_field
description: Serializar y devolver un campo de la transacción origen
---
import { Tabs, TabItem, Aside } from '@astrojs/starlight/components';

### Comportamiento

<Tabs>
<TabItem label="C">
* Busca el campo `sf` especificado en la transacción origen
* Escribe la versión serializada del campo en el buffer de salida
</TabItem>

<TabItem label="JavaScript">
* Obtiene el valor de un campo específico en la transacción origen.
* Devuelve el valor del campo como un array de números, o un ErrorCode si falla.
</TabItem>
</Tabs>

### Definición

<Tabs>
<TabItem label="C">
```c
int64_t otxn_field (
    uint32_t write_ptr,
    uint32_t write_len,
    uint32_t field_id
);
````

</TabItem>

<TabItem label="JavaScript">
```javascript
function otxn_field(field_id: number): ErrorCode | ByteArray
```
</TabItem>
</Tabs>

### Ejemplo

<Tabs>
<TabItem label="C">
```c
int64_t account_field_len = 
    otxn_field(account_field, 20, sfAccount);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
const account_field = otxn_field(sfAccount)
```
</TabItem>
</Tabs>

### Parámetros

<Tabs>
<TabItem label="C">
| Nombre    | Tipo     | Descripción                                                                                                                                                                                                                                                                                                                                 |
| --------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| write_ptr | uint32_t | Puntero a un buffer adecuado para almacenar la salida                                                                                                                                                                                                                                                                                      |
| write_len | uint32_t | Longitud del buffer de salida                                                                                                                                                                                                                                                                                                               |
| field_id  | uint32_t | <p>Código <code>sf</code> del campo que se desea obtener.<br /><br />Para calcularlo manualmente: toma el <code>type</code> serializado y desplázalo a los 16 bits más altos de un uint32_t, luego coloca el <code>field</code> en los 16 bits más bajos.<br /><br />Ejemplo:<br /><code>sfEmitNonce</code> → <code>type</code>=5, <code>field</code>=11 → <code>0x050BU</code></p> |

<Aside type="caution" title="Importante">
El código de campo **no** se escribe en el buffer de salida, solo el **contenido (payload)** del campo.

En el momento de escribir esto, en Hooks Public Testnet, los campos `STI_ACCOUNT` como `sfAccount` se devuelven **sin el byte inicial de longitud variable**.

</Aside>
</TabItem>

<TabItem label="JavaScript">
| Nombre   | Tipo   | Descripción                                                                                                  |
| -------- | ------ | ------------------------------------------------------------------------------------------------------------ |
| field_id | number | Devuelve el valor del campo especificado como un array de números, o un error si falla la búsqueda          |
</TabItem>
</Tabs>

### Código de retorno

<Tabs>
<TabItem label="C">
| Tipo     | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                  |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| int64_t  | <p>Número de bytes escritos<br /><br />Si es negativo, error:<br /><code>OUT_OF_BOUNDS</code><br />- punteros fuera de la memoria del hook<br /><br /><code>TOO_SMALL</code><br />- el buffer de salida es demasiado pequeño<br /><br /><code>INVALID_FIELD</code><br />- el <code>field_id</code> no es válido<br /><br /><code>DOESNT_EXIST</code><br />- el campo no existe en la transacción origen</p> |
</TabItem>

<TabItem label="JavaScript">
| Tipo                   | Descripción                                                                                                  |
| ---------------------- | ------------------------------------------------------------------------------------------------------------ |
| ByteArray o ErrorCode  | Devuelve el valor del campo como un array de números, o un ErrorCode si falla la búsqueda                   |
</TabItem>
</Tabs>
