---
title: util_raddr
description: Convertir un Account ID de 20 bytes en una dirección r
---
import { Tabs, TabItem } from '@astrojs/starlight/components';

### Comportamiento

<Tabs>
<TabItem label="C">
* Lee un Account ID de 20 bytes desde `read_ptr`
* Escribe la dirección r equivalente para ese Account ID en `write_ptr`
</TabItem>

<TabItem label="JavaScript">
* Lee un Account ID de 20 bytes desde `accountid`
* Devuelve la dirección r equivalente para ese Account ID
</TabItem>
</Tabs>

### Definición

<Tabs>
<TabItem label="C">
```c
int64_t util_raddr (
    uint32_t write_ptr,
    uint32_t write_len,
    uint32_t read_ptr,
    uint32_t read_len
);
````

</TabItem>

<TabItem label="JavaScript">
```javascript
function util_raddr(accountid: ByteArray | HexString): ErrorCode | string
```
</TabItem>
</Tabs>

### Ejemplo

<Tabs>
<TabItem label="C">
```c
uint8_t raddr_out[40];
uint8_t acc_id[20] =
{
    0x2dU, 0xd8U, 0xaaU, 0xdbU, 0x4eU, 0x15U,               
    0xebU, 0xeaU,  0x0eU, 0xfdU, 0x78U, 0xd1U, 0xb0U,
    0x35U, 0x91U,  0x04U, 0x7bU, 0xfaU, 0x1eU,  0x0eU
};
int64_t bytes_written = 
    util_raddr(raddr_out, sizeof(raddr_out), acc_id, 20);
```
</TabItem>

<TabItem label="JavaScript">
```javascript
const raddr = util_raddr('8D329C03074A98EF0488AB6ABBF5883F68CCFD4E')
// o
const raddr = util_raddr([
    0x8D, 0x32, 0x9C, 0x03, 0x07, 0x4A, 0x98, 0xEF, 0x04, 0x88,
    0xAB, 0x6A, 0xBB, 0xF5, 0x88, 0x3F, 0x68, 0xCC, 0xFD, 0x4E
])
```
</TabItem>
</Tabs>

### Parámetros

<Tabs>
<TabItem label="C">

<table>
<thead>
<tr><th>Nombre</th><th>Tipo</th><th>Descripción</th></tr>
</thead>
<tbody>
<tr><td>write_ptr</td><td>uint32_t</td><td>Puntero a un buffer de tamaño suficiente para almacenar la dirección r. Se recomiendan al menos 35 bytes.</td></tr>
<tr><td>write_len</td><td>uint32_t</td><td>Longitud del buffer de salida.</td></tr>
<tr><td>read_ptr</td><td>uint32_t</td><td>Puntero al Account ID.</td></tr>
<tr><td>read_len</td><td>uint32_t</td><td>Longitud de la entrada. Siempre 20.</td></tr>
</tbody>
</table>

</TabItem>

<TabItem label="JavaScript">
<table>
<thead>
<tr><th>Nombre</th><th>Tipo</th><th>Descripción</th></tr>
</thead>
<tbody>
<tr><td>accountid</td><td>number[] / string</td><td>El Account ID en HEX que se convertirá a dirección r. Puede proporcionarse como array de números o como string.</td></tr>
</tbody>
</table>
</TabItem>
</Tabs>

### Código de retorno

<Tabs>
<TabItem label="C">
| Tipo     | Descripción |
| -------- | ----------- |
| int64_t | <p>Número de bytes escritos (longitud de la dirección r generada).<br /><br />Si es negativo, indica error:<br /><code>OUT_OF_BOUNDS</code><br />- punteros/longitudes fuera de la memoria del hook.<br /><br /><code>INVALID_ARGUMENT</code><br />- <code>read_len</code> no era 20.<br /><br /><code>TOO_SMALL</code><br />- <code>write_len</code> no era suficiente para almacenar la dirección r.</p> |
</TabItem>

<TabItem label="JavaScript">
| Tipo          | Descripción |
| ------------- | ----------- |
| string / number | Devuelve un ErrorCode si hay error, o la dirección r como string |
</TabItem>
</Tabs>
